## 1. 개요

### 1.1 플랫폼 소개
DreamIT 부트캠프는 HTML/CSS 기초부터 React 프론트엔드, 생성형 AI 웹 서비스(FastAPI·Hugging Face), 그리고 Git·Docker·클라우드 배포·협업까지를 하나의 SPA에서 학습하도록 구성한 한국어 실무형 웹 개발 학습 플랫폼이다. NOVAWORKS 디자인 아티팩트(블루 그라데이션 + 오렌지 포인트, Pretendard, 플로팅 필 헤더)를 React로 포팅하고, 저장소 `docs/`의 교재 자료를 토대로 커리큘럼 본문을 작성했다.

### 1.2 규모 (실측치)
데이터 파일을 직접 로드해 집계한 수치는 다음과 같다.

| 항목 | 수치 | 근거 |
|------|------|------|
| 트랙 | 4개 (web / react / ai / ops) | `src/data/curriculum.js` `TRACKS` |
| 챕터 | 27개 (web 5 · react 8 · ai 8 · ops 6) | `CHAPTERS.length` |
| 강의(섹션) | 137개 | `TOTAL_LESSONS` |
| 본문 마크다운 | 27개 챕터 전체 (`src/content/*.md`, 총 27,975줄) | `src/content/` |
| 실습 예제(Labs) | 16개 챕터, 42개 | `src/data/labs.js` |
| 프로젝트 | 18개 (react/ai/fullstack/ops, 입문~고급) | `src/data/projects.js` |
| 핵심 용어 | 216개 | `TOTAL_TERMS` |
| 퀴즈 문항 | 135개 (27개 챕터 전부 커버) | `TOTAL_QUIZ` |

콘텐츠 커버리지가 균일하다는 점이 특징이다. 27개 챕터 전부가 본문 마크다운과 퀴즈를 갖추고 있어, "데이터는 채웠지만 본문은 비어 있는" 흔한 미완성 패턴이 보이지 않는다.

> 참고: README와 일부 화면 텍스트가 구버전 수치("16개 챕터·62개 강의", 2개 트랙)를 그대로 두고 있어 실제 데이터(27/137/4)와 불일치한다. 7.2의 개선 항목 참조.

### 1.3 기술 스택
- 프론트엔드: React 18.3 + Vite 5.4, `react-router-dom` 6 기반 클라이언트 라우팅 SPA
- 콘텐츠 렌더링: `react-markdown` 9 + `remark-gfm`, `react-syntax-highlighter`(PrismLight)
- 백엔드/인증/DB: Supabase(`@supabase/supabase-js` 2) — Google·Kakao OAuth + 진도 저장(RLS)
- OG 이미지: `sharp` 기반 1200×630 생성 스크립트(`scripts/gen-og.mjs`)
- 스타일: 인라인 스타일 토큰(`src/theme.js`) + 최소 전역 CSS(`src/styles/global.css`, 75줄), 외부 UI 라이브러리 없음

### 1.4 배포 방식
GitHub Actions(`.github/workflows/deploy.yml`)로 `main` 푸시 시 GitHub Pages에 자동 배포한다. `npm ci → vite build → dist/index.html을 404.html로 복제 → upload-pages-artifact → deploy-pages` 흐름이며, SPA 딥링크 404 방지를 위한 404 폴백과 `concurrency` 취소 설정이 들어 있다. 커스텀 도메인은 `public/CNAME`(bootcamp.dreamitbiz.com)으로 고정. Supabase 키는 GitHub Secrets로 빌드 시 주입한다. OG 이미지는 CI(우분투)에 한글 폰트가 없어 깨지는 문제를 알고, 로컬 생성 후 커밋하는 운영 결정을 워크플로 주석으로 명시해 두었다.

## 2. 콘텐츠 평가

### 2.1 4개 트랙 구성과 깊이
- web(5장): HTML 구조 → CSS → 레이아웃 → JS 기초 → DOM/이벤트. React 진입 전 기본기 정비용.
- react(8장): JSX → Props/State → UI/디자인 → 이벤트/Effect/Context → 상태관리 → API/인증 → 설계 → 배포/실무.
- ai(8장): FastAPI/Hugging Face → Transformers pipeline → NLP → 이미지 생성 → 음성 → 컴퓨터 비전 → 멀티모달 → 배포·운영.
- ops(6장): Git/GitHub 협업 → FastAPI/Spring/Vue 배포 → AWS/Docker 등 실전 배포·협업.

본문 분량이 충실하다. 챕터당 본문이 평균 약 1,000줄이며 ai 트랙 일부(ai-03 2,001줄, ai-05 2,270줄)는 단일 강의 교재 수준의 깊이를 가진다. 각 챕터는 `summary`, `level`, `tags`, `duration`, 그리고 섹션별 `points`(핵심 요점)까지 구조화되어 있다.

### 2.2 코드/주석 품질
콘텐츠와 데모·프로젝트 코드 모두 한국어 라인 주석이 촘촘하다. 예: `react-01.md`의 JSX 예제는 props 구조 분해·단일 루트 노드·`map`+`key` 규칙을 각 줄 주석으로 설명한다. AI 프로젝트(`projectCode.js`의 `p-sentiment`)는 FastAPI `lifespan`으로 모델 1회 로드, Pydantic 검증, 배치 엔드포인트, 예외→HTTP 상태 매핑까지 갖춘 **실행 가능한 완성형 예제**이며 모든 라인에 주석이 달려 있다. 비전 예제(`p-vision`)는 ViT 모델, 업로드 검증 헬퍼, GPU/CPU device 분기, Top-K 응답 스키마까지 실무 패턴을 보여준다. 교육 코드로서 가독성과 정확성이 모두 높다.

### 2.3 실습 예제
챕터별 핸즈온 랩(`labs.js`)이 16개 챕터 42건 정의되어 있고, `title/desc/stack` 구조로 강의 하단(`Lesson.jsx`의 HANDS-ON LABS 섹션)에 카드로 노출된다. 다만 랩이 16개 챕터에만 존재해 web/일부 ops 챕터는 랩 카드가 비는 부분이 있다(전 챕터 균일 커버는 아님).

### 2.4 학습 흐름 (개념 → 실습 → 프로젝트 → 검증)
설계상 학습 동선이 명확하다. 헤더 네비게이션 자체가 "소개 → 커리큘럼 → 트랙별 강의 → 자료·실습 → 프로젝트 → 복습·퀴즈" 순으로 배치되어 있고(`Header.jsx` 주석에 흐름이 명시됨), 강의 본문 → 하단 실습 랩 → 프로젝트 상세의 "관련 학습 자료" 역링크(`ProjectDetail.jsx`가 `p.chapters`로 해당 챕터에 링크) → 퀴즈 해설의 "강의 보기" 링크까지 양방향으로 연결되어 개념과 검증이 순환한다.

## 3. 기능 평가

### 3.1 인증 (Google / Kakao)
`useAuth.jsx`에서 Supabase `signInWithOAuth`로 google·kakao 두 프로바이더를 지원하고, `redirectTo`를 현재 origin 기준으로 동적 구성해 로컬/운영 양쪽에서 동작한다. `onAuthStateChange` 시 `upsertProfile`로 프로필(이름·아바타·provider·last_login)을 동기화한다. 로그인/콜백 페이지는 `bare` 레이아웃(헤더·푸터 제거)으로 처리.

### 3.2 진도 저장 (Supabase RLS)
진도는 (user_id, chapter_id, section_no) 복합 PK의 `webpro_progress` 테이블에 섹션 단위로 upsert된다(`db.js`). `useProgress.jsx`는 Context로 진도를 관리하며 **낙관적 업데이트** 후 실패 시 reload로 롤백하는 패턴을 구현했다. 스키마(`schema.sql`)는 profiles·progress·enrollments 3개 테이블 전부에 RLS를 켜고 `auth.uid() = user_id` 정책(select/insert/update/delete)을 명시해 사용자별 데이터 격리가 적절하다. 공용 단일 Supabase 프로젝트 정책에 맞춰 `webpro_` 접두사를 일관 사용한다.

### 3.3 퀴즈·용어
퀴즈(`Quiz.jsx`)는 전체 랜덤 10문항 또는 챕터별 출제를 선택해 풀고, 객관식 채점·정오답 색상 피드백·해설·해당 강의 링크를 제공한다. 용어집은 216개 term을 검색·트랙 필터로 탐색하고 출처 챕터로 링크한다. 퀴즈 최고점은 `localStorage`(`webpro_quizbest_*`)에 저장된다 — 즉 **퀴즈 결과는 서버에 저장되지 않고 기기 로컬에만 남는다**(개선 항목 7.1).

### 3.4 프로젝트 라이브 데모
`src/demos/`에 실제 동작하는 React 앱 6종(Todo·Pomodoro·Markdown·Profile·Weather·Kanban)을 등록하고, `?raw` import로 같은 소스를 코드 탭에 그대로 노출한다(데모 = 코드 일치 보장). 서버 실행이 필요한 AI/배포 프로젝트는 `projectCode.js`의 완성 코드 + 예상 결과(JSON/터미널) 미리보기로 대체하며, 브라우저 크롬 프레임 UI로 "구현된 상태"를 시각화한다(`ProjectDetail.jsx`). 18개 프로젝트 중 6개가 진짜 인터랙티브 데모다.

### 3.5 자료실
`Resources.jsx`는 트랙별로 27개 챕터를 펼쳐 본문·실습·소요시간을 한눈에 모아 보여주는 인덱스 역할을 한다. 외부 PDF 다운로드 링크는 없고, 모든 자료를 웹 본문으로 직접 제공하는 방식이다.

### 3.6 OG / SEO
`index.html`에 Open Graph·Twitter 카드, 한국어 description(27/137 수치 반영), `og:locale=ko_KR`, 1200×630 OG 이미지(`og-image-v2.png`)가 정적으로 구비되어 공유 시 카드가 정상 노출된다. 다만 SPA 특성상 라우트별 동적 메타 태그는 없어 모든 페이지가 동일한 OG를 공유한다.

## 4. UX·디자인 평가

### 4.1 디자인 시스템
`theme.js`의 토큰 기반으로 트랙별 시그니처 컬러(web 보라 #7C3AED, react 블루 #1A45D8, ai 오렌지 #F15A0C, ops 그린 #0E9F6E)를 hero 배경·진도 바·배지·용어 카드 전반에 일관 적용한다. 본문은 라이트 테마, 코드 블록은 GitHub Dark 기반 다크 테마로 대비를 준다. 디자인 일관성이 높고 트랙 식별성이 강하다.

### 4.2 반응형
`global.css`는 단 3개의 핵심 브레이크포인트로 반응형을 처리한다: 1340px에서 데스크톱 네비→햄버거 전환, 880px에서 강의 TOC 사이드바를 상단 박스로 접기 및 3열 그리드→2열, 560px에서 1열. 본문 전반은 `clamp()` 유체 타이포·여백을 광범위하게 사용해 별도 미디어쿼리 없이도 부드럽게 축소된다.

### 4.3 네비게이션
플로팅 필(pill) 헤더가 스크롤 시 블러·그림자를 강화하고, 활성 라우트에 밑줄 인디케이터를 표시한다. 강의 페이지의 좌측 TOC는 진도 체크박스와 결합되어 목차이자 진도판 역할을 겸한다. 이전/다음 챕터 카드와 트랙 완주 CTA로 연속 학습을 유도한다.

### 4.4 코드 가독성
`CodeBlock.jsx`는 PrismLight에 12개 언어(jsx/tsx/py/bash/sql/docker/yaml 등)를 등록하고, **주석을 녹색(#3FB950)으로 강조**하는 커스텀 GitHub Dark 테마를 적용해 한국어 라인 주석이 본문 코드와 시각적으로 분리된다. 언어 라벨 + 복사 버튼(클립보드 실패 시 execCommand 폴백, "복사됨" 피드백, `aria-label`)을 갖춰 교육용 코드 열람 경험이 잘 다듬어져 있다.

## 5. 기술 평가

### 5.1 React/Vite 구조
페이지(13) / 컴포넌트(7) / 훅(3) / lib(2) / data(5) / demos로 관심사가 깔끔히 분리된 표준적 구조다. 인증·진도는 Context Provider로 전역화하고, 라우팅은 `App.jsx`에 평면적으로 선언했다. 외부 상태관리·UI 라이브러리 없이 React 기본기로 구현해 의존성이 가볍다(런타임 deps 7개).

### 5.2 콘텐츠 lazy 로딩
`content/index.js`가 `import.meta.glob('./*.md', { query: '?raw' })`로 챕터 본문을 **챕터별 청크로 지연 로드**한다. `Lesson.jsx`는 진입 시 해당 챕터 .md만 비동기 로드하고 해시 진입 시 섹션 스크롤까지 처리해, 27개 챕터 본문(약 28k줄)이 초기 번들에 포함되지 않는다.

### 5.3 마크다운 렌더링
`Markdown.jsx`(react-markdown + remark-gfm)가 본문을 렌더하고, 코드 펜스를 `CodeBlock`으로 위임하며 섹션 제목에 앵커(`sectionAnchor`)를 부여해 TOC 딥링크를 지원한다.

### 5.4 CI/CD
GitHub Actions 단일 워크플로로 빌드·배포가 자동화되어 있고(2.4·1.4 참조), node 20 + npm 캐시, 권한 최소화(contents:read, pages:write, id-token:write), 환경별 Secrets 주입까지 모범적으로 구성됐다.

### 5.5 커스텀 도메인
`public/CNAME` + GitHub Pages로 bootcamp.dreamitbiz.com을 서비스. Vite `base` 관련 자산 404 이슈(과거 반복 사례)는 커스텀 도메인 환경에 맞게 정상 처리된 것으로 보인다(별도 base 미지정, 루트 서빙).

### 5.6 잠재적 기술 리스크
- 코드 분할 미적용: `App.jsx`가 모든 페이지를 정적 import한다. 본문 .md는 lazy지만 페이지 컴포넌트와 `react-syntax-highlighter`는 초기 번들에 포함되어 메인 청크가 커질 수 있다(`React.lazy` 미사용).
- 데이터 규모: `review.js`(2,845줄, 용어 216 + 퀴즈 135)와 `curriculum.js`(1,967줄)가 모두 동기 import되어 초기 JS에 실린다.

## 6. 강점 (Top 6)

1. **콘텐츠 커버리지의 완결성** — 27개 챕터 전부가 본문 마크다운 + 135개 퀴즈를 갖춰, 데이터만 있고 본문이 빈 미완성 구간이 없다.
2. **실행 가능한 실무형 코드와 한국어 주석** — AI 프로젝트(FastAPI+HF)·React 데모가 라인 단위 한국어 주석을 단 동작 가능한 코드이고, 녹색 주석 강조로 가독성을 극대화했다.
3. **데모-코드 일치 보장** — `?raw` import로 라이브 데모와 노출 소스가 동일 파일이라 "보이는 코드 ≠ 실제 코드" 괴리가 구조적으로 없다.
4. **안전한 진도 동기화** — 섹션 단위 RLS + 낙관적 업데이트/롤백으로 사용자 데이터 격리와 반응성을 함께 확보했다.
5. **일관된 트랙 기반 디자인 시스템** — 4색 트랙 컬러를 hero·진도·배지·용어에 일관 적용하고 다크 코드 테마로 대비를 줘 학습 동선 식별성이 높다.
6. **가벼운 의존성과 모범적 CI/CD** — 외부 UI 라이브러리 없이 구현, 단일 GitHub Actions로 빌드/404 폴백/Secrets 주입까지 자동화.

## 7. 개선 권장사항 (우선순위별)

### 7.1 높음
- **퀴즈 결과 서버 저장**: 현재 퀴즈 최고점은 `localStorage`에만 저장되어 기기 변경 시 소실되고 진도와 연동되지 않는다. `webpro_progress`와 유사한 quiz_results 테이블(RLS)로 이관 권장.
- **표기 수치 불일치 정정**: README("16개 챕터·62개 강의", 2개 트랙)와 `Resources.jsx` 통계 카드의 하드코딩 "16개"가 실제 데이터(27/137/4)와 다르다. `CHAPTERS.length` 등 동적 값으로 통일 필요(데이터에는 이미 `TOTAL_CHAPTERS`/`TOTAL_LESSONS` 존재).
- **용어집 챕터 링크 라벨 버그**: `Quiz.jsx`의 글로서리 링크가 트랙을 `ai`면 'AI', 그 외 전부 'React'로 표기해 web·ops 용어가 'React'로 잘못 라벨된다. `ProjectDetail.jsx`의 `chBadge`처럼 4트랙을 모두 분기하도록 수정.

### 7.2 중간
- **모바일 메뉴 9개 항목 정리**: 햄버거 메뉴에 9개 링크가 평면 나열된다. 트랙 4종을 그룹/아코디언으로 묶어 인지 부담을 줄일 여지.
- **코드 분할(React.lazy)**: 페이지·`react-syntax-highlighter`·대형 data 파일을 라우트 단위로 lazy 로드해 초기 번들·LCP 개선.
- **접근성 보강**: 다수 인터랙티브 요소가 인라인 스타일 버튼이라 포커스 표시·키보드 탐색·명시적 ARIA 라벨이 부족할 수 있다. `:focus-visible` 스타일과 랜드마크/헤딩 검증 권장.
- **랩 커버리지 확장**: 16개 챕터에만 있는 실습 랩을 web/ops 전 챕터로 확대.

### 7.3 낮음
- **라우트별 동적 메타/OG**: 트랙·강의별 title/description을 동적 주입하면 검색·공유 품질이 향상.
- **자동화 테스트 부재**: 단위/통합/E2E 테스트가 없다. 진도 토글·퀴즈 채점 같은 핵심 로직에 최소한의 테스트 도입.
- **번들 크기 모니터링**: data 파일 증가에 대비해 빌드 사이즈 가시화(예: rollup-plugin-visualizer).
