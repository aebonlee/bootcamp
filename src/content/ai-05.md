# 5장. 오디오 인공지능 웹 프로젝트

이 장에서는 오디오 데이터를 다루는 생성형 AI 기술을 FastAPI와 결합하여 음성 인식(ASR), 음성 합성(TTS), 음성 챗봇 등의 기능을 직접 구현합니다. Whisper, Bark, TTS 모델들을 사용하여 음성 → 텍스트, 텍스트 → 음성의 흐름을 이해하고, 업로드된 오디오 파일을 실시간으로 처리하는 RESTful API를 함께 구축합니다.

## 이 장의 학습 목표

- Whisper 모델로 음성 파일을 텍스트로 변환하는 ASR(자동 음성 인식) API를 구현합니다.
- Bark/TTS 모델로 텍스트를 음성으로 합성하는 API와 다양한 출력 형식을 제공합니다.
- ASR + 챗봇 + TTS를 파이프라인으로 연결한 완전한 음성 챗봇 서비스를 구축합니다.
- UploadFile, BackgroundTasks, StreamingResponse 등 FastAPI 고급 기능을 마스터합니다.

## 이 장의 개요

음성(오디오)은 텍스트, 이미지와 함께 AI가 처리하는 가장 핵심적인 데이터 유형 중 하나입니다. 스마트 스피커, 음성 비서, 자동 자막, 오디오북 등 실생활 속 AI 음성 기술은 이미 수십억 명의 일상에 자리 잡았습니다. 이러한 기술의 근간에는 ASR(Automatic Speech Recognition, 자동 음성 인식)과 TTS(Text-To-Speech, 음성 합성)가 있습니다.

Hugging Face는 OpenAI의 Whisper, Suno의 Bark, Microsoft의 SpeechT5 등 최고 수준의 오디오 AI 모델을 무료로 제공합니다. 이 모델들을 FastAPI와 결합하면, 기업 수준의 음성 처리 서비스를 빠르게 구축할 수 있습니다.

오디오 기반 AI 기술이 웹 서비스에 어떻게 접목될 수 있는지, 사용자 음성을 입력받아 챗봇처럼 응답하거나 자막을 자동 생성하는 방식 등을 단계별로 실습합니다.

## 5.1 Whisper로 음성 인식 API 만들기

OpenAI가 2022년 공개한 Whisper는 현재 세계 최고 수준의 다국어 음성 인식 모델입니다. 영어, 한국어를 포함한 99개 언어를 지원하며, Hugging Face Hub에도 공개되어 있어 Pipeline API 또는 `openai-whisper` 패키지로 바로 사용할 수 있습니다.

### 5.1.1 Hugging Face openai/whisper 모델 소개

Whisper는 680,000시간의 다국어 오디오 데이터로 학습된 Transformer 기반의 Encoder-Decoder 모델입니다. 음성 신호를 먼저 80채널 Log-Mel Spectrogram으로 변환한 후 인코더에 입력하고, 디코더가 자동 회귀 방식으로 텍스트를 생성합니다.

| 모델 크기 | 파라미터 | 디스크 크기 | 특징 |
| --- | --- | --- | --- |
| tiny | 39M | ~75MB | 가장 빠름. 실험/체험용. |
| base | 74M | ~145MB | 속도와 정확도 균형. 학습/테스트 추천. |
| small | 244M | ~480MB | 일반 사용에 충분한 품질. |
| medium | 769M | ~1.5GB | 고품질. GPU 권장. |
| large-v3 | 1550M | ~3.1GB | 최고 정확도. A100 GPU 권장. |

> **팁: 작은 버전으로도 충분히 체험 가능**
>
> CPU 환경에서는 tiny 또는 base 모델을 권장합니다.
>
> - tiny: 약 10초 길이 오디오를 CPU에서 15~30초 내 처리 가능
> - base: CPU에서 30~60초 처리 시간. 한국어 인식도 양호.
>
> GPU(T4) 환경에서는 medium 이상을 사용하면 실시간(1초 오디오 < 1초 처리) 가능.

Hugging Face Inference API를 사용하면 로컬 GPU 없이도 large-v3 이용 가능:

```python
# Hugging Face가 제공하는 원격 추론(Inference) 클라이언트 임포트
from huggingface_hub import InferenceClient

# 발급받은 HF 액세스 토큰으로 클라이언트 생성 (로컬 GPU 불필요, HF 서버가 대신 추론)
client = InferenceClient(token='hf_...')
# 오디오 파일 경로를 넘기면 HF 서버의 Whisper가 음성을 텍스트로 변환해 결과를 반환
result = client.automatic_speech_recognition('audio.wav')
```

### 5.1.2 Whisper의 장점: 언어 감지 + 정확한 자막 생성

Whisper가 다른 ASR 모델 대비 특별히 강점을 갖는 영역은 다음과 같습니다.

| Whisper의 강점 | 구체적인 내용 |
| --- | --- |
| 자동 언어 감지 | 입력 오디오 첫 30초를 분석하여 99개 언어 중 자동 감지. 별도 언어 지정 불필요. |
| 타임스탬프 지원 | 단어/문장 단위 타임스탬프 생성 가능. 자막(SRT/VTT) 자동 생성에 활용. |
| 잡음 환경 강인성 | 배경 음악, 잡음이 있는 환경에서도 높은 인식률. 6dB 이상 SNR에서 90%+ 정확도. |
| 다국어 번역 | 음성을 인식하면서 동시에 영어로 번역 가능 (translate 태스크). |
| 긴 오디오 처리 | 30초 단위로 청크(Chunk) 분할 후 연속 처리. 수 시간 분량 오디오도 처리 가능. |
| 구어체 강인성 | 말끝 흐림, 겹쳐 말하기, 방언 등 자연스러운 대화 인식에 강함. |

Whisper가 내부적으로 오디오를 처리하는 과정을 단계별로 살펴봅니다.

1. **오디오 전처리**: 입력 오디오를 16kHz 모노(Mono) WAV로 변환
2. **Log-Mel Spectrogram 변환**: 80채널, 30초 단위 스펙트로그램 생성 (3,000 프레임 × 80채널)
3. **인코더(Encoder)**: Transformer 인코더로 오디오 특징(Feature) 추출
4. **언어 감지**: 디코더 첫 토큰 예측으로 언어 자동 판별
5. **디코더(Decoder)**: 자기 회귀 방식으로 텍스트 토큰 순서대로 예측
6. **타임스탬프 정렬**: 각 단어의 시작/종료 시간 계산 (DTW 알고리즘)

### 5.1.3 오디오 파일(mp3, wav 등) → 텍스트 변환 FastAPI API 만들기

Whisper ASR 서비스 로직을 먼저 구현하겠습니다. 서비스 레이어에서 Hugging Face Pipeline 방식과 `openai-whisper` 패키지 방식을 모두 지원하도록 설계합니다.

**services/asr_service.py**

```python
# services/asr_service.py
import os                       # 임시 파일 삭제(os.unlink) 등 OS 기능
import time                     # 처리 시간 측정용
import torch                    # PyTorch: GPU 사용 여부 판단, dtype 지정
import tempfile                 # 바이트 입력을 임시 파일로 저장할 때 사용
import soundfile as sf          # WAV 파일을 읽어 길이(duration) 계산
import numpy as np
from pathlib import Path
from typing import Optional, Dict, Any

# Hugging Face Pipeline 방식 (권장) - 모델/프로세서를 묶어 추론을 단순화
from transformers import pipeline, AutoModelForSpeechSeq2Seq, AutoProcessor


class ASRService:
    """Whisper 기반 자동 음성 인식(ASR) 서비스"""

    # 사용자가 고르는 크기 키워드 → 실제 Hugging Face 모델 ID 매핑
    SUPPORTED_MODELS = {
        'tiny':   'openai/whisper-tiny',
        'base':   'openai/whisper-base',
        'small':  'openai/whisper-small',
        'medium': 'openai/whisper-medium',
        'large':  'openai/whisper-large-v3',
    }

    def __init__(self, model_size: str = 'base'):
        self.model_size = model_size          # 사용할 모델 크기(기본 base)
        self.pipe = None                      # 로드 전엔 None, load_model()에서 채워짐
        # GPU(cuda)가 있으면 GPU, 없으면 CPU 사용
        self.device = 'cuda:0' if torch.cuda.is_available() else 'cpu'
        # GPU에서는 float16(절반 정밀도)로 메모리/속도 이득, CPU는 float32
        self.torch_dtype = (
            torch.float16 if torch.cuda.is_available() else torch.float32
        )

    def load_model(self):
        """Whisper 파이프라인 로드"""
        # 크기 키워드를 실제 모델 ID로 변환 (없으면 base로 폴백)
        model_id = self.SUPPORTED_MODELS.get(
            self.model_size, 'openai/whisper-base'
        )

        print(f'[ASR] Loading Whisper model: {model_id}')
        print(f'[ASR] Device: {self.device}, dtype: {self.torch_dtype}')

        # 음성→텍스트(Seq2Seq) 모델 가중치 다운로드/로드
        model = AutoModelForSpeechSeq2Seq.from_pretrained(
            model_id,
            torch_dtype=self.torch_dtype,
            low_cpu_mem_usage=True,           # 로딩 중 CPU 메모리 사용 최소화
            use_safetensors=True,             # 안전한 safetensors 포맷 우선 사용
        )
        model.to(self.device)                 # 모델을 선택한 장치(GPU/CPU)로 이동

        # 전처리/토크나이저가 묶인 프로세서 로드
        processor = AutoProcessor.from_pretrained(model_id)

        # 추론 파이프라인 구성: 오디오를 넣으면 텍스트가 나오도록 묶음
        self.pipe = pipeline(
            task='automatic-speech-recognition',  # ASR(음성 인식) 작업
            model=model,
            tokenizer=processor.tokenizer,         # 텍스트 토큰화 담당
            feature_extractor=processor.feature_extractor,  # 오디오→스펙트로그램 변환
            torch_dtype=self.torch_dtype,
            device=self.device,
        )
        print(f'[ASR] Whisper {self.model_size} loaded successfully.')

    def transcribe(
        self,
        audio_path: str,
        language: Optional[str] = None,
        task: str = 'transcribe',
        return_timestamps: bool = False,
        chunk_length_s: int = 30,
    ) -> Dict[str, Any]:
        """
        오디오 파일을 텍스트로 변환합니다.

        Args:
            audio_path: 오디오 파일 경로 (wav, mp3, m4a 등)
            language: 언어 코드 (None이면 자동 감지, 'ko'=한국어, 'en'=영어)
            task: 'transcribe'(인식) 또는 'translate'(영어로 번역)
            return_timestamps: True면 단어 단위 타임스탬프 포함
            chunk_length_s: 청크 분할 길이 (초). 긴 오디오 처리 시 사용.
        """
        # 모델이 아직 로드되지 않았으면 즉시 에러 (안전장치)
        if self.pipe is None:
            raise RuntimeError('ASR 모델이 로드되지 않았습니다.')

        start_time = time.time()              # 처리 시간 측정 시작

        # generate_kwargs 구성: 작업 종류(인식/번역)와 언어를 모델 생성 옵션으로 전달
        generate_kwargs = {'task': task}
        if language:                          # 언어가 지정되면 자동 감지 대신 강제 지정
            generate_kwargs['language'] = language

        # 긴 오디오를 위한 청크 파이프라인 설정 (30초 단위로 잘라 순차 처리)
        result = self.pipe(
            audio_path,
            generate_kwargs=generate_kwargs,
            return_timestamps=return_timestamps,  # 자막용 타임스탬프 포함 여부
            chunk_length_s=chunk_length_s,        # 각 청크 길이(초)
            stride_length_s=chunk_length_s // 6,  # 청크 간 오버랩(경계 단어 손실 방지)
        )

        elapsed = round(time.time() - start_time, 2)  # 실제 추론 소요 시간(초)

        # 오디오 파일 길이 측정 (샘플 수 ÷ 샘플레이트 = 초)
        try:
            audio_data, sample_rate = sf.read(audio_path)
            audio_duration = round(len(audio_data) / sample_rate, 2)
        except Exception:
            audio_duration = None             # 읽기 실패 시 길이 미상 처리

        # 인식 결과와 메타데이터를 딕셔너리로 반환
        return {
            'text': result['text'].strip(),                # 최종 인식 텍스트
            'chunks': result.get('chunks', []),            # 타임스탬프 청크 목록
            'language': language or 'auto-detected',       # 사용 언어(또는 자동 감지)
            'task': task,
            'audio_duration_sec': audio_duration,
            'processing_time_sec': elapsed,
            'model': f'openai/whisper-{self.model_size}',
            # realtime_factor < 1.0 이면 오디오 길이보다 빠르게 처리(실시간 가능)
            'realtime_factor': (
                round(elapsed / audio_duration, 2)
                if audio_duration else None
            ),
        }

    def transcribe_bytes(
        self,
        audio_bytes: bytes,
        suffix: str = '.wav',
        **kwargs
    ) -> Dict[str, Any]:
        """바이트 데이터로 직접 변환 (임시 파일 사용)"""
        # Whisper 파이프라인은 파일 경로를 받으므로, 메모리 바이트를 임시 파일로 저장
        with tempfile.NamedTemporaryFile(suffix=suffix, delete=False) as f:
            f.write(audio_bytes)
            tmp_path = f.name
        try:
            return self.transcribe(tmp_path, **kwargs)  # 저장한 경로로 일반 변환 수행
        finally:
            os.unlink(tmp_path)               # 변환 후 임시 파일 반드시 삭제

    def generate_srt(
        self,
        audio_path: str,
        language: Optional[str] = None,
    ) -> str:
        """SRT 형식 자막 파일 생성"""
        # 타임스탬프를 포함해 전사 → 자막 생성에 필요한 시간 정보 확보
        result = self.transcribe(
            audio_path,
            language=language,
            return_timestamps=True,
        )
        chunks = result.get('chunks', [])
        # 타임스탬프 청크가 없으면 전체 텍스트를 하나의 자막 블록으로 반환
        if not chunks:
            return f'1\n00:00:00,000 --> 00:10:00,000\n{result["text"]}\n'

        srt_lines = []
        # 각 청크를 SRT 한 블록(번호/시간범위/텍스트/빈줄)으로 변환
        for i, chunk in enumerate(chunks, 1):
            ts = chunk.get('timestamp', (0, 1))           # (시작초, 종료초)
            start = self._sec_to_srt_time(ts[0] or 0)
            # 종료 시간이 없으면 시작+2초로 보정
            end = self._sec_to_srt_time(ts[1] or (ts[0] or 0) + 2)
            srt_lines.append(f'{i}')                      # 자막 순번
            srt_lines.append(f'{start} --> {end}')        # 표시 구간
            srt_lines.append(chunk['text'].strip())       # 자막 내용
            srt_lines.append('')                          # 블록 구분 빈 줄
        return '\n'.join(srt_lines)

    @staticmethod
    def _sec_to_srt_time(seconds: float) -> str:
        """초 → SRT 타임코드 변환 (hh:mm:ss,mmm)"""
        ms = int((seconds % 1) * 1000)        # 소수부 → 밀리초
        s = int(seconds) % 60                 # 초
        m = (int(seconds) // 60) % 60         # 분
        h = int(seconds) // 3600              # 시
        # 0으로 패딩하여 SRT 표준 형식(hh:mm:ss,mmm)으로 조립
        return f'{h:02d}:{m:02d}:{s:02d},{ms:03d}'


# 싱글턴 인스턴스: 앱 전체에서 모델을 한 번만 로드해 공유 (base 크기로 시작)
asr_service = ASRService(model_size='base')
```

**services/audio_utils.py – 오디오 전처리 유틸리티**

```python
# services/audio_utils.py
import os
import uuid                     # 충돌 없는 고유 파일명 생성
import numpy as np
from pathlib import Path
from typing import Tuple, Optional

# pydub는 ffmpeg 래퍼로 포맷 변환에 강력하지만 없을 수도 있으므로 try로 감지
try:
    from pydub import AudioSegment
    PYDUB_AVAILABLE = True       # pydub 사용 가능
except ImportError:
    PYDUB_AVAILABLE = False      # 없으면 scipy 폴백 경로 사용

import soundfile as sf

TEMP_DIR = Path('temp')          # 업로드/변환 중간 파일 보관 폴더
OUTPUT_DIR = Path('output')      # 생성 결과물(합성 음성 등) 폴더
TEMP_DIR.mkdir(exist_ok=True)    # 폴더 없으면 생성(이미 있으면 통과)
OUTPUT_DIR.mkdir(exist_ok=True)

# 업로드 허용 확장자 집합 / 최대 허용 용량
SUPPORTED_AUDIO_FORMATS = {'.wav', '.mp3', '.m4a', '.ogg', '.flac', '.webm', '.mp4'}
MAX_FILE_SIZE_MB = 50


def validate_audio_file(filename: str, size_bytes: int) -> None:
    """오디오 파일 유효성 검사"""
    ext = Path(filename).suffix.lower()       # 확장자 추출(소문자화)
    # 1) 지원하지 않는 확장자면 거부
    if ext not in SUPPORTED_AUDIO_FORMATS:
        raise ValueError(
            f'지원하지 않는 파일 형식: {ext}. '
            f'지원 형식: {SUPPORTED_AUDIO_FORMATS}'
        )
    # 2) 최대 용량 초과 시 거부 (MB → 바이트로 환산해 비교)
    if size_bytes > MAX_FILE_SIZE_MB * 1024 * 1024:
        raise ValueError(
            f'파일 크기 초과: {size_bytes / 1024 / 1024:.1f}MB. '
            f'최대 {MAX_FILE_SIZE_MB}MB까지 허용.'
        )


def convert_to_wav(
    input_path: str,
    output_path: Optional[str] = None,
    sample_rate: int = 16000,             # Whisper가 요구하는 16kHz
    channels: int = 1,                    # 모노(1채널)
) -> str:
    """
    오디오 파일을 Whisper 입력 형식(16kHz, 모노 WAV)으로 변환합니다.
    """
    # 출력 경로 미지정 시 temp/에 고유 이름으로 생성
    if output_path is None:
        output_path = str(TEMP_DIR / f'{uuid.uuid4()}.wav')

    if PYDUB_AVAILABLE:
        # pydub 경로: ffmpeg로 어떤 포맷이든 읽어 샘플레이트/채널 조정 후 WAV 저장
        audio = AudioSegment.from_file(input_path)
        audio = audio.set_frame_rate(sample_rate)   # 16kHz로 리샘플
        audio = audio.set_channels(channels)        # 모노로 변환
        audio.export(output_path, format='wav')
    else:
        # 폴백: soundfile로 읽고 scipy로 리샘플링
        import scipy.signal as signal
        data, orig_sr = sf.read(input_path)
        if orig_sr != sample_rate:
            # 목표 샘플레이트에 맞춰 샘플 개수 재계산 후 리샘플
            num_samples = int(len(data) * sample_rate / orig_sr)
            data = signal.resample(data, num_samples)
        if data.ndim > 1:
            data = data.mean(axis=1)  # 스테레오 → 모노 (채널 평균)
        sf.write(output_path, data, sample_rate)
    return output_path                            # 변환된 WAV 경로 반환


def get_audio_info(audio_path: str) -> dict:
    """오디오 파일 기본 정보 반환"""
    try:
        data, sr = sf.read(audio_path)            # 샘플 배열과 샘플레이트 읽기
        duration = len(data) / sr                 # 길이(초) = 샘플 수 ÷ 샘플레이트
        channels = 1 if data.ndim == 1 else data.shape[1]  # 1차원=모노
        return {
            'duration_sec': round(duration, 2),
            'sample_rate': sr,
            'channels': channels,
            'samples': len(data),
            'format': Path(audio_path).suffix.lower(),
        }
    except Exception as e:
        return {'error': str(e)}                  # 읽기 실패 시 오류 메시지 반환


def save_upload_file(content: bytes, filename: str) -> str:
    """업로드 파일을 임시 디렉터리에 저장"""
    ext = Path(filename).suffix.lower()
    unique_name = f'{uuid.uuid4()}{ext}'          # 원본명 충돌 방지용 UUID 파일명
    save_path = str(TEMP_DIR / unique_name)
    with open(save_path, 'wb') as f:              # 바이너리 쓰기 모드로 저장
        f.write(content)
    return save_path                              # 저장 경로 반환


def cleanup_temp_files(max_age_sec: int = 3600):
    """1시간 이상 된 임시 파일 정리"""
    import time
    now = time.time()
    for f in TEMP_DIR.iterdir():                  # temp/ 내부 항목 순회
        # 파일이면서 마지막 수정 후 경과 시간이 기준 초과면 삭제
        if f.is_file() and now - f.stat().st_mtime > max_age_sec:
            f.unlink(missing_ok=True)             # 없어도 에러 안 나게 삭제
```

### 5.1.5 비동기 업로드 파일 처리 (UploadFile 활용)

FastAPI의 `UploadFile`은 업로드된 파일을 비동기로 처리하는 핵심 클래스입니다. 파일 메타데이터(이름, 크기, MIME 타입)에 접근하고, `async`/`await`로 파일 내용을 읽을 수 있습니다.

| UploadFile 속성/메서드 | 설명 |
| --- | --- |
| `filename` | 업로드된 파일의 원본 파일명 (문자열) |
| `content_type` | MIME 타입 (예: 'audio/wav', 'audio/mpeg') |
| `size` | 파일 크기(바이트). None일 수 있으므로 직접 읽어서 측정 권장. |
| `await file.read()` | 파일 전체 내용을 bytes로 비동기 읽기 |
| `await file.read(size)` | 지정 바이트 수만큼 읽기 (스트리밍 처리용) |
| `await file.seek(0)` | 파일 포인터를 처음으로 이동 (재읽기 시 필요) |
| `await file.close()` | 파일 스트림 종료. FastAPI가 자동 처리하지만 명시 권장. |
| `file.file` | 실제 SpooledTemporaryFile 객체. 저수준 파일 작업 시 활용. |

**routers/asr.py – 음성 인식 라우터**

```python
# routers/asr.py
import os
from pathlib import Path
# FastAPI 파일 업로드/폼/예외/백그라운드 작업 관련 도구들
from fastapi import (
    APIRouter, UploadFile, File, Form,
    HTTPException, status, BackgroundTasks,
)
from fastapi.responses import PlainTextResponse   # SRT를 평문으로 응답
from typing import Optional
from models.schemas import ASRResponse, SRTResponse  # 응답 스키마
from services.asr_service import asr_service          # 싱글턴 ASR 서비스
from services.audio_utils import (
    validate_audio_file, save_upload_file,
    convert_to_wav, cleanup_temp_files, get_audio_info,
)

# 이 라우터의 모든 경로는 /asr 로 시작, Swagger 문서엔 ASR 그룹으로 묶임
router = APIRouter(prefix='/asr', tags=['Speech Recognition (ASR)'])


@router.post(
    '/transcribe',                            # 음성→텍스트 핵심 엔드포인트
    response_model=ASRResponse,               # 응답 형태 자동 검증/문서화
    summary='오디오 파일을 텍스트로 변환 (STT)',
    description='''
    WAV, MP3, M4A, OGG 등 오디오 파일을 업로드하면
    Whisper 모델이 텍스트로 변환합니다.
    language를 비워두면 자동으로 언어를 감지합니다.
    ''',
)
async def transcribe_audio(
    background_tasks: BackgroundTasks,        # 응답 후 임시 파일 정리에 사용
    file: UploadFile = File(                  # 업로드된 오디오 파일(필수)
        ...,
        description='오디오 파일 (wav/mp3/m4a/ogg/flac, 최대 50MB)',
    ),
    language: Optional[str] = Form(           # 언어 코드(폼 필드, 비우면 자동 감지)
        None,
        description='언어 코드 (ko, en, ja 등). 비우면 자동 감지.',
    ),
    task: str = Form(                         # 인식 또는 영어 번역
        'transcribe',
        description='transcribe(인식) 또는 translate(영어로 번역)',
    ),
    return_timestamps: bool = Form(           # 자막용 타임스탬프 포함 여부
        False,
        description='True면 단어 단위 타임스탬프 포함',
    ),
):
    # 1. 파일 유효성 검사 - 전체 바이트를 읽어 확장자/용량 확인
    content = await file.read()               # UploadFile은 비동기로 읽어야 함
    try:
        validate_audio_file(file.filename, len(content))
    except ValueError as e:
        # 형식/용량 위반은 422(처리 불가)로 응답
        raise HTTPException(
            status_code=status.HTTP_422_UNPROCESSABLE_ENTITY,
            detail=str(e)
        )

    # 2. 임시 파일 저장 - 읽은 바이트를 temp/에 파일로 기록
    tmp_path = save_upload_file(content, file.filename)

    # 3. WAV 변환 (필요 시) - Whisper 입력 포맷(16kHz 모노 WAV)으로 맞춤
    ext = Path(file.filename).suffix.lower()
    if ext != '.wav':
        wav_path = tmp_path.replace(ext, '.wav')
        convert_to_wav(tmp_path, wav_path)
        process_path = wav_path               # 변환본을 처리 대상으로
    else:
        process_path = tmp_path               # 이미 WAV면 그대로 사용

    # 4. 오디오 정보 수집 (길이/샘플레이트 등)
    audio_info = get_audio_info(process_path)

    # 5. ASR 추론 - Whisper로 실제 음성 인식 수행
    try:
        result = asr_service.transcribe(
            audio_path=process_path,
            language=language,
            task=task,
            return_timestamps=return_timestamps,
        )
    except Exception as e:
        # 추론 중 예외는 500(서버 오류)로 변환
        raise HTTPException(
            status_code=status.HTTP_500_INTERNAL_SERVER_ERROR,
            detail=f'음성 인식 처리 중 오류: {str(e)}'
        )
    finally:
        # 6. 임시 파일 백그라운드 정리 - 응답을 먼저 보내고 나중에 삭제
        background_tasks.add_task(os.unlink, tmp_path)
        if ext != '.wav' and os.path.exists(process_path):
            background_tasks.add_task(os.unlink, process_path)

    # 인식 결과와 메타데이터를 응답 스키마에 담아 반환
    return ASRResponse(
        filename=file.filename,
        transcribed_text=result['text'],
        language=result['language'],
        task=task,
        audio_duration_sec=audio_info.get('duration_sec'),
        processing_time_sec=result['processing_time_sec'],
        realtime_factor=result.get('realtime_factor'),
        model_used=result['model'],
        # 타임스탬프 요청한 경우에만 청크 포함
        chunks=result.get('chunks', []) if return_timestamps else [],
    )


@router.post(
    '/generate-srt',                          # 음성 → SRT 자막 파일
    response_class=PlainTextResponse,         # 텍스트(자막)로 응답
    summary='SRT 자막 파일 생성',
    description='오디오 파일로부터 SRT 형식 자막을 생성합니다.',
)
async def generate_srt(
    background_tasks: BackgroundTasks,
    file: UploadFile = File(...),
    language: Optional[str] = Form(None),
):
    content = await file.read()
    try:
        validate_audio_file(file.filename, len(content))   # 형식/용량 검사
    except ValueError as e:
        raise HTTPException(status_code=422, detail=str(e))

    tmp_path = save_upload_file(content, file.filename)
    ext = Path(file.filename).suffix.lower()

    # WAV가 아니면 변환 후 처리
    if ext != '.wav':
        wav_path = tmp_path.replace(ext, '.wav')
        convert_to_wav(tmp_path, wav_path)
        process_path = wav_path
    else:
        process_path = tmp_path

    try:
        # 타임스탬프 기반으로 SRT 문자열 생성
        srt_content = asr_service.generate_srt(process_path, language)
    except Exception as e:
        raise HTTPException(status_code=500, detail=str(e))
    finally:
        # 임시 파일 비동기 정리
        background_tasks.add_task(os.unlink, tmp_path)
        if ext != '.wav':
            background_tasks.add_task(os.unlink, process_path)

    # 브라우저가 .srt 파일로 다운로드하도록 헤더 지정
    return PlainTextResponse(
        content=srt_content,
        media_type='text/plain; charset=utf-8',
        headers={'Content-Disposition':
                 f'attachment; filename="{Path(file.filename).stem}.srt"'},
    )
```

## 5.2 텍스트 → 음성 API (TTS)

TTS(Text-To-Speech, 음성 합성)는 텍스트를 자연스러운 음성 오디오로 변환하는 기술입니다. 이 절에서는 Suno Bark와 Microsoft SpeechT5, 두 가지 접근 방식으로 TTS API를 구현합니다. 두 모델은 성능, 속도, 메모리 요구사항이 서로 다르므로, 운영 환경에 맞게 선택할 수 있도록 모두 구현합니다.

### 5.2.1 Hugging Face suno/bark 또는 SpeechT5 모델 활용

Bark는 Suno AI가 2023년 공개한 오픈소스 TTS 모델로, 다양한 음성 캐릭터, 감정 표현, 비언어 사운드(웃음, 울음, 노래 등)를 생성할 수 있는 혁신적인 모델입니다. SpeechT5는 Microsoft의 T5 기반 다목적 음성 처리 모델로, 더 빠른 처리 속도와 안정성을 제공합니다.

| 특성 | suno/bark | microsoft/speecht5_tts |
| --- | --- | --- |
| 모델 크기 | ~1.7GB (3개 서브모델) | ~400MB |
| 처리 속도 | 느림 (CPU: 수 분) | 빠름 (CPU: 수 초) |
| 음성 품질 | 매우 자연스러움 | 자연스러움 |
| 감정 표현 | 매우 다양 (특수 태그) | 제한적 |
| 음성 캐릭터 | 100+ 프리셋 | 화자 임베딩 필요 |
| 비언어 사운드 | 웃음, 울음, 노래 가능 | 불가 |
| 다국어 지원 | 영어 최적화 (기타 언어 실험적) | 다국어 화자 임베딩 활용 |
| GPU 요구사항 | 강력히 권장 | 선택 사항 |

### 5.2.2 텍스트 입력 → 음성 합성 → 오디오 스트리밍 응답

**services/tts_service.py**

```python
# services/tts_service.py
import os
import io                       # 메모리 버퍼(BytesIO)로 오디오를 바이트화
import time
import uuid
import torch
import numpy as np
import soundfile as sf          # numpy 음성 배열을 WAV로 쓰기
from pathlib import Path
from typing import Optional, Literal

OUTPUT_DIR = Path('output')     # 합성 결과 저장 폴더
OUTPUT_DIR.mkdir(exist_ok=True)


class BarkTTSService:
    """Suno Bark 기반 고품질 음성 합성 서비스"""

    # 화자 별칭 → Bark 내장 화자 프리셋 코드 매핑
    VOICE_PRESETS = {
        # 영어 남성
        'en_male_1':    'v2/en_speaker_0',
        'en_male_2':    'v2/en_speaker_1',
        'en_male_deep': 'v2/en_speaker_3',
        # 영어 여성
        'en_female_1':  'v2/en_speaker_9',
        'en_female_2':  'v2/en_speaker_7',
        # 한국어
        'ko_male_1':    'v2/ko_speaker_0',
        'ko_female_1':  'v2/ko_speaker_2',
        # 기타
        'ja_male':      'v2/ja_speaker_0',
        'zh_male':      'v2/zh_speaker_0',
    }

    # Bark 감정/스타일 태그 - 텍스트에 끼워 넣어 웃음/한숨 등 비언어 사운드 표현
    EMOTION_TAGS = {
        'neutral':  '',
        'laugh':    '[laughter]',
        'sigh':     '[sighs]',
        'music':    '[music]',
        'applause': '[applause]',
        'hesitate': '[hesitates]',
    }

    def __init__(self):
        self.processor = None                 # 텍스트→토큰 전처리기(로드 전 None)
        self.model = None                     # Bark 모델(로드 전 None)
        self.device = 'cuda' if torch.cuda.is_available() else 'cpu'
        self.sample_rate = 24000  # Bark 기본 샘플링 레이트

    def load_model(self):
        from transformers import AutoProcessor, BarkModel
        model_id = 'suno/bark-small'  # 'suno/bark' (더 큰 모델)
        print(f'[TTS-Bark] Loading model: {model_id}')
        self.processor = AutoProcessor.from_pretrained(model_id)   # 전처리기 로드
        self.model = BarkModel.from_pretrained(
            model_id,
            # GPU면 float16으로 메모리 절약, CPU면 float32
            torch_dtype=torch.float16 if self.device == 'cuda' else torch.float32,
        ).to(self.device)
        self.model.enable_cpu_offload()  # 메모리 절약 (서브모델을 필요할 때만 GPU에 올림)
        print('[TTS-Bark] Model loaded.')

    def synthesize(
        self,
        text: str,
        voice_preset: str = 'en_male_1',
        emotion: str = 'neutral',
        temperature: float = 0.7,             # 높을수록 다양/불안정, 낮을수록 안정
    ) -> np.ndarray:
        """텍스트를 음성 numpy 배열로 변환"""
        if self.model is None:
            raise RuntimeError('Bark 모델이 로드되지 않았습니다.')

        # 감정 태그 추가 - 선택한 감정 태그를 텍스트 앞에 붙임
        emotion_tag = self.EMOTION_TAGS.get(emotion, '')
        input_text = f'{emotion_tag} {text}'.strip() if emotion_tag else text

        # 음성 프리셋 - 별칭을 실제 화자 코드로 변환(없으면 기본 화자)
        preset = self.VOICE_PRESETS.get(voice_preset, 'v2/en_speaker_0')

        # 입력 처리 - 텍스트와 화자 프리셋을 모델 입력 텐서로 변환
        inputs = self.processor(
            input_text,
            voice_preset=preset,
        )
        # 모든 입력 텐서를 모델과 같은 장치(GPU/CPU)로 이동
        inputs = {k: v.to(self.device) for k, v in inputs.items()}

        # 생성 - 기울기 계산 비활성화(추론 전용)로 음성 파형 생성
        with torch.no_grad():
            audio_array = self.model.generate(
                **inputs,
                do_sample=True,               # 확률적 샘플링(다양한 발화)
                temperature=temperature,
            )

        # GPU 텐서를 CPU numpy 1차원 배열로 변환해 반환
        return audio_array.cpu().numpy().squeeze()

    def save_to_wav(self, audio_array: np.ndarray, filepath: str) -> str:
        # numpy 음성 배열을 지정 경로에 WAV로 저장
        sf.write(filepath, audio_array, self.sample_rate)
        return filepath

    def to_bytes(self, audio_array: np.ndarray, fmt: str = 'wav') -> bytes:
        """numpy 배열을 bytes로 변환"""
        buf = io.BytesIO()                    # 디스크 없이 메모리에 기록
        sf.write(buf, audio_array, self.sample_rate, format=fmt.upper())
        return buf.getvalue()                 # 버퍼의 바이트 반환(스트리밍 응답용)


class SpeechT5Service:
    """Microsoft SpeechT5 기반 빠른 음성 합성 서비스"""

    def __init__(self):
        self.processor = None
        self.model = None
        self.vocoder = None                   # 스펙트로그램→실제 파형 변환기
        self.speaker_embeddings = None        # 화자 음색을 결정하는 벡터
        self.device = 'cuda' if torch.cuda.is_available() else 'cpu'
        self.sample_rate = 16000              # SpeechT5 출력 샘플레이트

    def load_model(self):
        from transformers import (
            SpeechT5Processor, SpeechT5ForTextToSpeech,
            SpeechT5HifiGan,
        )
        from datasets import load_dataset

        print('[TTS-SpeechT5] Loading model...')
        # 텍스트 전처리기 로드
        self.processor = SpeechT5Processor.from_pretrained(
            'microsoft/speecht5_tts'
        )
        # 텍스트→멜 스펙트로그램 생성 모델
        self.model = SpeechT5ForTextToSpeech.from_pretrained(
            'microsoft/speecht5_tts'
        ).to(self.device)
        # HiFi-GAN 보코더: 스펙트로그램을 듣기 좋은 파형으로 변환
        self.vocoder = SpeechT5HifiGan.from_pretrained(
            'microsoft/speecht5_hifigan'
        ).to(self.device)

        # 기본 화자 임베딩 로드 (CMU ARCTIC 데이터셋)
        embeddings_dataset = load_dataset(
            'Matthijs/cmu-arctic-xvectors', split='validation'
        )
        # 화자 인덱스 7306 = 영어 여성 화자
        # unsqueeze(0)으로 배치 차원 추가 후 장치로 이동
        self.speaker_embeddings = torch.tensor(
            embeddings_dataset[7306]['xvector']
        ).unsqueeze(0).to(self.device)
        print('[TTS-SpeechT5] Model loaded.')

    def synthesize(self, text: str) -> np.ndarray:
        if self.model is None:
            raise RuntimeError('SpeechT5 모델이 로드되지 않았습니다.')

        # 텍스트를 토큰 텐서로 변환
        inputs = self.processor(
            text=text, return_tensors='pt'
        ).to(self.device)

        with torch.no_grad():
            # 입력 토큰 + 화자 임베딩 + 보코더로 최종 음성 파형 생성
            speech = self.model.generate_speech(
                inputs['input_ids'],
                self.speaker_embeddings,
                vocoder=self.vocoder,
            )
        return speech.cpu().numpy()           # CPU numpy 배열로 반환

    def to_bytes(self, audio_array: np.ndarray) -> bytes:
        buf = io.BytesIO()                    # 메모리 버퍼에 WAV로 기록
        sf.write(buf, audio_array, self.sample_rate, format='WAV')
        return buf.getvalue()


# 싱글턴 - 두 TTS 엔진을 앱 전역에서 공유(모델은 startup 때 로드)
bark_tts = BarkTTSService()
speecht5_tts = SpeechT5Service()
```

### 5.2.3 음성 캐릭터 선택, 샘플링 속도, 감정 톤 조정

Bark 모델은 특수 텍스트 태그를 통해 다양한 음성 스타일과 감정을 제어할 수 있습니다. 이는 Bark의 가장 독특한 특징으로, 단순한 TTS를 넘어 오디오 드라마, 감정 표현 콘텐츠 생성 등에 활용됩니다.

| Bark 특수 태그 | 효과 및 사용 예시 |
| --- | --- |
| `[laughter]` | 웃음 소리 삽입. '그거 정말 웃겨! [laughter] 하하하' |
| `[sighs]` | 한숨 소리 삽입. '[sighs] 이건 너무 어렵군요...' |
| `[music]` | 배경 음악 분위기. '[music] 아름다운 선율이 흐릅니다' |
| `[applause]` | 박수 소리. '수고하셨습니다! [applause]' |
| `[gasps]` | 놀람 소리. '[gasps] 세상에, 믿을 수가 없네요!' |
| `[clears throat]` | 헛기침. '[clears throat] 이제 발표를 시작합니다.' |
| `♪ ... ♪` | 노래 형식. '♪ 생일 축하합니다 ♪' |

음성 합성 속도와 음조를 조정하는 파라미터들도 중요합니다.

| 파라미터 | 역할 | 권장 범위 |
| --- | --- | --- |
| temperature (text) | 텍스트 코덱 토큰 샘플링 다양성. 낮을수록 안정적. | 0.6 ~ 0.9 |
| temperature (waveform) | 파형 생성 다양성. 낮을수록 깔끔한 발음. | 0.6 ~ 0.8 |
| fine_temperature | 세밀한 음향 조정. 낮을수록 노이즈 감소. | 0.4 ~ 0.6 |
| voice_preset | 화자 스타일 선택. 언어별 프리셋 제공. | v2/ko_speaker_0 등 |

### 5.2.4 반환 형식: .wav / .mp3 / base64 지원

API 사용자가 다양한 환경에서 오디오를 활용할 수 있도록, WAV, MP3 스트리밍 응답과 Base64 인코딩 JSON 응답을 모두 지원합니다.

**routers/tts.py**

```python
# routers/tts.py
import base64                   # base64 출력 형식 인코딩
import io
import uuid
from pathlib import Path
from typing import Optional, Literal

from fastapi import APIRouter, HTTPException, status, BackgroundTasks
# StreamingResponse: 오디오를 파일 저장 없이 바로 스트리밍 응답
from fastapi.responses import StreamingResponse, FileResponse
from pydantic import BaseModel, Field

from services.tts_service import bark_tts, speecht5_tts   # 두 TTS 엔진
import soundfile as sf
import numpy as np

router = APIRouter(prefix='/tts', tags=['Text-to-Speech (TTS)'])  # /tts 경로 그룹

OUTPUT_DIR = Path('output')
OUTPUT_DIR.mkdir(exist_ok=True)


class TTSRequest(BaseModel):
    """음성 합성 요청 본문(JSON) 스키마"""
    text: str = Field(
        ...,
        min_length=1,
        max_length=500,                       # 과도한 길이 방지(최대 500자)
        description='음성으로 변환할 텍스트 (최대 500자)',
    )
    engine: Literal['bark', 'speecht5'] = Field(   # 두 값만 허용
        'speecht5',
        description='TTS 엔진 선택 (bark=고품질/느림, speecht5=빠름)',
    )
    voice_preset: Optional[str] = Field(
        'en_male_1',
        description='Bark 음성 캐릭터 프리셋 (bark 엔진 전용)',
    )
    emotion: Optional[str] = Field(
        'neutral',
        description='감정 태그: neutral/laugh/sigh (bark 엔진 전용)',
    )
    output_format: Literal['wav', 'mp3', 'base64'] = Field(
        'wav',
        description='출력 형식: wav(스트리밍), mp3(스트리밍), base64(JSON)',
    )
    temperature: float = Field(0.7, ge=0.1, le=1.5)  # 0.1~1.5 범위 제한

    model_config = {
        'json_schema_extra': {                # Swagger 문서에 표시될 예시 입력
            'examples': [{
                'text': 'Hello! Welcome to our AI audio service.',
                'engine': 'speecht5',
                'output_format': 'wav',
            }]
        }
    }


@router.post(
    '/synthesize',                            # 텍스트→음성 핵심 엔드포인트
    summary='텍스트를 음성으로 합성',
    description='텍스트를 입력하면 선택한 TTS 엔진으로 음성 파일을 생성합니다.',
    response_class=StreamingResponse,
    responses={                               # 문서용 응답 코드 설명
        200: {'description': 'WAV 또는 MP3 오디오 스트림'},
        422: {'description': '입력 데이터 오류'},
        503: {'description': '모델 미로드'},
    },
)
async def synthesize_speech(request: TTSRequest):
    try:
        # ── 음성 합성 ──────────────────────────────────
        if request.engine == 'bark':
            # 모델이 로드 안 됐으면 503(서비스 불가)
            if bark_tts.model is None:
                raise HTTPException(503, 'Bark 모델 미로드')
            # Bark로 합성 (감정/화자/temperature 반영)
            audio_array = bark_tts.synthesize(
                text=request.text,
                voice_preset=request.voice_preset or 'en_male_1',
                emotion=request.emotion or 'neutral',
                temperature=request.temperature,
            )
            sample_rate = bark_tts.sample_rate   # 24kHz
        else:  # speecht5
            if speecht5_tts.model is None:
                raise HTTPException(503, 'SpeechT5 모델 미로드')
            audio_array = speecht5_tts.synthesize(text=request.text)  # 빠른 합성
            sample_rate = speecht5_tts.sample_rate   # 16kHz

        # ── 출력 형식별 응답 ────────────────────────────
        if request.output_format == 'base64':
            # WAV를 메모리에 쓰고 base64 문자열로 인코딩해 JSON에 담아 반환
            buf = io.BytesIO()
            sf.write(buf, audio_array, sample_rate, format='WAV')
            b64 = base64.b64encode(buf.getvalue()).decode()
            return {
                'text': request.text,
                'engine': request.engine,
                'audio_base64': b64,
                'format': 'wav',
                'sample_rate': sample_rate,
                # 길이(초) = 샘플 수 ÷ 샘플레이트
                'duration_sec': round(len(audio_array) / sample_rate, 2),
            }

        elif request.output_format == 'mp3':
            # pydub으로 MP3 변환 (WAV→MP3 인코딩에 ffmpeg 필요)
            try:
                from pydub import AudioSegment
                wav_buf = io.BytesIO()
                sf.write(wav_buf, audio_array, sample_rate, format='WAV')
                wav_buf.seek(0)               # 읽기 위해 포인터 처음으로
                audio_seg = AudioSegment.from_wav(wav_buf)
                mp3_buf = io.BytesIO()
                audio_seg.export(mp3_buf, format='mp3', bitrate='128k')  # 128kbps
                mp3_buf.seek(0)
                # MP3 바이트 스트림을 다운로드 응답으로 반환
                return StreamingResponse(
                    mp3_buf,
                    media_type='audio/mpeg',
                    headers={'Content-Disposition':
                             'attachment; filename="speech.mp3"'},
                )
            except ImportError:
                # pydub 미설치 시 400으로 안내
                raise HTTPException(
                    400, 'MP3 변환에는 pydub 설치 필요'
                )

        else:  # wav (기본)
            buf = io.BytesIO()
            sf.write(buf, audio_array, sample_rate, format='WAV')
            buf.seek(0)
            # WAV 스트림 반환 + 커스텀 헤더에 길이/샘플레이트 메타 포함
            return StreamingResponse(
                buf,
                media_type='audio/wav',
                headers={
                    'Content-Disposition': 'attachment; filename="speech.wav"',
                    'X-Audio-Duration': str(round(len(audio_array) / sample_rate, 2)),
                    'X-Sample-Rate': str(sample_rate),
                },
            )

    except HTTPException:
        raise                                 # 이미 만든 HTTP 예외는 그대로 전달
    except Exception as e:
        raise HTTPException(500, f'TTS 처리 오류: {str(e)}')  # 기타 오류는 500

    return {
        'bark_presets': list(bark_tts.VOICE_PRESETS.keys()),
        'bark_emotions': list(bark_tts.EMOTION_TAGS.keys()),
        'bark_loaded': bark_tts.model is not None,
        'speecht5_loaded': speecht5_tts.model is not None,
    }
```

## 5.3 FastAPI 음성 챗봇 (텍스트 변환 기반)

이 절에서는 지금까지 구현한 ASR, 챗봇, TTS 서비스를 하나의 파이프라인으로 연결합니다. 사용자가 음성 파일을 업로드하면 Whisper가 텍스트로 변환하고, LLM이 응답 텍스트를 생성하며, 마지막으로 TTS가 그 응답을 다시 음성으로 변환하는 완전한 음성 챗봇 서비스입니다.

### 5.3.1 음성 챗봇 전체 흐름: 음성 → 텍스트 → AI 응답 → 음성

음성 챗봇의 전체 데이터 흐름을 단계별로 정리합니다.

| 처리 단계 | 수행 내용 및 사용 기술 |
| --- | --- |
| ① 입력 수신 | 클라이언트가 HTTP POST로 오디오 파일(WAV/MP3) 업로드 |
| ② 오디오 전처리 | pydub/soundfile로 16kHz 모노 WAV 변환 |
| ③ ASR (음성→텍스트) | Whisper로 음성 인식, 텍스트 추출 |
| ④ 언어 처리 | 챗봇 서비스(DialoGPT/KoGPT)로 응답 텍스트 생성 |
| ⑤ TTS (텍스트→음성) | SpeechT5 또는 Bark로 응답 텍스트를 음성 합성 |
| ⑥ 응답 반환 | WAV 오디오 스트리밍 또는 Base64+텍스트 JSON 반환 |

음성 챗봇 처리 파이프라인:

```
[사용자] ──→ [오디오 파일 업로드] ──→ [FastAPI 라우터]
                        |
                  [오디오 전처리 (WAV 변환)]
                        |
                  [Whisper ASR (음성→텍스트)]
                        |
                  [챗봇 모델 (텍스트→응답)]
                        |
                  [SpeechT5/Bark (텍스트→음성)]
                        |
[사용자] ←── [오디오 응답 스트리밍] ←── [WAV/MP3/Base64 반환]
```

### 5.3.2 챗봇 텍스트 응답 생성 서비스

**services/chatbot_service.py (음성 챗봇용 경량 버전)**

```python
# services/chatbot_service.py (음성 챗봇 통합용)
import torch
# 토크나이저 + 인과적 언어모델(GPT 계열) 자동 로더
from transformers import AutoTokenizer, AutoModelForCausalLM
from typing import Optional


class VoiceChatbotTextService:
    """음성 챗봇용 텍스트 생성 서비스"""

    def __init__(self):
        self.tokenizer = None                 # 텍스트↔토큰 변환기
        self.model = None                     # 응답 생성 LLM
        self.device = 'cuda' if torch.cuda.is_available() else 'cpu'

    def load_model(self, model_id: str = 'microsoft/DialoGPT-small'):
        """
        음성 챗봇에는 DialoGPT-small 사용 권장.
        (음성 처리 비용이 크므로 텍스트 모델은 경량 사용)
        """
        print(f'[ChatBot] Loading model: {model_id}')
        self.tokenizer = AutoTokenizer.from_pretrained(model_id)
        self.model = AutoModelForCausalLM.from_pretrained(
            model_id
        ).to(self.device)
        self.model.eval()                     # 추론 모드(드롭아웃 등 비활성화)
        print('[ChatBot] Loaded.')

    def generate_response(
        self,
        user_text: str,
        max_length: int = 150,                # 생성 최대 토큰 길이
        temperature: float = 0.8,             # 응답 다양성
    ) -> str:
        if self.model is None:
            # 폴백: 모델이 없으면 간단한 규칙 기반 응답으로 대체
            return self._rule_based_response(user_text)

        # 사용자 입력 끝에 EOS 토큰을 붙여 토큰 ID 텐서로 변환
        input_ids = self.tokenizer.encode(
            user_text + self.tokenizer.eos_token,
            return_tensors='pt'
        ).to(self.device)

        with torch.no_grad():                 # 추론이므로 기울기 계산 끔
            output_ids = self.model.generate(
                input_ids,
                max_length=max_length,
                pad_token_id=self.tokenizer.eos_token_id,  # 패딩=EOS 지정
                do_sample=True,               # 확률적 샘플링으로 자연스러운 응답
                temperature=temperature,
                top_p=0.9,                    # 누적확률 90% 토큰만 후보(nucleus)
                repetition_penalty=1.3,       # 같은 말 반복 억제
            )

        # 입력 길이 이후 부분만 잘라 실제 '응답'만 추출
        response_ids = output_ids[:, input_ids.shape[-1]:]
        response = self.tokenizer.decode(
            response_ids[0], skip_special_tokens=True   # 특수토큰 제거
        ).strip()
        # 빈 응답이면 기본 문구 반환
        return response if response else 'I understand. Please continue.'

    def _rule_based_response(self, text: str) -> str:
        """모델 미로드 시 간단한 규칙 기반 응답"""
        greetings = ['hello', 'hi', 'hey', '안녕', '안녕하세요']
        for g in greetings:
            if g in text.lower():             # 인사말 포함 시 인사로 응답
                return '안녕하세요! 무엇을 도와드릴까요?'
        return f'말씀하신 내용을 잘 들었습니다. 더 자세히 말씀해 주시겠어요?'


# 싱글턴 - 음성 챗봇 라우터에서 공유하는 텍스트 생성 서비스
voice_chatbot_text = VoiceChatbotTextService()
```

### 5.3.3 음성 챗봇 통합 라우터 구현

**routers/voice_chat.py**

```python
# routers/voice_chat.py
import os
import io
import time                     # 단계별 처리 시간 측정
from pathlib import Path
from typing import Optional, Literal

from fastapi import (
    APIRouter, UploadFile, File, Form,
    HTTPException, BackgroundTasks,
)
from fastapi.responses import StreamingResponse
from pydantic import BaseModel
import soundfile as sf
import base64

# 음성 챗봇 = ASR + 챗봇 + TTS 세 서비스를 조합
from services.asr_service import asr_service
from services.tts_service import bark_tts, speecht5_tts
from services.chatbot_service import voice_chatbot_text
from services.audio_utils import (
    validate_audio_file, save_upload_file, convert_to_wav,
)

router = APIRouter(prefix='/voice-chat', tags=['Voice Chatbot'])  # /voice-chat 그룹


class VoiceChatResponse(BaseModel):
    """JSON 응답 형식 스키마 (텍스트 + 단계별 시간 + base64 오디오)"""
    user_speech_text: str    # ASR 변환 텍스트
    bot_response_text: str   # 챗봇 응답 텍스트
    asr_time_sec: float                       # 음성 인식 소요 시간
    chat_time_sec: float                      # 응답 생성 소요 시간
    tts_time_sec: float                       # 음성 합성 소요 시간
    total_time_sec: float                     # 전체 파이프라인 시간
    audio_base64: Optional[str] = None  # base64 모드
    audio_format: str = 'wav'


@router.post(
    '/chat',                                  # 음성 챗봇 메인 엔드포인트
    summary='음성 챗봇 (음성→텍스트→AI 응답→음성)',
    description='''
    사용자의 음성 파일을 업로드하면:
    1. Whisper ASR로 텍스트 변환
    2. 챗봇 AI로 응답 텍스트 생성
    3. TTS로 응답 음성 합성 후 반환
    ''',
)
async def voice_chat(
    background_tasks: BackgroundTasks,
    file: UploadFile = File(
        ...,
        description='사용자 음성 파일 (wav/mp3, 최대 20MB)'
    ),
    tts_engine: str = Form(                   # 응답 합성에 쓸 TTS 엔진
        'speecht5',
        description='TTS 엔진 (bark / speecht5)',
    ),
    response_format: Literal['audio', 'json'] = Form(   # 오디오 vs JSON
        'audio',
        description='응답 형식: audio(WAV 스트리밍), json(텍스트+Base64)',
    ),
    language: Optional[str] = Form(
        None,
        description='ASR 언어 코드 (None=자동감지)',
    ),
):
    total_start = time.time()                 # 전체 처리 시간 측정 시작
    content = await file.read()               # 업로드 오디오 바이트 읽기

    try:
        validate_audio_file(file.filename, len(content))   # 형식/용량 검사
    except ValueError as e:
        raise HTTPException(422, str(e))

    # ── STEP 1: 파일 저장 및 WAV 변환 ─────────────
    tmp_path = save_upload_file(content, file.filename)
    ext = Path(file.filename).suffix.lower()
    if ext != '.wav':
        wav_path = tmp_path.replace(ext, '_conv.wav')
        convert_to_wav(tmp_path, wav_path)    # Whisper용 16kHz 모노 WAV로 변환
        asr_input = wav_path
    else:
        asr_input = tmp_path

    try:
        # ── STEP 2: ASR ──────────────────────────── (음성→텍스트)
        asr_start = time.time()
        asr_result = asr_service.transcribe(
            audio_path=asr_input,
            language=language,
        )
        asr_time = round(time.time() - asr_start, 2)
        user_text = asr_result['text']        # 사용자가 말한 내용

        # 인식 결과가 비면 422로 다시 말하도록 안내
        if not user_text.strip():
            raise HTTPException(
                422,
                '음성에서 텍스트를 인식하지 못했습니다. 더 명확하게 말씀해 주세요.'
            )

        # ── STEP 3: 챗봇 응답 ────────────────────── (텍스트→응답)
        chat_start = time.time()
        bot_text = voice_chatbot_text.generate_response(user_text)
        chat_time = round(time.time() - chat_start, 2)

        # ── STEP 4: TTS 합성 ─────────────────────── (응답 텍스트→음성)
        tts_start = time.time()
        # bark 요청 + 모델 로드돼 있으면 bark, 아니면 빠른 SpeechT5로 폴백
        if tts_engine == 'bark' and bark_tts.model:
            audio_array = bark_tts.synthesize(bot_text)
            sample_rate = bark_tts.sample_rate
        else:
            audio_array = speecht5_tts.synthesize(bot_text)
            sample_rate = speecht5_tts.sample_rate
        tts_time = round(time.time() - tts_start, 2)

        total_time = round(time.time() - total_start, 2)   # 전체 소요 시간

        # ── STEP 5: 응답 반환 ──────────────────────
        if response_format == 'json':
            # JSON: 텍스트 + 시간 + base64 오디오를 한 번에 반환
            buf = io.BytesIO()
            sf.write(buf, audio_array, sample_rate, format='WAV')
            audio_b64 = base64.b64encode(buf.getvalue()).decode()
            return VoiceChatResponse(
                user_speech_text=user_text,
                bot_response_text=bot_text,
                asr_time_sec=asr_time,
                chat_time_sec=chat_time,
                tts_time_sec=tts_time,
                total_time_sec=total_time,
                audio_base64=audio_b64,
                audio_format='wav',
            )
        else:  # audio 스트리밍
            buf = io.BytesIO()
            sf.write(buf, audio_array, sample_rate, format='WAV')
            buf.seek(0)
            # 오디오 본문은 스트리밍, 텍스트/시간 정보는 헤더에 담아 반환
            return StreamingResponse(
                buf,
                media_type='audio/wav',
                headers={
                    'X-User-Text': user_text[:100],   # 헤더 길이 제한 위해 100자만
                    'X-Bot-Text': bot_text[:100],
                    'X-Total-Time': str(total_time),
                    'X-ASR-Time': str(asr_time),
                    'X-TTS-Time': str(tts_time),
                },
            )

    except HTTPException:
        raise
    except Exception as e:
        raise HTTPException(500, f'음성 챗봇 오류: {str(e)}')
    finally:
        # 응답 후 임시 파일들 비동기 정리
        background_tasks.add_task(os.unlink, tmp_path)
        if ext != '.wav' and os.path.exists(asr_input):
            background_tasks.add_task(os.unlink, asr_input)


@router.post(
    '/transcribe-only',                       # 챗봇/TTS 없이 ASR만 수행
    summary='음성→텍스트만 반환 (챗봇 응답 없음)',
)
async def transcribe_only(
    background_tasks: BackgroundTasks,
    file: UploadFile = File(...),
    language: Optional[str] = Form(None),
):
    """디버깅 또는 ASR 단독 테스트용"""
    content = await file.read()
    validate_audio_file(file.filename, len(content))   # 형식/용량 검사
    tmp = save_upload_file(content, file.filename)      # 임시 저장
    ext = Path(file.filename).suffix.lower()
    if ext != '.wav':
        wav = tmp.replace(ext, '_c.wav')
        convert_to_wav(tmp, wav)              # 필요 시 WAV 변환
        process = wav
    else:
        process = tmp
    try:
        result = asr_service.transcribe(process, language=language)  # ASR만 실행
        return {'text': result['text'], 'language': result['language']}
    finally:
        # 응답 후 임시 파일 비동기 삭제
        background_tasks.add_task(os.unlink, tmp)
        if ext != '.wav':
            background_tasks.add_task(os.unlink, process)
```

>
> 음성 챗봇의 전체 응답 시간은 ASR + 텍스트 생성 + TTS 시간의 합입니다. 각 단계별 최적화 방법:

> - Whisper tiny/base 사용 (CPU 환경)

> **챗봇 텍스트 생성 최적화:**
> - DialoGPT-small 또는 경량 LLM 사용
> - 응답 max_length를 50~100 토큰으로 제한
> - Hugging Face Inference API 활용 (서버 자원 절약)
>
> **TTS 최적화:**
> - SpeechT5가 Bark보다 10~50배 빠름 (Bark는 GPU 필수)
> - 자주 사용되는 응답 문구는 미리 합성하여 캐시
>
> **GPU 사용 시 전체 파이프라인 목표:**
> - Whisper base + DialoGPT-small + SpeechT5 = 약 2~4초 (T4 GPU)

## 5.4 오디오 파일 업로드 및 처리

이 절에서는 FastAPI의 파일 업로드 처리 기능을 깊이 있게 살펴봅니다. UploadFile의 동작 원리, 대용량 파일 스트리밍 처리, BackgroundTasks를 이용한 비동기 후처리, 그리고 처리된 파일을 다운로드하는 API까지 완성합니다.

### 5.4.1 FastAPI에서 UploadFile로 .wav, .mp3 파일 처리하기

FastAPI의 파일 업로드는 크게 두 가지 방식으로 이루어집니다. 첫 번째는 `UploadFile`을 사용하는 multipart/form-data 방식이며, 두 번째는 `Request.body()`로 바이너리 데이터를 직접 받는 방식입니다. 오디오 파일처럼 큰 파일은 `UploadFile` 방식을 권장합니다.

**UploadFile 내부 동작 원리**

FastAPI의 `UploadFile`은 파이썬의 `SpooledTemporaryFile`을 래핑한 클래스입니다. 파일 크기가 작으면 메모리에, 크면 임시 파일로 자동 관리합니다.

```python
from fastapi import UploadFile, File, Form
from typing import List


# ─── 단일 파일 업로드 ────────────────────────────
@router.post('/upload/single')
async def upload_single(
    file: UploadFile = File(
        ...,
        description='오디오 파일 (WAV, MP3)',
    ),
):
    # 파일 메타데이터 접근 - 본문을 읽지 않아도 이름/타입은 바로 확인 가능
    print(f'파일명: {file.filename}')
    print(f'MIME 타입: {file.content_type}')

    # 전체 내용 읽기 (소용량 파일용) - 비동기 read로 전체 바이트 확보
    content = await file.read()
    file_size = len(content)
    print(f'파일 크기: {file_size / 1024:.1f} KB')

    # 파일 포인터 초기화 (재읽기 시 필요) - 다시 read하려면 처음으로 되감아야 함
    await file.seek(0)

    return {
        'filename': file.filename,

    file: UploadFile = File(...),
):
    """대용량 파일을 청크 단위로 처리 (메모리 절약)"""
    import uuid
    # 충돌 없는 저장 경로 생성(원본 확장자 유지)
    save_path = f'temp/{uuid.uuid4()}{Path(file.filename).suffix}'
    total_size = 0
    chunk_size = 1024 * 1024  # 1MB 청크

    with open(save_path, 'wb') as f:
        # 전체를 한 번에 읽지 않고 1MB씩 읽어 바로 디스크에 기록(메모리 절약)
        while chunk := await file.read(chunk_size):
            f.write(chunk)
            total_size += len(chunk)          # 누적 크기 합산

    return {
        'saved_path': save_path,
        'total_size_mb': round(total_size / 1024 / 1024, 2)
    }
```

### 5.4.2 업로드된 파일을 서버 임시 저장 → 모델에 전달 → 결과 반환

파일 처리 파이프라인에서 임시 파일 관리는 매우 중요합니다. 처리가 완료된 임시 파일은 반드시 삭제하여 디스크 공간을 낭비하지 않아야 하며, BackgroundTasks를 활용하면 응답을 먼저 반환한 후 정리 작업을 수행할 수 있습니다.

| 파일 관리 시점 | 처리 방법 |
| --- | --- |
| 업로드 즉시 | `save_upload_file()`로 temp/ 디렉터리에 UUID 기반 파일명으로 저장 |
| 포맷 변환 | `convert_to_wav()`로 변환 후 원본 임시파일 즉시 삭제 |
| 모델 처리 중 | 처리 중인 파일은 유지 (try/finally 블록으로 보장) |
| 응답 반환 직후 | `BackgroundTasks.add_task(os.unlink, path)`로 비동기 삭제 |
| 주기적 정리 | `cleanup_temp_files()`를 스케줄러 또는 startup 이벤트에 등록 |

**완전한 오디오 처리 라우터 구현**

```python
# routers/audio_upload.py
import os
import uuid
import shutil
import time
from pathlib import Path
from typing import Optional, List

from fastapi import (
    APIRouter, UploadFile, File, Form,
    HTTPException, BackgroundTasks, status,
)
# FileResponse: 디스크의 파일을 다운로드로 응답
from fastapi.responses import FileResponse, StreamingResponse
from pydantic import BaseModel

from services.audio_utils import (
    validate_audio_file, convert_to_wav,
    get_audio_info, save_upload_file,
    SUPPORTED_AUDIO_FORMATS, OUTPUT_DIR, TEMP_DIR,
)

router = APIRouter(prefix='/audio', tags=['Audio File Management'])  # /audio 그룹

# 처리된 파일 목록 (메모리 내 간단 레지스트리)
# file_id → 경로/원본명/상태 등을 저장 (실제 서비스라면 DB 사용 권장)
processed_files: dict = {}


class AudioFileInfo(BaseModel):
    """업로드/처리된 파일 메타데이터 응답 스키마"""
    file_id: str                              # 서버가 부여한 고유 ID
    filename: str
    duration_sec: Optional[float]
    sample_rate: Optional[int]
    channels: Optional[int]
    size_bytes: int
    format: str
    status: str  # 'uploaded' | 'processing' | 'done' | 'error'


@router.post(
    '/upload',
    response_model=AudioFileInfo,
    summary='오디오 파일 업로드',
    description='오디오 파일을 서버에 업로드하고 파일 정보를 반환합니다.',
)
async def upload_audio(
    file: UploadFile = File(
        ...,
        description=f'오디오 파일. 지원 형식: {SUPPORTED_AUDIO_FORMATS}',
    ),
    convert_to_wav_flag: bool = Form(
        True,
        description='True면 16kHz 모노 WAV로 자동 변환',
    ),
):
    content = await file.read()
    size = len(content)

    try:
        validate_audio_file(file.filename, size)   # 형식/용량 검사
    except ValueError as e:
        raise HTTPException(422, str(e))

    # 저장 - file_id를 파일명으로 사용해 temp/에 기록
    file_id = str(uuid.uuid4())
    ext = Path(file.filename).suffix.lower()
    save_path = str(TEMP_DIR / f'{file_id}{ext}')
    with open(save_path, 'wb') as f:
        f.write(content)

    # WAV 변환 - 플래그가 켜져 있고 원본이 WAV가 아니면 변환 후 원본 삭제
    if convert_to_wav_flag and ext != '.wav':
        wav_path = str(TEMP_DIR / f'{file_id}.wav')
        convert_to_wav(save_path, wav_path)
        os.unlink(save_path)                  # 원본 임시파일 제거(공간 절약)
        final_path = wav_path
        final_ext = '.wav'
    else:
        final_path = save_path
        final_ext = ext

    audio_info = get_audio_info(final_path)    # 길이/샘플레이트 등 추출
    # 레지스트리에 등록 → 이후 download/process/delete에서 file_id로 조회
    processed_files[file_id] = {
        'path': final_path,
        'original_name': file.filename,
        'status': 'uploaded',
        'uploaded_at': time.time(),
    }

    return AudioFileInfo(
        file_id=file_id,
        filename=file.filename,
        duration_sec=audio_info.get('duration_sec'),
        sample_rate=audio_info.get('sample_rate'),
        channels=audio_info.get('channels'),
        size_bytes=size,
        format=final_ext.lstrip('.'),         # 점 제거(.wav → wav)
        status='uploaded',
    )


@router.get(
    '/download/{file_id}',                    # 경로 변수로 파일 ID 받기
    summary='처리된 오디오 파일 다운로드',
    response_class=FileResponse,
)
async def download_audio(file_id: str):
    # 레지스트리에 없는 ID면 404
    if file_id not in processed_files:
        raise HTTPException(
            status_code=status.HTTP_404_NOT_FOUND,
            detail=f'파일 ID {file_id}를 찾을 수 없습니다.',
        )
    file_info = processed_files[file_id]
    file_path = file_info['path']

    # 등록은 됐지만 실제 파일이 사라졌으면 레지스트리에서도 제거 후 404
    if not os.path.exists(file_path):
        del processed_files[file_id]
        raise HTTPException(404, '파일이 서버에 존재하지 않습니다.')

    # 파일을 다운로드 응답으로 반환(원본명 기반 파일명 지정)
    return FileResponse(
        path=file_path,
        media_type='audio/wav',
        filename=f'processed_{file_info["original_name"]}',
    )


@router.delete('/delete/{file_id}', summary='업로드 파일 삭제')
async def delete_audio(file_id: str):
    if file_id not in processed_files:
        raise HTTPException(404, '파일을 찾을 수 없습니다.')
    info = processed_files.pop(file_id)       # 레지스트리에서 제거하며 정보 획득
    path = info['path']
    if os.path.exists(path):
        os.unlink(path)                       # 실제 파일 삭제
    return {'message': f'파일 {file_id} 삭제 완료'}


@router.get('/list', summary='업로드된 파일 목록')
async def list_audio_files():
    result = []
    # 등록된 모든 파일의 상태와 실제 존재 여부를 함께 반환
    for fid, info in processed_files.items():
        result.append({
            'file_id': fid,
            'original_name': info['original_name'],

            'path_exists': os.path.exists(info['path']),

    return {'files': result, 'total': len(result)}
```

### 5.4.3 업로드 후 처리 API + 다운로드 API 구현 예제

업로드된 파일을 특정 작업(포맷 변환, 음성 향상, 구간 추출 등)으로 처리한 뒤, 처리된 파일을 다운로드할 수 있는 완전한 파이프라인을 구현합니다.

```python
# routers/audio_upload.py (처리 엔드포인트 추가)

@router.post(
    '/process/{file_id}',                     # 업로드된 파일을 변형하는 엔드포인트
    summary='업로드 파일 처리 (포맷 변환, 구간 추출 등)',
)
async def process_audio(
    file_id: str,
    background_tasks: BackgroundTasks,
    operation: str = Form(                    # 어떤 처리를 할지 선택
        'convert',
        description='처리 유형: convert(변환)/trim(구간추출)/normalize(정규화)',
    ),
    start_sec: float = Form(0.0, description='구간 추출 시작 시간(초)'),
    end_sec: float = Form(0.0, description='구간 추출 종료 시간(초). 0=끝까지'),
    target_sample_rate: int = Form(16000, description='목표 샘플링 레이트'),
):
    if file_id not in processed_files:        # 등록 안 된 ID면 404
        raise HTTPException(404, '파일을 찾을 수 없습니다.')

    info = processed_files[file_id]
    input_path = info['path']

    if not os.path.exists(input_path):        # 실제 파일 없으면 404
        raise HTTPException(404, '파일이 서버에 없습니다.')

    try:
        import soundfile as sf
        import numpy as np

        data, sr = sf.read(input_path)        # 원본 샘플 배열 + 샘플레이트
        output_id = str(uuid.uuid4())         # 결과물 새 ID
        output_path = str(OUTPUT_DIR / f'{output_id}.wav')

        if operation == 'trim':
            # 구간 추출 - 시작/종료 초를 샘플 인덱스로 환산해 슬라이싱
            s_sample = int(start_sec * sr)
            e_sample = int(end_sec * sr) if end_sec > 0 else len(data)
            data = data[s_sample:e_sample]

        elif operation == 'normalize':
            # 볼륨 정규화 (RMS 기반) - 평균 음량을 일정 수준으로 맞춤
            rms = np.sqrt(np.mean(data ** 2))
            if rms > 0:                       # 무음(0) 나눗셈 방지
                data = data / rms * 0.1

        elif operation == 'convert':
            # 샘플링 레이트 변환 - 목표와 다를 때만 리샘플
            if sr != target_sample_rate:
                import scipy.signal as signal
                n = int(len(data) * target_sample_rate / sr)
                data = signal.resample(data, n)
                sr = target_sample_rate

        sf.write(output_path, data, sr)       # 처리 결과를 output/에 저장

        # 처리된 파일 등록 - 결과물도 레지스트리에 추가해 다운로드 가능하게
        processed_files[output_id] = {
            'path': output_path,
            'original_name': f'processed_{info["original_name"]}',
            'status': 'done',
            'uploaded_at': time.time(),
        }

        return {
            'source_file_id': file_id,
            'output_file_id': output_id,
            'operation': operation,
            'download_url': f'/audio/download/{output_id}',   # 결과 다운로드 경로
            'audio_info': get_audio_info(output_path),
        }

    except Exception as e:
        raise HTTPException(500, f'파일 처리 오류: {str(e)}')
```

### 5.4.4 File, Form, BackgroundTasks 사용법 포함

FastAPI에서 파일과 폼 데이터를 함께 처리할 때 주의할 점과 BackgroundTasks의 다양한 활용 패턴을 정리합니다.

| FastAPI 파라미터 타입 | 역할 및 사용 상황 |
| --- | --- |
| `File(...)` | 필수 파일 업로드 파라미터. UploadFile과 함께 사용. |
| `File(None)` | 선택적 파일 업로드. Optional[UploadFile]과 함께 사용. |
| `Form(...)` | 필수 폼 데이터. multipart/form-data 요청에서 텍스트 파라미터. |
| `Form(default)` | 선택적 폼 데이터. 기본값과 함께 사용. |
| `BackgroundTasks` | 응답 반환 후 실행될 작업 등록. 임시 파일 삭제, 알림 전송 등. |

> **⚠ 파일 + JSON Body 동시 사용 불가**
>
> FastAPI에서 `UploadFile`과 Pydantic BaseModel(JSON Body)은 같은 엔드포인트에 동시에 사용할 수 없습니다. (multipart/form-data vs application/json 충돌)

**해결책 1: 모든 파라미터를 Form()으로 정의**

```python
async def endpoint(
    file: UploadFile = File(...),             # 파일은 File()로
    name: str = Form(...),                    # 텍스트는 모두 Form()으로 통일
    language: str = Form('ko'),               # 같은 multipart 요청으로 묶임
):
    ...
```

**해결책 2: JSON을 문자열 폼으로 전달 후 파싱**

```python
async def endpoint(
    file: UploadFile = File(...),
    metadata: str = Form('{}'),  # JSON 문자열 (Body 대신 문자열 폼으로 전달)

    import json
    data = json.loads(metadata)               # 문자열을 딕셔너리로 파싱
```

**해결책 3: 파일 업로드와 설정을 별도 엔드포인트로 분리**

**BackgroundTasks 고급 활용 패턴**

```python
    logger.info(f'Post-processing {file_id}: {text[:50]}')
    # await db.save(file_id, text)            # 예: DB 저장

@router.post('/transcribe-async')
async def transcribe_async(

    content = await file.read()
    tmp = save_upload_file(content, file.filename)
```

> **TIP: UploadFile 완전 정리**
>
> FastAPI UploadFile 사용 시 꼭 알아야 할 핵심 사항:
>
> 1. **항상 `await file.read()`로 비동기 읽기**
>    - `file.read()` (동기) 사용 시 블로킹 발생
> 2. **대용량 파일은 청크 읽기 사용**
>    - `while chunk := await file.read(1024*1024): ...`
> 3. **multipart/form-data 헤더 필수**
>    - Swagger UI는 자동 처리
>    - curl: `-F 'file=@path.wav'` (Content-Type 자동 설정)
>    - Python requests: `files={'file': open('audio.wav', 'rb')}`
> 4. **파일명 인코딩 주의**
>    - 한글 파일명은 `urllib.parse.unquote(file.filename)` 처리 권장
> 5. **content_type 신뢰 금지**
>    - 클라이언트가 임의 설정 가능. 파일 확장자로 검증 권장.

## 5.5 통합 main.py 및 전체 서비스 실행

지금까지 구현한 모든 서비스를 하나의 FastAPI 앱으로 통합합니다. lifespan 이벤트 핸들러로 모델을 초기화하고, 각 라우터를 등록하는 완성된 main.py를 작성합니다.

```python
# main.py
import os
from contextlib import asynccontextmanager    # lifespan 핸들러 작성용
from fastapi import FastAPI, Request
from fastapi.middleware.cors import CORSMiddleware   # 교차 출처 허용
from fastapi.responses import JSONResponse
import logging

# 로거 설정 - 시간/이름/레벨/메시지 형식으로 콘솔 출력
logging.basicConfig(
    level=logging.INFO,
    format='%(asctime)s | %(name)-20s | %(levelname)-8s | %(message)s',
    datefmt='%Y-%m-%d %H:%M:%S',

@asynccontextmanager
async def lifespan(app: FastAPI):
    """서버 시작/종료 시 실행되는 이벤트 핸들러"""
    # yield 이전 = 서버 시작 시 1회 실행(모델 로드)
    logger.info('=' * 60)
    logger.info('Audio AI Services Starting...')
    logger.info('=' * 60)

    # 환경변수로 로드할 모델 제어 - 필요한 모델만 켜서 메모리 절약
    if os.getenv('LOAD_ASR', 'true').lower() == 'true':
        asr_service.load_model()

    if os.getenv('LOAD_SPEECHT5', 'true').lower() == 'true':
        speecht5_tts.load_model()

    if os.getenv('LOAD_BARK', 'false').lower() == 'true':
        bark_tts.load_model()  # 메모리 큼, 선택적 로드 (기본 false)

    if os.getenv('LOAD_CHATBOT', 'true').lower() == 'true':
        voice_chatbot_text.load_model()

    # 임시 파일 초기 정리
    cleanup_temp_files(max_age_sec=0)  # 이전 실행 파일 전체 삭제

    logger.info('All models loaded. Server ready!')
    logger.info('Docs: http://localhost:8000/docs')
    yield                                     # 이 시점부터 요청 처리

    # 종료 시 정리 - yield 이후는 서버 종료 시 실행
    logger.info('Shutting down...')
    cleanup_temp_files(max_age_sec=0)
```

## 5장 학습 정리

- Whisper 모델의 아키텍처와 5가지 크기 버전을 이해하고 ASR API를 완성했습니다.
- SRT 자막 자동 생성 기능을 타임스탬프 청크 기반으로 구현했습니다.
- Bark와 SpeechT5 두 가지 TTS 엔진을 비교하고, WAV/MP3/Base64 출력을 지원했습니다.
- ASR + 챗봇 + TTS를 연결한 완전한 음성 챗봇 파이프라인을 구현했습니다.
- UploadFile의 동작 원리와 청크 처리, 폼 데이터 처리 방법을 심층 이해했습니다.
- BackgroundTasks로 응답 후 임시 파일을 비동기 정리하는 패턴을 마스터했습니다.
- FileResponse와 StreamingResponse로 파일 다운로드 API를 구현했습니다.
- CORS 미들웨어, 전역 예외 핸들러, lifespan 이벤트를 활용한 완성도 높은 앱을 완성했습니다.

## 5장 요약

이 장에서는 오디오 AI 기술을 FastAPI와 결합하여 4가지 실용적인 서비스를 구현했습니다.

- **Whisper ASR API**: OpenAI Whisper 모델을 Hugging Face Pipeline으로 래핑하여 WAV, MP3 등 다양한 포맷의 음성 파일을 텍스트로 변환하는 API를 구현했습니다. 타임스탬프 기반 SRT 자막 자동 생성 기능도 추가했습니다.
- **TTS API**: Bark(고품질, 감정 표현 가능)와 SpeechT5(빠른 처리)를 모두 구현하고 WAV 스트리밍, MP3 변환, Base64 JSON 세 가지 응답 형식을 지원했습니다.
- **음성 챗봇**: Whisper ASR → DialoGPT 챗봇 → TTS 세 단계 파이프라인을 하나의 엔드포인트로 구성했습니다. 응답 헤더에 처리 시간 메타데이터를 포함하여 성능 모니터링을 지원합니다.
- **오디오 파일 관리**: UploadFile의 심층 활용, 청크 스트리밍 읽기, File/Form 혼합 처리, BackgroundTasks 기반 비동기 파일 정리, FileResponse 다운로드 API를 구현했습니다.
