# 6장. 컴퓨터 비전 웹 서비스

이 장에서는 Hugging Face와 외부 CV 모델을 활용해 이미지를 분석하고 처리하는 AI 웹 서비스를 FastAPI 기반으로 구축합니다. 이미지 분류, 객체 탐지, 이미지 설명 생성 API를 차례로 구현하고, 마지막으로 Jinja2 템플릿을 이용해 세 서비스를 하나의 웹 UI로 통합합니다. 이 장을 마치면 이미지 파일 업로드 → AI 추론 → 결과 반환의 전 과정을 경험하게 됩니다.

## 이 장의 학습 목표

- 이미지 분류 모델(ResNet, ViT)을 pipeline으로 래핑하여 업로드 기반 분류 API를 구현합니다.
- YOLOv8로 객체 탐지 API를 만들고 바운딩 박스 시각화 이미지를 반환합니다.
- BLIP 모델로 이미지 내용을 자연어로 설명하는 이미지 캡셔닝 API를 구현합니다.
- Jinja2 템플릿으로 세 가지 CV 서비스를 하나의 웹 UI로 통합하여 완성합니다.

## 이 장의 개요

컴퓨터 비전(Computer Vision, CV)은 컴퓨터가 이미지와 영상에서 의미 있는 정보를 인식하고 이해하는 AI 분야입니다. 스마트폰 얼굴 인식, 자율주행 차량 장애물 감지, 의료 영상 진단, 보안 CCTV 분석 등 수많은 현실 서비스의 핵심 기술입니다.

Hugging Face는 ResNet, ViT, ConvNeXt 등 이미지 분류 모델과 BLIP, GIT 등 멀티모달 모델을 Pipeline API로 통합 제공하며, ultralytics의 YOLOv8은 객체 탐지 분야의 사실상 표준 모델입니다. 이 두 생태계를 FastAPI로 결합하면 엔터프라이즈 수준의 CV 웹 서비스를 빠르게 구축할 수 있습니다.

## 6.1 이미지 분류 모델로 웹 앱 만들기

이미지 분류(Image Classification)는 입력된 이미지가 어떤 카테고리에 속하는지를 판별하는 가장 기본적인 컴퓨터 비전 태스크입니다. '이 사진은 고양이입니다', '이 X-Ray 이미지에는 폐렴 소견이 있습니다' 등의 판별이 이에 해당합니다.

### 6.1.1 이미지 분류란? (ResNet, ViT, ConvNeXt 등 모델 구조 개요)

이미지 분류 모델의 역사는 CNN(Convolutional Neural Network)에서 시작합니다. 2012년 AlexNet이 ImageNet 대회에서 압도적인 성능을 보인 이후, 더 깊고 정확한 CNN 모델들이 개발되었습니다. 2020년대에는 NLP에서 성공한 Transformer 구조가 이미지 분야로 이전되어 ViT(Vision Transformer) 계열이 CNN을 능가하기 시작했습니다.

| 모델 | 아키텍처 | ImageNet Top-1 | 특징 |
| --- | --- | --- | --- |
| ResNet-50 | CNN (잔차 연결) | 76.1% | 경량·빠름·범용. 전이학습 베이스라인. |
| EfficientNet-B4 | CNN (NAS 최적화) | 82.9% | 파라미터 효율 최고. 모바일 배포용. |
| ViT-Base/16 | Pure Transformer | 81.8% | 패치 기반 자기 어텐션. 대규모 데이터 필요. |
| DeiT-Base | Distilled ViT | 81.8% | 지식 증류로 소규모 데이터에서도 학습 가능. |
| ConvNeXt-Base | CNN + Transformer | 83.8% | CNN 구조에 Transformer 기법 적용. |
| Swin-Transformer-B | Hierarchical ViT | 83.5% | 윈도우 어텐션. 밀집 예측 태스크에 강점. |

이미지 분류의 핵심 처리 파이프라인을 단계별로 살펴봅니다.

1. **이미지 로드**: PIL 또는 OpenCV로 이미지 읽기
2. **전처리**: 크기 조정(224×224 등), 정규화(채널 평균/표준편차 기준)
3. **텐서 변환**: numpy 배열 → PyTorch Tensor, [H,W,C] → [B,C,H,W]
4. **모델 추론**: model(tensor) → 클래스 점수 벡터(logits)
5. **후처리**: Softmax → 확률값 → Top-K 클래스 추출
6. **결과 반환**: 클래스 레이블, 신뢰도 점수 JSON 반환

### 6.1.2 Hugging Face pipeline(task='image-classification') 실습

Hugging Face Pipeline API는 이미지 분류의 전처리~후처리를 모두 자동화해줍니다. 단 몇 줄의 코드로 최신 이미지 분류 모델을 즉시 사용할 수 있습니다.

#### services/classification_service.py

```python
# services/classification_service.py
# 이미지 분류 비즈니스 로직을 담당하는 서비스 계층. 라우터(API)와 분리해 재사용성을 높인다.
import io, time, torch, base64   # io: 바이트 스트림, time: 추론 시간 측정, torch: GPU 확인, base64: 인코딩 디코딩
from typing import List, Dict, Optional
from PIL import Image            # PIL(Pillow): 이미지 파일을 열고 RGB 변환 등 전처리에 사용
from transformers import pipeline  # Hugging Face pipeline: 전처리~추론~후처리를 한 번에 묶어주는 고수준 API


class ImageClassificationService:
    # 키(별칭) → 실제 Hugging Face 모델 ID 매핑. 사용자는 짧은 키만 알면 되도록 추상화한다.
    SUPPORTED_MODELS = {
        'vit-base':      'google/vit-base-patch16-224',          # Vision Transformer 기본형 (권장 베이스라인)
        'convnext-base': 'facebook/convnext-base-224-22k-1k',    # CNN+Transformer 기법, 고품질
        'resnet-50':     'microsoft/resnet-50',                  # 가볍고 빠른 전통적 CNN
        'efficientnet':  'google/efficientnet-b4',               # 파라미터 효율 최적화 모델
        'deit-base':     'facebook/deit-base-distilled-patch16-224',  # 지식 증류로 소규모 데이터에도 강한 ViT
    }

    def __init__(self, model_key: str = 'vit-base'):
        self.model_key = model_key   # 현재 선택된 모델 키 (기본값 vit-base)
        self.pipe = None             # pipeline 객체. 모델 로드 전에는 None
        # GPU(CUDA)가 있으면 0번 장치, 없으면 -1(CPU). pipeline의 device 인자 규칙에 맞춘다.
        self.device = 0 if torch.cuda.is_available() else -1

    def load_model(self, model_key: Optional[str] = None):
        # 모델을 메모리에 적재. 키가 주어지면 그 키, 아니면 현재 키를 사용한다.
        key = model_key or self.model_key
        # 잘못된 키가 들어오면 안전하게 기본 모델(vit-base)로 대체한다.
        model_id = self.SUPPORTED_MODELS.get(key, self.SUPPORTED_MODELS['vit-base'])
        print(f'[Classification] Loading: {model_id}')
        # 핵심: image-classification 태스크 pipeline 생성. 이미지 크기 조정/정규화/추론/Softmax를 자동 처리한다.
        self.pipe = pipeline(
            task='image-classification',  # 태스크 종류 지정 → 적절한 전처리기와 후처리기가 자동 선택됨
            model=model_id,               # 사용할 사전학습 모델
            device=self.device,           # GPU(0) 또는 CPU(-1)
        )
        self.model_key = key   # 실제 로드된 모델 키를 저장해 중복 로드를 방지한다.

    def classify_from_file(self, image_bytes: bytes, top_k: int = 5) -> Dict:
        # 바이트 형태의 이미지를 받아 분류 결과 dict를 반환하는 핵심 메서드.
        if self.pipe is None:
            raise RuntimeError('분류 모델이 로드되지 않았습니다.')  # 모델 미로드 시 방어
        start = time.time()   # 추론 시작 시각 기록
        # 바이트 → PIL 이미지 변환 후 RGB로 통일 (PNG의 투명 채널 등 변수 제거)
        image = Image.open(io.BytesIO(image_bytes)).convert('RGB')
        # pipeline 실행: 상위 top_k개 클래스를 점수 순으로 반환 (각 항목은 label/score를 가짐)
        results = self.pipe(image, top_k=top_k)
        elapsed = round(time.time() - start, 3)   # 추론에 걸린 시간(초)
        return {
            # 예측 목록을 순위(rank), 라벨, 점수(0~1), 백분율(%) 형태로 가공
            'predictions': [
                {'rank': i + 1, 'label': r['label'],
                 'score': round(r['score'], 6),
                 'percentage': round(r['score'] * 100, 2)}
                for i, r in enumerate(results)
            ],
            'top_prediction': results[0]['label'],          # 1순위 라벨
            'top_score': round(results[0]['score'], 6),     # 1순위 신뢰도
            'model_used': self.SUPPORTED_MODELS[self.model_key],  # 실제 사용된 모델 ID
            'inference_time_sec': elapsed,                  # 추론 소요 시간
            'image_size': f'{image.width}x{image.height}',  # 원본 이미지 해상도
        }

    def classify_from_base64(self, b64_string: str, top_k: int = 5) -> Dict:
        # Base64 문자열로 받은 이미지를 분류. JSON 본문으로 이미지를 보낼 때 사용한다.
        if ',' in b64_string:
            # "data:image/jpeg;base64,XXXX" 형태면 콤마 뒤 실제 데이터만 추출
            b64_string = b64_string.split(',', 1)[1]
        # Base64 디코딩 → 바이트로 변환 후 공통 메서드 재사용
        return self.classify_from_file(base64.b64decode(b64_string), top_k)

    def classify_from_url(self, url: str, top_k: int = 5) -> Dict:
        # URL의 원격 이미지를 내려받아 분류한다.
        import httpx
        resp = httpx.get(url, timeout=10)   # 10초 타임아웃으로 이미지 요청
        resp.raise_for_status()             # HTTP 오류(4xx/5xx)면 예외 발생
        return self.classify_from_file(resp.content, top_k)  # 응답 바이트를 분류


# 앱 전역에서 공유하는 싱글톤 인스턴스. 모델을 한 번만 로드해 재사용한다.
classification_service = ImageClassificationService(model_key='vit-base')
```

### 6.1.3 UploadFile로 이미지 업로드 받고 분류 결과 반환

> **TIP: Base64 인코딩 이미지를 JSON으로 받는 방법**
>
> 파일 업로드(multipart/form-data) 대신 Base64 인코딩으로 이미지를 JSON Body에 포함시키면 단일 JSON 요청으로 이미지와 메타데이터를 함께 전송할 수 있습니다.
>
> Python에서 이미지를 Base64로 변환:
>
> ```python
 import base64
> # 이미지 파일을 바이너리('rb')로 열기
> with open('image.jpg', 'rb') as f:
>     # 파일 바이트를 Base64로 인코딩 후 ASCII 문자열로 디코딩
>     b64 = base64.b64encode(f.read()).decode()
> # data URI 형식(접두어 포함)으로 JSON 페이로드에 담는다
> payload = {'image_base64': f'data:image/jpeg;base64,{b64}'}
> ```
>
> 단점: 파일 크기가 약 33% 증가합니다. 10MB 이미지 → 약 13.3MB JSON. 대용량 이미지는 UploadFile 방식을 권장합니다.

#### models/schemas.py – 이미지 분류 스키마

```python
# models/schemas.py
# Pydantic 모델로 요청/응답의 데이터 형태(스키마)를 정의한다. FastAPI가 이를 검증·문서화에 활용한다.
from pydantic import BaseModel, Field
from typing import List, Dict, Optional


class PredictionItem(BaseModel):
    # 예측 1건의 구조: 순위/라벨/점수(0~1)/백분율
    rank: int
    label: str
    score: float
    percentage: float


class ClassificationResponse(BaseModel):
    # 분류 API의 응답 스키마. 이 형태를 벗어나면 FastAPI가 오류를 낸다.
    filename: str
    predictions: List[PredictionItem]   # 위 PredictionItem의 리스트
    top_prediction: str
    top_score: float
    model_used: str
    inference_time_sec: float
    image_size: str


class Base64ClassifyRequest(BaseModel):
    # Base64 분류 요청 본문 스키마.
    # Field(..., ...)의 ...은 '필수값'을 의미한다.
    image_base64: str = Field(..., description='Base64 인코딩 이미지')
    # top_k는 기본 5, 1~20 범위로 제한(ge=이상, le=이하)
    top_k: int = Field(5, ge=1, le=20)

    # Swagger 문서에 표시될 입력 예시를 지정한다.
    model_config = {
        'json_schema_extra': {
            'examples': [{'image_base64': 'data:image/jpeg;base64,...', 'top_k': 5}]
        }
    }
```

#### routers/classification.py

```python
# routers/classification.py
# 이미지 분류 관련 HTTP 엔드포인트(API 경로)를 정의하는 라우터 계층.
from fastapi import APIRouter, UploadFile, File, Form, HTTPException
from typing import Optional
from models.schemas import ClassificationResponse, Base64ClassifyRequest
from services.classification_service import classification_service

# prefix='/classify' → 이 라우터의 모든 경로 앞에 /classify가 붙는다. tags는 Swagger 그룹명.
router = APIRouter(prefix='/classify', tags=['Image Classification'])

# 허용할 이미지 확장자 집합 (set이라 in 검사가 빠름)
SUPPORTED_FORMATS = {'.jpg', '.jpeg', '.png', '.bmp', '.webp', '.tiff'}


def _validate(filename, size):
    # 업로드 파일의 확장자와 크기를 검증하는 헬퍼.
    from pathlib import Path
    ext = Path(filename).suffix.lower()   # 확장자를 소문자로 추출 (.JPG → .jpg)
    if ext not in SUPPORTED_FORMATS:
        raise ValueError(f'지원하지 않는 형식: {ext}')
    if size > 10 * 1024 * 1024:           # 10MB = 10 * 1024 * 1024 바이트
        raise ValueError('10MB 초과 파일은 업로드 불가')


@router.post('/upload', response_model=ClassificationResponse,
             summary='이미지 파일 업로드 후 분류')
async def classify_upload(
    # File(...): multipart/form-data로 업로드되는 필수 파일
    file: UploadFile = File(..., description='분류할 이미지 (JPEG/PNG/WEBP)'),
    top_k: int = Form(5, ge=1, le=20),    # 폼 필드. 반환할 상위 예측 개수(1~20)
    model_key: str = Form('vit-base'),    # 사용할 모델 키(폼 필드)
):
    content = await file.read()   # 업로드 파일 전체를 바이트로 읽음(비동기)
    try:
        _validate(file.filename, len(content))   # 형식·크기 검증
    except ValueError as e:
        raise HTTPException(422, str(e))   # 검증 실패 → 422 Unprocessable Entity
    # 요청한 모델이 현재 로드된 모델과 다르면 새로 로드(불필요한 재로드 방지)
    if model_key != classification_service.model_key:
        try:
            classification_service.load_model(model_key)
        except Exception as e:
            raise HTTPException(400, f'모델 로드 실패: {e}')
    try:
        result = classification_service.classify_from_file(content, top_k=top_k)  # 실제 분류 수행
    except Exception as e:
        raise HTTPException(500, f'분류 오류: {e}')   # 추론 중 오류 → 500
    # 결과 dict를 응답 스키마로 변환. **result로 dict 키들을 인자로 펼친다.
    return ClassificationResponse(filename=file.filename, **result)


@router.post('/base64', response_model=ClassificationResponse,
             summary='Base64 이미지 분류')
async def classify_base64(request: Base64ClassifyRequest):
    # JSON 본문(Base64ClassifyRequest)으로 이미지를 받는다.
    try:
        result = classification_service.classify_from_base64(
            request.image_base64, top_k=request.top_k
        )
    except Exception as e:
        raise HTTPException(500, f'Base64 분류 오류: {e}')
    return ClassificationResponse(filename='base64_image', **result)


@router.post('/url', response_model=ClassificationResponse,
             summary='URL 이미지 분류')
async def classify_url(url: str, top_k: int = 5):
    # 쿼리 파라미터로 받은 이미지 URL을 분류한다.
    try:
        result = classification_service.classify_from_url(url, top_k=top_k)
    except Exception as e:
        raise HTTPException(500, f'URL 분류 오류: {e}')
    return ClassificationResponse(filename=url, **result)


@router.get('/models', summary='지원 모델 목록')
async def list_models():
    # 현재 지원 모델 목록과 로드 상태를 조회하는 GET 엔드포인트.

        'models': classification_service.SUPPORTED_MODELS,   # 키→모델ID 매핑

    }
```

## 6.2 객체 탐지 API 만들기 (YOLOv8)

객체 탐지(Object Detection)는 이미지 내에서 여러 객체의 위치(Bounding Box)와 클래스(종류)를 동시에 예측하는 태스크입니다. 단순히 '이미지에 무엇이 있는가'를 넘어 '어디에 있는가'까지 알려주는 더 복잡한 비전 태스크입니다.

### 6.2.1 객체 탐지란? 이미지 속 객체 위치(Box) + 클래스 예측

객체 탐지 결과는 탐지된 각 객체에 대해 바운딩 박스 좌표 [x1, y1, x2, y2], 클래스 이름, 신뢰도 점수를 함께 반환합니다. IoU(Intersection over Union)와 NMS(Non-Maximum Suppression)가 핵심 알고리즘입니다.

| 탐지 방식 | 대표 모델 | 특징 |
| --- | --- | --- |
| 1-Stage Detector | YOLO, SSD, RetinaNet | 단일 네트워크로 박스+클래스 동시 예측. 빠름. 실시간 처리 적합. |
| 2-Stage Detector | Faster R-CNN, Mask R-CNN | 후보 영역 추출 후 분류. 정확도 높음. 느림. |
| Anchor-Free | FCOS, CenterNet | 앵커 박스 없이 키포인트 기반. 최신 트렌드. |
| Transformer 기반 | DETR, Deformable DETR | End-to-End 탐지. NMS 불필요. |

### 6.2.2 YOLOv8 개요 및 설치 (ultralytics 라이브러리 사용)

YOLOv8은 Ultralytics사가 2023년 발표한 최신 YOLO 시리즈 모델입니다. 객체 탐지뿐만 아니라 세그멘테이션, 포즈 추정, 분류, OBB 등 다양한 CV 태스크를 단일 프레임워크로 지원합니다.

| 모델 | 파라미터 | mAP50-95 | 속도 및 적합 환경 |
| --- | --- | --- | --- |
| YOLOv8n | 3.2M | 37.3 | 초고속(80+FPS). 모바일/엣지 기기용. |
| YOLOv8s | 11.2M | 44.9 | 빠름. 경량 서버/API에 적합. |
| YOLOv8m | 25.9M | 50.2 | 속도-정확도 균형. API 서비스 권장. |
| YOLOv8l | 43.7M | 52.9 | 높은 정확도. GPU 권장. |
| YOLOv8x | 68.2M | 53.9 | 최고 정확도. 고사양 GPU 필요. |

### 6.2.3 model.predict(source=image)로 이미지 추론

#### services/detection_service.py

```python
# services/detection_service.py
# YOLOv8 객체 탐지 비즈니스 로직을 담는 서비스 계층.
import io, time
from PIL import Image, ImageDraw   # ImageDraw: 이미지 위에 사각형/텍스트를 그릴 때 사용
from typing import List, Dict, Optional

# ultralytics가 설치되지 않은 환경에서도 모듈 임포트는 깨지지 않도록 try/except로 감싼다.
try:
    from ultralytics import YOLO
    YOLO_AVAILABLE = True    # 설치돼 있으면 사용 가능 플래그 True
except ImportError:
    YOLO_AVAILABLE = False   # 미설치면 False (실제 호출 시점에 오류를 낸다)

# 클래스별 바운딩 박스 색상(RGB) 팔레트. 클래스 ID로 색을 순환 선택한다.
COCO_COLORS = [
    (255, 0, 0), (0, 255, 0), (0, 0, 255), (255, 255, 0),
    (255, 0, 255), (0, 255, 255), (128, 0, 0), (0, 128, 0),
    (0, 0, 128), (128, 128, 0), (128, 0, 128), (0, 128, 128),
]


class YOLODetectionService:
    # 키 → YOLOv8 가중치 파일명 매핑. n/s/m/l 순으로 무겁고 정확해진다.
    SUPPORTED_MODELS = {
        'yolov8n': 'yolov8n.pt',   # nano: 가장 빠름
        'yolov8s': 'yolov8s.pt',   # small
        'yolov8m': 'yolov8m.pt',   # medium: 속도-정확도 균형
        'yolov8l': 'yolov8l.pt',   # large: 정확도 높음
    }

    def __init__(self, model_key: str = 'yolov8n'):
        self.model_key = model_key   # 현재 모델 키
        self.model = None            # YOLO 모델 객체(로드 전 None)

    def load_model(self, model_key: Optional[str] = None):
        if not YOLO_AVAILABLE:
            raise RuntimeError('ultralytics 패키지 미설치')   # 설치 안 됐으면 즉시 중단
        key = model_key or self.model_key
        model_file = self.SUPPORTED_MODELS.get(key, 'yolov8n.pt')  # 잘못된 키면 nano로 대체
        print(f'[Detection] Loading YOLOv8: {model_file}')
        self.model = YOLO(model_file)   # 가중치 로드(없으면 자동 다운로드)
        self.model_key = key

    def detect(
        self, image_bytes: bytes,
        confidence: float = 0.25,    # 이 값보다 낮은 신뢰도의 탐지는 버린다(낮출수록 더 많이 탐지)
        iou_threshold: float = 0.7,  # NMS에서 겹치는 박스를 제거하는 기준(IoU)
        max_det: int = 300,          # 한 이미지에서 최대 탐지 개수
    ) -> Dict:
        if self.model is None:
            raise RuntimeError('탐지 모델 미로드')
        start = time.time()
        pil_image = Image.open(io.BytesIO(image_bytes)).convert('RGB')  # 바이트→RGB 이미지
        img_w, img_h = pil_image.size   # 정규화 좌표 계산을 위해 원본 크기 저장
        # 핵심: YOLOv8 추론 실행. source에 PIL 이미지를 직접 넘긴다. verbose=False로 로그 억제.
        results = self.model.predict(
            source=pil_image, conf=confidence,
            iou=iou_threshold, max_det=max_det, verbose=False,
        )
        elapsed = round(time.time() - start, 3)
        detections = []
        result = results[0]   # 이미지 1장이므로 첫 번째 결과만 사용
        # 탐지된 박스가 하나라도 있으면 각 박스를 순회하며 정보를 추출한다.
        if result.boxes is not None and len(result.boxes) > 0:
            boxes = result.boxes
            for i in range(len(boxes)):
                x1, y1, x2, y2 = boxes.xyxy[i].tolist()   # 박스 좌표(좌상단·우하단)
                conf = float(boxes.conf[i])               # 신뢰도
                cls_id = int(boxes.cls[i])                # 클래스 ID(정수)
                cls_name = self.model.names[cls_id]       # ID → 클래스 이름(예: person)
                detections.append({
                    'id': i + 1,
                    'class_id': cls_id,
                    'class_name': cls_name,
                    'confidence': round(conf, 4),
                    # 픽셀 단위 박스 + 너비/높이
                    'bbox': {'x1': round(x1), 'y1': round(y1),
                             'x2': round(x2), 'y2': round(y2),
                             'width': round(x2 - x1), 'height': round(y2 - y1)},
                    # 0~1 정규화 좌표(이미지 크기에 독립적이라 프런트엔드 렌더링에 유용)
                    'bbox_normalized': {
                        'x1': round(x1 / img_w, 4), 'y1': round(y1 / img_h, 4),
                        'x2': round(x2 / img_w, 4), 'y2': round(y2 / img_h, 4),
                    },
                })
        # 클래스별 탐지 개수 집계 (예: {'person': 3, 'car': 2})
        class_counts = {}
        for d in detections:
            class_counts[d['class_name']] = class_counts.get(d['class_name'], 0) + 1
        return {
            'detections': detections,            # 탐지 목록
            'total_objects': len(detections),    # 총 객체 수
            'class_counts': class_counts,        # 클래스별 개수
            'image_size': f'{img_w}x{img_h}',
            'model_used': self.model_key,
            'inference_time_sec': elapsed,
        }

    def draw_boxes(self, image_bytes: bytes, detections: List[Dict]) -> bytes:
        # 탐지 결과를 받아 원본 이미지에 박스와 라벨을 그려 JPEG 바이트로 반환한다.
        pil_image = Image.open(io.BytesIO(image_bytes)).convert('RGB')
        draw = ImageDraw.Draw(pil_image)   # 그리기 객체 생성
        for det in detections:
            bbox = det['bbox']
            x1, y1, x2, y2 = bbox['x1'], bbox['y1'], bbox['x2'], bbox['y2']
            color = COCO_COLORS[det['class_id'] % len(COCO_COLORS)]   # 클래스별 색 선택(순환)
            # 두께 2px 박스: 1px 사각형을 안쪽으로 살짝 줄여 두 번 그린다.
            for t in range(2):
                draw.rectangle([x1 + t, y1 + t, x2 - t, y2 - t], outline=color)
            label = f"{det['class_name']} {det['confidence']:.2f}"   # 라벨 텍스트(이름+신뢰도)
            tb = draw.textbbox((x1, y1), label)   # 텍스트가 차지할 영역 계산
            tw, th = tb[2] - tb[0], tb[3] - tb[1]  # 텍스트 너비/높이
            # 라벨 배경(박스 색)을 채워 글자가 잘 보이게 한다.
            draw.rectangle([x1, y1 - th - 4, x1 + tw + 4, y1], fill=color)
            draw.text((x1 + 2, y1 - th - 2), label, fill=(255, 255, 255))   # 흰색 라벨 텍스트
        buf = io.BytesIO()
        pil_image.save(buf, format='JPEG', quality=90)   # 메모리 버퍼에 JPEG로 저장
        return buf.getvalue()   # 바이트 반환


# 앱 전역 공유 싱글톤 (기본 yolov8n)
detection_service = YOLODetectionService(model_key='yolov8n')
```

```python
# routers/detection.py
# 객체 탐지 HTTP 엔드포인트를 정의하는 라우터.
import base64
from fastapi import APIRouter, UploadFile, File, Form, HTTPException
from fastapi.responses import Response   # 이미지 등 비-JSON 응답을 직접 반환할 때 사용
from pathlib import Path
from typing import Optional, List
from pydantic import BaseModel
from services.detection_service import detection_service

# 이 라우터의 모든 경로 앞에 /detect가 붙는다.
router = APIRouter(prefix='/detect', tags=['Object Detection (YOLOv8)'])


class BoundingBox(BaseModel):
    # 바운딩 박스 스키마(픽셀 좌표 + 너비/높이). 세미콜론은 한 줄에 여러 필드를 쓴 것.
    x1: int; y1: int; x2: int; y2: int
    width: int; height: int


class DetectionItem(BaseModel):
    # 탐지 1건의 응답 구조
    id: int
    class_id: int
    class_name: str
    confidence: float
    bbox: BoundingBox


class DetectionResponse(BaseModel):
    # 탐지 API 전체 응답 구조
    filename: str
    detections: List[DetectionItem]
    total_objects: int
    class_counts: dict
    image_size: str
    model_used: str
    inference_time_sec: float
    visualization_base64: Optional[str] = None   # 선택: 박스 그린 이미지(Base64)


@router.post('/json', response_model=DetectionResponse,
             summary='객체 탐지 JSON 결과 반환')
async def detect_json(
    file: UploadFile = File(...),
    confidence: float = Form(0.25, ge=0.01, le=0.99),     # 신뢰도 임계값
    iou_threshold: float = Form(0.7, ge=0.1, le=0.99),    # NMS IoU 임계값
    max_detections: int = Form(100, ge=1, le=500),        # 최대 탐지 개수
    include_visualization: bool = Form(False),            # True면 시각화 이미지도 함께 반환
    model_key: str = Form('yolov8n'),
):
    content = await file.read()   # 파일 바이트 읽기
    ext = Path(file.filename).suffix.lower()
    if ext not in {'.jpg', '.jpeg', '.png', '.bmp', '.webp'}:
        raise HTTPException(422, f'지원하지 않는 형식: {ext}')   # 형식 검증

    if model_key != detection_service.model_key:
        detection_service.load_model(model_key)   # 모델 변경 시에만 재로드

    try:
        result = detection_service.detect(
            content, confidence=confidence,
            iou_threshold=iou_threshold, max_det=max_detections
        )   # 탐지 수행
    except Exception as e:
        raise HTTPException(500, f'탐지 오류: {e}')

    viz_b64 = None
    # 시각화 요청이 있고 탐지된 객체가 있을 때만 박스 이미지를 생성한다.
    if include_visualization and result['detections']:
        viz_bytes = detection_service.draw_boxes(content, result['detections'])
        # 이미지 바이트를 Base64 data URI로 변환 → JSON에 문자열로 담는다
        viz_b64 = 'data:image/jpeg;base64,' + base64.b64encode(viz_bytes).decode()

    return DetectionResponse(
        filename=file.filename,
        visualization_base64=viz_b64,
        **result
    )


@router.post('/visualize', summary='객체 탐지 시각화 이미지 반환',
             response_class=Response,                          # JSON이 아닌 원시 응답으로 반환
             responses={200: {'content': {'image/jpeg': {}}}}) # Swagger에 응답 타입 명시
async def detect_visualize(
    file: UploadFile = File(...),
    confidence: float = Form(0.25, ge=0.01, le=0.99),
):
    content = await file.read()
    result = detection_service.detect(content, confidence=confidence)   # 탐지
    if not result['detections']:
        # 탐지된 게 없으면 원본 이미지를 그대로 반환
        return Response(content=content, media_type='image/jpeg')
    viz_bytes = detection_service.draw_boxes(content, result['detections'])   # 박스 그리기
    return Response(
        content=viz_bytes, media_type='image/jpeg',   # JPEG 이미지로 응답
        headers={
            # 커스텀 헤더로 메타데이터 전달(프런트엔드에서 읽어 표시)
            'X-Total-Objects': str(result['total_objects']),
            'X-Inference-Time': str(result['inference_time_sec']),
        },
    )


@router.get('/classes', summary='탐지 가능한 COCO 클래스 목록')
async def get_classes():
    # 현재 모델이 탐지 가능한 클래스 이름 목록을 반환한다.
    if detection_service.model is None:
        raise HTTPException(503, '모델 미로드')   # 모델 없으면 503 Service Unavailable
    return {'classes': detection_service.model.names,           # {id: name} 매핑
            'total': len(detection_service.model.names)}        # 클래스 총 개수(보통 80)
```

### 6.3.1 이미지 캡셔닝(Image Captioning)이란?

이미지 캡셔닝은 컴퓨터 비전과 자연어 처리가 결합된 멀티모달 태스크입니다. Vision Encoder로 이미지를 특징 벡터로 변환하고, 크로스 어텐션을 통해 언어 모델이 설명 문장을 자동 회귀 방식으로 생성합니다.

| 모델 | 아키텍처 | 특징 및 활용 |
| --- | --- | --- |
| BLIP | ViT + BERT 계열 | 이미지 캡셔닝, VQA, 검색. 경량. Salesforce 개발. |
| BLIP-2 | ViT + Q-Former + LLM | LLM 연결로 강력한 추론. GPU 권장. 고품질. |
| GIT (GenerativeIT) | ViT + GPT-2 | Microsoft 개발. 이미지 캡셔닝에 특화. |
| InstructBLIP | BLIP-2 + Instruction | 지시 기반 다양한 시각 추론. |
| LLaVA | CLIP + LLaMA/Mistral | 강력한 오픈소스 멀티모달 LLM. |

### 6.3.2 Salesforce/blip-image-captioning-base 모델 개요

BLIP(Bootstrapping Language-Image Pre-training)은 Salesforce Research가 2022년 발표한 멀티모달 모델입니다. 이미지-텍스트 쌍을 필터링하고 캡션 생성으로 품질을 향상시키는 Bootstrapping 전략이 핵심입니다.

| BLIP 모델 변형 | 모델 크기 및 특징 |
| --- | --- |
| Salesforce/blip-image-captioning-base | 약 250MB. 기본 캡셔닝. CPU에서도 동작 가능. |
| Salesforce/blip-image-captioning-large | 약 900MB. 더 풍부한 캡션. GPU 권장. |
| Salesforce/blip-vqa-base | Visual Question Answering용 파인튜닝 버전. |
| Salesforce/blip2-opt-2.7b | BLIP-2 + OPT 2.7B. 강력한 추론. 큰 메모리 필요. |

```python
# services/captioning_service.py
# BLIP 기반 이미지 캡셔닝 + 시각 질의응답(VQA) 비즈니스 로직.
import io, time, torch
from PIL import Image
from typing import Optional, Dict
from transformers import (
    BlipProcessor,                    # 이미지+텍스트를 모델 입력 텐서로 변환/디코딩하는 전처리기
    BlipForConditionalGeneration,     # 이미지→캡션 생성 모델
    BlipForQuestionAnswering,         # 이미지+질문→답변 생성 모델(VQA)
)


class BLIPCaptioningService:
    # 캡셔닝 모델 키 → Hugging Face 모델 ID
    CAPTIONING_MODELS = {
        'base': 'Salesforce/blip-image-captioning-base',    # 경량, CPU 가능
        'large': 'Salesforce/blip-image-captioning-large',  # 고품질, GPU 권장
    }
    VQA_MODEL = 'Salesforce/blip-vqa-base'   # 시각 질의응답 전용 모델

    def __init__(self, model_key: str = 'base'):
        self.model_key = model_key
        # 캡셔닝/VQA용 프로세서와 모델을 각각 보관(로드 전 None)
        self.processor: Optional[BlipProcessor] = None
        self.caption_model: Optional[BlipForConditionalGeneration] = None
        self.vqa_processor: Optional[BlipProcessor] = None
        self.vqa_model: Optional[BlipForQuestionAnswering] = None
        # GPU 사용 가능하면 cuda, 아니면 cpu
        self.device = torch.device('cuda' if torch.cuda.is_available() else 'cpu')

    def load_captioning_model(self, model_key: Optional[str] = None):
        # 캡셔닝 프로세서와 모델을 메모리에 적재한다.
        key = model_key or self.model_key
        model_id = self.CAPTIONING_MODELS.get(key, self.CAPTIONING_MODELS['base'])
        print(f'[BLIP] Loading captioning model: {model_id}')
        self.processor = BlipProcessor.from_pretrained(model_id)   # 전처리기 로드
        self.caption_model = BlipForConditionalGeneration.from_pretrained(
            model_id,
            # GPU면 float16(메모리 절약·속도↑), CPU면 float32(호환성)
            torch_dtype=torch.float16 if self.device.type == 'cuda' else torch.float32,
        ).to(self.device)   # 모델을 선택된 장치로 이동
        self.caption_model.eval()   # 추론 모드(드롭아웃 등 비활성화)
        self.model_key = key

    def load_vqa_model(self):
        # 시각 질의응답(VQA) 모델 로드.
        print(f'[BLIP] Loading VQA model: {self.VQA_MODEL}')
        self.vqa_processor = BlipProcessor.from_pretrained(self.VQA_MODEL)
        self.vqa_model = BlipForQuestionAnswering.from_pretrained(
            self.VQA_MODEL
        ).to(self.device)
        self.vqa_model.eval()

    def generate_caption(
        self, image_bytes: bytes,
        conditional_text: Optional[str] = None,   # 캡션을 특정 방향으로 유도하는 시작 문구
        max_new_tokens: int = 50,                 # 생성할 최대 토큰(단어 조각) 수
        num_beams: int = 5,                        # 빔 서치 폭(클수록 품질↑·속도↓)
        num_captions: int = 1,                     # 반환할 캡션 개수
    ) -> Dict:
        """
        이미지에 대한 자연어 설명을 생성합니다.
        conditional_text: 생성 방향을 유도하는 텍스트 (예: 'a photo of')
        """
        if self.caption_model is None:
            raise RuntimeError('캡셔닝 모델이 로드되지 않았습니다.')
        start = time.time()
        image = Image.open(io.BytesIO(image_bytes)).convert('RGB')   # 바이트→RGB 이미지

        if conditional_text:
            # 조건부 캡션: 이미지와 함께 시작 텍스트를 넣어 그 뒤를 이어 생성하게 한다.
            inputs = self.processor(
                images=image, text=conditional_text, return_tensors='pt',  # pt=PyTorch 텐서
            ).to(self.device)
        else:
            # 무조건부 캡션: 이미지만 입력하면 모델이 스스로 설명을 생성한다.
            inputs = self.processor(
                images=image, return_tensors='pt',
            ).to(self.device)

        # torch.no_grad(): 추론이므로 기울기 계산을 끄고 메모리·속도를 절약한다.
        with torch.no_grad():
            output_ids = self.caption_model.generate(   # 토큰 ID 시퀀스 생성
                **inputs,
                max_new_tokens=max_new_tokens,
                num_beams=num_beams,
                # 빔 수를 초과하는 캡션은 만들 수 없으므로 둘 중 작은 값으로 제한
                num_return_sequences=min(num_captions, num_beams),
                early_stopping=True,        # 모든 빔이 끝나면 조기 종료
                no_repeat_ngram_size=3,     # 같은 3-gram 반복 방지(어색한 반복 억제)
            )

        # 생성된 토큰 ID를 사람이 읽는 문자열로 디코딩(특수 토큰 제거)
        captions = self.processor.batch_decode(output_ids, skip_special_tokens=True)
        elapsed = round(time.time() - start, 3)
        return {
            'captions': captions,                  # 생성된 캡션 목록
            'primary_caption': captions[0],        # 대표(1순위) 캡션
            'conditional_text': conditional_text,  # 사용한 유도 텍스트
            'image_size': f'{image.width}x{image.height}',
            'model_used': self.CAPTIONING_MODELS[self.model_key],
            'inference_time_sec': elapsed,
        }

    def answer_question(self, image_bytes: bytes, question: str) -> Dict:
        # 이미지와 질문을 받아 답변을 생성하는 VQA 메서드.
        if self.vqa_model is None:
            raise RuntimeError('VQA 모델이 로드되지 않았습니다.')
        start = time.time()
        image = Image.open(io.BytesIO(image_bytes)).convert('RGB')
        # 이미지 + 질문 텍스트를 함께 전처리해 입력 텐서로 만든다.
        inputs = self.vqa_processor(
            images=image, text=question, return_tensors='pt',
        ).to(self.device)
        with torch.no_grad():
            out = self.vqa_model.generate(**inputs, max_new_tokens=50)   # 답변 토큰 생성
        answer = self.vqa_processor.decode(out[0], skip_special_tokens=True)  # 첫 결과 디코딩
        return {'question': question, 'answer': answer,
                'inference_time_sec': round(time.time() - start, 3)}


# 앱 전역 공유 싱글톤 (기본 base 모델)
captioning_service = BLIPCaptioningService(model_key='base')
```

```python
# routers/captioning.py
# 이미지 캡셔닝/VQA HTTP 엔드포인트를 정의하는 라우터.
from fastapi import APIRouter, UploadFile, File, Form, HTTPException
from typing import Optional, List
from pydantic import BaseModel
from services.captioning_service import captioning_service

# 모든 경로 앞에 /caption이 붙는다.
router = APIRouter(prefix='/caption', tags=['Image Captioning (BLIP)'])


class CaptionResponse(BaseModel):
    # 캡션 생성 응답 스키마
    filename: str
    captions: List[str]            # 생성된 캡션 목록
    primary_caption: str           # 대표 캡션
    conditional_text: Optional[str]
    image_size: str
    model_used: str
    inference_time_sec: float


class VQAResponse(BaseModel):
    # 시각 질의응답 응답 스키마
    filename: str
    question: str
    answer: str
    inference_time_sec: float


@router.post('/generate', response_model=CaptionResponse,
             summary='이미지 캡션 생성')
async def generate_caption(
    file: UploadFile = File(...),
    conditional_text: Optional[str] = Form(None,                # 선택적 유도 텍스트
        description='캡션 유도 텍스트 (예: a photo of)'),
    max_new_tokens: int = Form(50, ge=10, le=200),             # 생성 길이 한도
    num_beams: int = Form(5, ge=1, le=10),                      # 빔 서치 폭
    num_captions: int = Form(1, ge=1, le=5),                    # 반환 캡션 수
    model_key: str = Form('base'),
):
    content = await file.read()
    if model_key != captioning_service.model_key:
        captioning_service.load_captioning_model(model_key)   # 모델 변경 시 재로드
    try:
        result = captioning_service.generate_caption(
            content, conditional_text=conditional_text,
            max_new_tokens=max_new_tokens, num_beams=num_beams,
            num_captions=num_captions,
        )   # 캡션 생성 실행
    except Exception as e:
        raise HTTPException(500, f'캡셔닝 오류: {e}')
    return CaptionResponse(filename=file.filename, **result)


@router.post('/vqa', response_model=VQAResponse,
             summary='이미지 질의응답 (VQA)')
async def visual_qa(
    file: UploadFile = File(...),
    question: str = Form(..., description='이미지에 대한 질문 (영어 권장)'),  # 필수 질문 텍스트
):
    content = await file.read()
    if captioning_service.vqa_model is None:
        raise HTTPException(503, 'VQA 모델 미로드')   # VQA 모델 미로드 시 503
    try:
        result = captioning_service.answer_question(content, question)   # 답변 생성
    except Exception as e:
        raise HTTPException(500, f'VQA 오류: {e}')
    return VQAResponse(filename=file.filename, **result)
```

```html
// 파일 선택 시 미리보기 표시
document.getElementById('imageFile').addEventListener('change', e => {
    const file = e.target.files[0];   // 선택된 첫 파일
    if (!file) return;
    const reader = new FileReader();  // 파일을 읽는 브라우저 API
    reader.onload = ev => {
        // 읽기가 끝나면 data URL을 img src에 넣고 미리보기 표시
        document.getElementById('previewImg').src = ev.target.result;
        document.getElementById('previewSection').style.display = 'block';
    };
    reader.readAsDataURL(file);   // 파일을 data URL(base64)로 읽기 시작
});
// 폼 제출 시 비동기로 분류 API 호출
document.getElementById('classifyForm').addEventListener('submit', async e => {
    e.preventDefault();   // 폼 기본 제출(페이지 새로고침) 방지
    const loading = document.getElementById('loading');
    const resultSection = document.getElementById('resultSection');
    const errorSection = document.getElementById('errorSection');
    loading.style.display = 'inline';     // 로딩 표시 ON
    resultSection.style.display = 'none'; // 이전 결과 숨김
    errorSection.style.display = 'none';  // 이전 오류 숨김

    // FormData로 파일과 옵션을 multipart 형식으로 담는다
    const fd = new FormData();
    fd.append('file', document.getElementById('imageFile').files[0]);
    fd.append('model_key', document.getElementById('modelKey').value);
    fd.append('top_k', document.getElementById('topK').value);
    try {
        // 분류 API 호출(POST)
        const resp = await fetch('/classify/upload', { method:'POST', body:fd });
        loading.style.display = 'none';   // 로딩 표시 OFF
        if (!resp.ok) {
            // HTTP 오류면 detail 메시지를 표시하고 종료
            const err = await resp.json();
            errorSection.textContent = '오류: ' + JSON.stringify(err.detail);
            errorSection.style.display = 'block'; return;
        }
        const data = await resp.json();   // 응답 JSON 파싱
        // 최고 예측과 신뢰도(%) 표시
        document.getElementById('topResult').innerHTML =
            `<strong>최고 예측: ${data.top_prediction}</strong>` +
            ` (신뢰도: ${(data.top_score*100).toFixed(1)}%)`;
        // 예측 목록을 순회하며 라벨+신뢰도 막대를 HTML로 생성
        let html = '';
        data.predictions.forEach(p => {
            const pct = (p.score*100).toFixed(2);   // 백분율 문자열
            html += `<div style="margin-bottom:0.6rem;">
                <div style="display:flex;justify-content:space-between;">
                    <span>${p.rank}. ${p.label}</span><span>${pct}%</span>
                </div>
                <div class="progress-bar">
                    <div class="progress-fill" style="width:${pct}%"></div>
                </div></div>`;   // width를 신뢰도에 비례시켜 막대 길이 표현
        });
        document.getElementById('predBars').innerHTML = html;   // 막대 차트 삽입
        // 모델/추론시간/크기 등 메타 정보 표시
        document.getElementById('metaInfo').textContent =
            `모델: ${data.model_used} | 추론: ${data.inference_time_sec}s | 크기: ${data.image_size}`;
        resultSection.style.display = 'block';   // 결과 영역 표시
    } catch(err) {
        // 네트워크 등 예외 처리
        loading.style.display = 'none';
        errorSection.textContent = '오류: ' + err.message;
        errorSection.style.display = 'block';
    }
});
```
