# HTML 문서 구조와 핵심 태그

HTML(HyperText Markup Language)은 웹 페이지의 뼈대를 만드는 마크업 언어입니다. 이 장에서는 HTML 문서가 어떤 구조로 이루어지는지, 그리고 텍스트·링크·이미지·리스트·테이블·폼 같은 핵심 태그를 어떻게 쓰는지 단계별로 배웁니다. 코드 예제마다 한 줄씩 한국어 주석을 달았으니, 직접 따라 작성하면서 의미를 익혀 보세요.

## 1.1 HTML이란? 문서의 기본 구조

### 개념: HTML은 "구조"를 담당한다

웹 페이지는 보통 세 가지 기술의 조합으로 만들어집니다.

- **HTML**: 문서의 구조와 내용(제목, 문단, 이미지, 링크 등)
- **CSS**: 보이는 모양(색, 글꼴, 배치 등)
- **JavaScript**: 동작과 상호작용(버튼 클릭, 데이터 처리 등)

HTML은 이 중에서 **구조와 의미**를 담당합니다. 글자에 "이건 제목이다", "이건 문단이다", "이건 목록이다" 같은 의미를 부여하는 것이 HTML의 핵심입니다.

### 태그와 요소(element) 이해하기

HTML은 **태그(tag)** 로 내용을 감싸서 의미를 표현합니다.

- 여는 태그: `<p>`
- 닫는 태그: `</p>`
- 여는 태그 + 내용 + 닫는 태그 = **요소(element)**
- 태그 안의 추가 정보는 **속성(attribute)**: `<a href="...">`에서 `href`가 속성

```html
<!-- p 요소: 하나의 문단을 의미한다 -->
<!-- <p> 는 여는 태그, </p> 는 닫는 태그 -->
<p>안녕하세요, 첫 번째 문단입니다.</p>

<!-- a 요소: href 속성으로 연결할 주소를 지정한다 -->
<!-- href 가 '속성 이름', "..." 안의 값이 '속성 값' -->
<a href="https://example.com">예제 사이트로 이동</a>
```

### HTML 문서의 기본 골격

모든 HTML 문서는 정해진 기본 구조를 가집니다. 아래가 가장 표준적인 HTML5 문서의 뼈대입니다.

```html
<!-- DOCTYPE: 이 문서가 HTML5 문서임을 브라우저에 알리는 선언 -->
<!-- 반드시 문서 맨 위 첫 줄에 위치해야 한다 -->
<!DOCTYPE html>

<!-- html: 문서 전체를 감싸는 최상위(루트) 요소 -->
<!-- lang="ko" 는 문서의 주 언어가 한국어임을 의미(접근성·검색에 중요) -->
<html lang="ko">

  <!-- head: 화면에 직접 보이지 않는 '문서 정보(메타데이터)' 영역 -->
  <head>
    <!-- charset: 문자 인코딩 지정. 한글이 깨지지 않으려면 UTF-8 필수 -->
    <meta charset="UTF-8" />

    <!-- viewport: 모바일에서 화면 너비에 맞춰 반응형으로 보이게 하는 설정 -->
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />

    <!-- title: 브라우저 탭에 표시되는 문서 제목 -->
    <title>나의 첫 HTML 페이지</title>
  </head>

  <!-- body: 실제 화면에 보이는 모든 내용이 들어가는 영역 -->
  <body>
    <h1>환영합니다</h1>
    <p>이 글은 브라우저 화면에 표시됩니다.</p>
  </body>

</html>
```

### 각 부분이 하는 일

| 부분 | 역할 |
| --- | --- |
| `<!DOCTYPE html>` | HTML5 문서임을 선언(맨 위 한 줄) |
| `<html lang="ko">` | 문서 전체를 감싸는 루트, 언어 지정 |
| `<head>` | 화면에 안 보이는 메타데이터(제목, 인코딩, CSS 링크 등) |
| `<meta charset="UTF-8">` | 한글 등 문자가 깨지지 않게 인코딩 지정 |
| `<meta name="viewport">` | 모바일 반응형 표시 설정 |
| `<title>` | 브라우저 탭 제목 |
| `<body>` | 실제 화면에 보이는 내용 |

### 왜 이 구조를 지켜야 할까?

- 브라우저가 문서를 **정확하게 해석**하도록 돕습니다.
- `DOCTYPE`가 없으면 브라우저가 옛날 방식(쿼크 모드)으로 동작해 레이아웃이 깨질 수 있습니다.
- `charset`이 없거나 잘못되면 **한글이 깨져서** 표시됩니다.
- `lang` 속성은 스크린 리더(시각장애인 보조기기)와 검색엔진에 도움을 줍니다.

### 자주 하는 실수

- `DOCTYPE` 선언을 빠뜨림 → 일부 CSS가 의도와 다르게 동작
- `charset`을 `<head>` 안에 안 넣거나 `UTF-8`이 아님 → 한글 깨짐(``)
- 파일을 저장할 때 인코딩 자체를 UTF-8이 아닌 다른 것으로 저장
- 태그를 닫지 않음(`<p>`만 쓰고 `</p>` 생략) → 구조가 꼬임
- `head`에 들어갈 내용과 `body`에 들어갈 내용을 헷갈림

## 1.2 텍스트 태그와 시맨틱 마크업

### 개념: 글에도 "의미"를 부여한다

화면에 글자를 그냥 크게 보이게 만드는 것이 목적이 아니라, **이 글이 무엇인지** 의미를 정확히 표현하는 것이 좋은 HTML입니다. 예를 들어 "큰 글씨"가 필요하다고 `<h1>`을 쓰는 게 아니라, **그 내용이 진짜 제목일 때** `<h1>`을 씁니다.

### 핵심 텍스트 태그

```html
<!-- 제목(heading): h1 ~ h6, 숫자가 작을수록 더 중요한(큰) 제목 -->
<!-- 한 페이지에 h1 은 보통 1개만 쓰고, 단계를 건너뛰지 않는 것이 원칙 -->
<h1>가장 큰 제목 (문서의 주제)</h1>
<h2>중간 제목</h2>
<h3>작은 제목</h3>

<!-- p: 하나의 문단(paragraph) -->
<p>여러 문장을 담는 일반적인 본문 문단입니다.</p>

<!-- br: 줄바꿈(닫는 태그가 없는 빈 요소) -->
<!-- 단, 문단 사이 간격 조절용으로 br 을 남발하지 말 것 -->
<p>첫째 줄<br />둘째 줄</p>

<!-- strong: 의미상 중요한 강조(보통 굵게 표시됨) -->
<!-- em: 의미상 강조(보통 기울임으로 표시됨) -->
<p><strong>중요:</strong> 마감일은 <em>오늘</em>까지입니다.</p>
```

### 굵게/기울임: 의미 태그 vs 모양 태그

`<b>`와 `<i>`도 굵게·기울임을 표시하지만, **의미가 없는 시각 효과**입니다. 의미가 있는 강조라면 `<strong>`, `<em>`을 쓰는 것이 권장됩니다.

| 태그 | 의미 | 권장 상황 |
| --- | --- | --- |
| `<strong>` | 내용상 중요함 | 경고, 핵심 단어 |
| `<em>` | 내용상 강조 | 어조 강조 |
| `<b>` | 의미 없는 굵게 | 단순 시각 효과(가급적 CSS 사용) |
| `<i>` | 의미 없는 기울임 | 단순 시각 효과 |

### 시맨틱 마크업(semantic markup)이란?

**시맨틱(semantic)** 은 "의미가 있는"이라는 뜻입니다. `<div>`처럼 의미 없는 상자 대신, **그 영역이 무슨 역할인지 알려주는 태그**를 쓰는 것이 시맨틱 마크업입니다.

왜 중요할까요?

- **접근성**: 스크린 리더가 "여기는 내비게이션", "여기는 본문"이라고 안내할 수 있습니다.
- **검색엔진 최적화(SEO)**: 검색엔진이 페이지 구조를 더 잘 이해합니다.
- **유지보수**: 코드를 읽는 사람이 구조를 한눈에 파악합니다.

| 시맨틱 태그 | 역할 |
| --- | --- |
| `<header>` | 페이지·섹션의 머리말(로고, 제목 영역) |
| `<nav>` | 내비게이션(메뉴 링크 모음) |
| `<main>` | 페이지의 핵심 본문(페이지당 1개) |
| `<section>` | 주제로 묶이는 영역 |
| `<article>` | 독립적으로 배포 가능한 콘텐츠(글, 게시물) |
| `<aside>` | 본문과 구분되는 부가 정보(사이드바) |
| `<footer>` | 바닥글(저작권, 연락처) |
| `<figure>` / `<figcaption>` | 이미지·도표와 그 설명 |

### 시맨틱 구조 예제

```html
<!-- header: 사이트 상단 머리말 영역 -->
<header>
  <h1>나의 블로그</h1>

  <!-- nav: 메뉴 링크들을 묶는 내비게이션 영역 -->
  <nav>
    <a href="/">홈</a>
    <a href="/about">소개</a>
  </nav>
</header>

<!-- main: 이 페이지의 핵심 본문(한 페이지에 하나만) -->
<main>
  <!-- article: 그 자체로 독립적인 글 한 편 -->
  <article>
    <h2>오늘의 글 제목</h2>
    <p>글 본문 내용이 이곳에 들어갑니다.</p>
  </article>

  <!-- aside: 본문과 직접 관련은 적은 부가 정보(사이드바 등) -->
  <aside>
    <h3>관련 링크</h3>
    <p>추천 글 목록 등</p>
  </aside>
</main>

<!-- footer: 페이지 바닥글(저작권, 연락처 등) -->
<footer>
  <p>© 2026 나의 블로그</p>
</footer>
```

### 자주 하는 실수

- 모든 것을 `<div>`로만 감쌈 → 구조가 의미 없이 복잡해짐
- 제목 단계를 건너뜀(`<h1>` 다음 바로 `<h4>`) → 문서 개요가 망가짐
- "글씨를 크게 하려고" 제목 태그를 사용 → 크기 조절은 CSS의 역할
- `<main>`을 한 페이지에 여러 개 사용 → 페이지당 1개가 원칙

## 1.3 링크 · 이미지 · 리스트 · 테이블

### 링크: `<a>` 태그

링크는 다른 페이지나 위치로 이동하게 해 주는 **하이퍼링크**입니다. `href` 속성에 이동할 주소를 적습니다.

```html
<!-- 외부 사이트로 이동하는 절대 경로 링크 -->
<a href="https://www.example.com">예제 사이트</a>

<!-- 같은 사이트 내 다른 페이지로 이동하는 상대 경로 링크 -->
<a href="/about.html">소개 페이지</a>

<!-- target="_blank": 새 탭에서 열기 -->
<!-- rel="noopener noreferrer": 새 탭 보안 취약점을 막는 권장 옵션 -->
<a href="https://www.example.com" target="_blank" rel="noopener noreferrer">
  새 탭에서 열기
</a>

<!-- 같은 페이지 안의 특정 위치(id)로 이동하는 앵커 링크 -->
<a href="#contact">연락처 영역으로 이동</a>

<!-- 이메일 보내기 / 전화 걸기 링크 -->
<a href="mailto:hello@example.com">메일 보내기</a>
<a href="tel:0212345678">전화 걸기</a>
```

### 이미지: `<img>` 태그

이미지는 `<img>`로 넣습니다. 닫는 태그가 없는 **빈 요소**이며, `alt` 속성이 매우 중요합니다.

```html
<!-- src: 이미지 파일 경로(필수) -->
<!-- alt: 이미지를 못 볼 때 대신 보여줄 설명 텍스트(접근성·SEO에 중요) -->
<!-- width/height: 크기 지정(레이아웃 흔들림 방지에 도움) -->
<img
  src="images/cat.jpg"
  alt="창밖을 바라보는 갈색 고양이"
  width="400"
  height="300"
/>

<!-- 장식용 이미지처럼 의미가 없는 경우 alt 는 빈 문자열로 둔다 -->
<img src="images/divider.png" alt="" />
```

`alt`가 왜 중요할까요?

- 시각장애인용 스크린 리더가 이미지를 **소리로 읽어** 줍니다.
- 이미지가 깨졌을 때 **대체 텍스트**가 보입니다.
- 검색엔진이 이미지 내용을 이해합니다.

### 리스트: 목록 만들기

목록은 세 종류가 있습니다.

| 태그 | 의미 | 표시 |
| --- | --- | --- |
| `<ul>` | 순서 없는 목록(unordered) | 점(•) |
| `<ol>` | 순서 있는 목록(ordered) | 번호(1, 2, 3) |
| `<dl>` | 정의 목록(description) | 용어-설명 쌍 |

```html
<!-- ul: 순서가 중요하지 않은 목록. 각 항목은 li 로 감싼다 -->
<ul>
  <li>사과</li>
  <li>바나나</li>
  <li>포도</li>
</ul>

<!-- ol: 순서가 중요한 목록(단계, 순위 등). 자동으로 번호가 매겨진다 -->
<ol>
  <li>물을 끓인다</li>
  <li>면을 넣는다</li>
  <li>3분 후 건진다</li>
</ol>

<!-- dl: 용어(dt)와 그 설명(dd)을 짝지어 보여주는 정의 목록 -->
<dl>
  <dt>HTML</dt>
  <dd>웹 페이지의 구조를 만드는 마크업 언어</dd>

  <dt>CSS</dt>
  <dd>웹 페이지의 스타일을 지정하는 언어</dd>
</dl>
```

리스트는 **중첩**도 가능합니다.

```html
<!-- 목록 안에 또 목록을 넣어 계층 구조를 표현할 수 있다 -->
<ul>
  <li>과일
    <ul>
      <li>사과</li>
      <li>배</li>
    </ul>
  </li>
  <li>채소</li>
</ul>
```

### 테이블: 표 만들기

테이블은 **행과 열로 이루어진 데이터**(숫자 표, 일정표 등)를 표현할 때 씁니다. 레이아웃(화면 배치)용으로 쓰지 않는 것이 원칙입니다.

```html
<!-- table: 표 전체를 감싸는 태그 -->
<table>

  <!-- thead: 표의 머리(제목 행) 영역 -->
  <thead>
    <!-- tr: 표의 한 행(table row) -->
    <tr>
      <!-- th: 제목 칸(table header), 보통 굵게 가운데 정렬됨 -->
      <th>이름</th>
      <th>나이</th>
      <th>도시</th>
    </tr>
  </thead>

  <!-- tbody: 표의 본문(실제 데이터) 영역 -->
  <tbody>
    <tr>
      <!-- td: 일반 데이터 칸(table data) -->
      <td>홍길동</td>
      <td>25</td>
      <td>서울</td>
    </tr>
    <tr>
      <td>김영희</td>
      <td>30</td>
      <td>부산</td>
    </tr>
  </tbody>

</table>
```

칸을 합치고 싶을 때는 병합 속성을 씁니다.

```html
<table>
  <tr>
    <!-- colspan: 가로로 칸을 합친다(여기서는 2칸을 하나로) -->
    <th colspan="2">사용자 정보</th>
  </tr>
  <tr>
    <td>이름</td>
    <td>홍길동</td>
  </tr>
  <tr>
    <!-- rowspan: 세로로 칸을 합친다(여기서는 2행을 하나로) -->
    <td rowspan="2">연락처</td>
    <td>전화</td>
  </tr>
  <tr>
    <td>이메일</td>
  </tr>
</table>
```

### 자주 하는 실수

- `<img>`에 `alt`를 빠뜨림 → 접근성·SEO 손해
- `<li>`를 `<ul>`/`<ol>` 밖에 단독 사용 → 잘못된 구조
- 테이블을 화면 레이아웃 용도로 사용 → 유지보수·접근성 문제(레이아웃은 CSS로)
- 상대 경로/절대 경로를 혼동해 이미지·링크가 깨짐
- `target="_blank"`만 쓰고 `rel="noopener"`를 빠뜨림(보안)

## 1.4 폼(form)과 입력 요소

### 개념: 폼은 "사용자 입력"을 받는다

회원가입, 로그인, 검색, 댓글… 사용자에게 정보를 입력받아 서버로 보내는 모든 것이 **폼(form)** 입니다. 폼은 `<form>` 태그로 감싸고, 그 안에 입력 요소들을 배치합니다.

```html
<!-- form: 입력 요소들을 묶어 서버로 보내는 컨테이너 -->
<!-- action: 데이터를 보낼 주소(서버 URL) -->
<!-- method: 전송 방식. GET(주소에 노출, 검색용) / POST(본문에 담아 전송, 로그인·가입용) -->
<form action="/signup" method="post">
  <!-- 이 안에 입력 요소들이 들어간다 -->
</form>
```

### label과 input은 짝꿍이다

모든 입력 칸에는 **무엇을 입력하는지 알려주는 라벨**이 필요합니다. `<label>`의 `for`와 `<input>`의 `id`를 같은 값으로 연결하면, 라벨을 클릭해도 입력 칸이 활성화되고 접근성도 좋아집니다.

```html
<!-- label 의 for 와 input 의 id 를 같은 값으로 맞춰 서로 연결한다 -->
<label for="username">아이디</label>
<input type="text" id="username" name="username" />

<!--
  name: 서버로 전송될 때 사용되는 '데이터 이름'(필수에 가까움)
  id  : label 연결 및 CSS·JS 선택용 식별자
-->
```

### 다양한 input 타입

`<input>`은 `type` 속성에 따라 완전히 다른 입력 칸이 됩니다. 모바일에서 적절한 키보드가 뜨고, 브라우저가 기본 검증도 해 주므로 **상황에 맞는 type**을 고르는 것이 중요합니다.

| type | 용도 |
| --- | --- |
| `text` | 한 줄 일반 텍스트 |
| `password` | 비밀번호(●●● 가려짐) |
| `email` | 이메일(형식 자동 검증) |
| `number` | 숫자 입력 |
| `tel` | 전화번호 |
| `url` | 웹 주소 |
| `date` | 날짜 선택 |
| `checkbox` | 여러 개 선택 |
| `radio` | 여러 보기 중 하나 선택 |
| `file` | 파일 업로드 |
| `submit` | 폼 제출 버튼 |

```html
<form action="/signup" method="post">

  <!-- 텍스트 입력 -->
  <label for="name">이름</label>
  <input type="text" id="name" name="name" placeholder="홍길동" required />

  <!-- 이메일: 형식(@ 포함)을 브라우저가 검사한다 -->
  <label for="email">이메일</label>
  <input type="email" id="email" name="email" required />

  <!-- 비밀번호: 입력값이 가려진다 -->
  <label for="pw">비밀번호</label>
  <input type="password" id="pw" name="password" required />

  <!-- 숫자: min/max 로 허용 범위를 제한할 수 있다 -->
  <label for="age">나이</label>
  <input type="number" id="age" name="age" min="0" max="120" />

  <!-- 날짜 선택기 -->
  <label for="birth">생년월일</label>
  <input type="date" id="birth" name="birth" />

  <!-- 제출 버튼: 누르면 form 의 action 으로 데이터가 전송된다 -->
  <button type="submit">가입하기</button>

</form>
```

### 자주 쓰는 속성

| 속성 | 의미 |
| --- | --- |
| `placeholder` | 입력 전 보이는 안내 문구(예시) |
| `required` | 반드시 입력해야 제출 가능 |
| `value` | 기본값(미리 채워둘 값) |
| `disabled` | 비활성화(입력 불가) |
| `readonly` | 읽기 전용(수정 불가) |
| `maxlength` | 최대 입력 글자 수 |
| `min` / `max` | 숫자·날짜의 최소·최대값 |

### 체크박스 · 라디오 · 선택 목록

```html
<form action="/order" method="post">

  <!-- checkbox: 여러 개를 동시에 선택할 수 있다 -->
  <p>관심 분야(중복 선택 가능)</p>
  <label><input type="checkbox" name="interest" value="html" /> HTML</label>
  <label><input type="checkbox" name="interest" value="css" /> CSS</label>

  <!-- radio: 같은 name 으로 묶인 보기 중 '하나만' 선택된다 -->
  <p>등급 선택</p>
  <label><input type="radio" name="grade" value="basic" /> 기본</label>
  <label><input type="radio" name="grade" value="pro" /> 프로</label>

  <!-- select: 펼침 목록(드롭다운). option 하나가 보기 하나 -->
  <label for="city">도시</label>
  <select id="city" name="city">
    <option value="">-- 선택하세요 --</option>
    <option value="seoul">서울</option>
    <option value="busan">부산</option>
  </select>

  <!-- textarea: 여러 줄 텍스트 입력칸(input 과 달리 닫는 태그 있음) -->
  <label for="memo">남길 말</label>
  <textarea id="memo" name="memo" rows="4" cols="30"></textarea>

  <button type="submit">주문하기</button>

</form>
```

### GET vs POST 한눈에

| 구분 | GET | POST |
| --- | --- | --- |
| 데이터 위치 | 주소(URL)에 노출 | 요청 본문에 담김 |
| 길이 제한 | 있음 | 사실상 없음 |
| 적합한 상황 | 검색 등 조회 | 로그인·회원가입 등 |
| 민감 정보 | 부적합 | 더 적합 |

### 자주 하는 실수

- `<label>`을 생략하거나 `for`/`id`를 연결하지 않음 → 접근성·사용성 저하
- `<input>`에 `name`을 안 줌 → 서버로 값이 전송되지 않음
- 라디오 버튼의 `name`을 서로 다르게 줌 → 여러 개가 동시에 선택됨(하나만 되게 하려면 name을 같게)
- 비밀번호·로그인을 `method="get"`으로 보냄 → 주소창에 그대로 노출(POST 사용)
- 상황에 안 맞는 type 사용(이메일에 `text`) → 검증·모바일 키보드 이점을 놓침
