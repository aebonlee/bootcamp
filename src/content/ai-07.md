## 7.1 텍스트+이미지 입력 모델 실습 (CLIP)

### (1) CLIP(OpenAI): 이미지와 텍스트를 같은 벡터 공간에 매핑

CLIP(Contrastive Language–Image Pre-Training)은 2021년 OpenAI가 발표한 혁신적인 멀티모달 모델입니다. CLIP은 이미지 인코더(Image Encoder)와 텍스트 인코더(Text Encoder)를 동시에 학습하여, 이미지와 텍스트를 동일한 고차원 벡터 공간(shared embedding space)에 투영합니다. 이를 통해 이미지와 텍스트 간의 의미적 유사성을 직접 비교할 수 있습니다.

핵심 아이디어는 "의미적으로 일치하는 이미지-텍스트 쌍은 벡터 공간에서 가깝게, 관계없는 쌍은 멀리 위치하도록" 대조 학습(Contrastive Learning)을 수행하는 것입니다. 4억 개의 (이미지, 텍스트) 쌍 데이터로 학습되었으며, 이미지 분류, 이미지 검색, 시각적 질의응답 등 다양한 다운스트림 태스크에 활용됩니다.

| 구성 요소 | 구조 | 입력 | 출력 | 역할 |
|---|---|---|---|---|
| Image Encoder | ViT (Vision Transformer) 또는 ResNet | PIL Image (224×224) | 512/768차원 벡터 | 이미지를 의미 벡터로 변환 |
| Text Encoder | GPT-2 유사 Transformer | 텍스트 문자열 (max 77 tokens) | 512/768차원 벡터 | 텍스트를 의미 벡터로 변환 |
| Projection Layer | 선형 변환 레이어 | 각 인코더 출력 | 공통 임베딩 공간 벡터 | 두 벡터를 같은 공간으로 정렬 |

CLIP의 학습 방식인 대조 학습을 좀 더 자세히 살펴봅니다. 배치(batch) 내 N개의 이미지-텍스트 쌍이 있을 때, 대각선에 위치한 N개의 일치 쌍의 코사인 유사도는 최대화하고, 나머지 N²-N개의 불일치 쌍의 유사도는 최소화하도록 학습됩니다.

#### CLIP 대조 학습 (Contrastive Learning) 개념

학습 데이터 예시 (3쌍):

```
이미지 1: 고양이 사진      텍스트 1: "a photo of a cat"
이미지 2: 강아지 사진      텍스트 2: "a cute dog playing"
이미지 3: 자동차 사진      텍스트 3: "a red sports car"
```

학습 목표 (유사도 행렬):

```
        텍스트1 텍스트2 텍스트3
이미지1 → [높음]  [낮음]  [낮음]   ← 대각선 최대화
이미지2 → [낮음]  [높음]  [낮음]
이미지3 → [낮음]  [낮음]  [높음]
```

이렇게 학습하면 CLIP은 의미가 일치하는 이미지-텍스트 쌍을 벡터 공간에서 가까운 위치에 배치하는 능력을 획득합니다.

CLIP이 실용적으로 강력한 이유 중 하나는 Zero-shot 전이 학습(Zero-shot Transfer) 능력입니다. 별도의 파인튜닝 없이도 텍스트 레이블만 작성하면 새로운 분류 태스크를 즉시 수행할 수 있습니다.

| CLIP 활용 태스크 | 방식 | 실용 예시 |
|---|---|---|
| 이미지 분류 | 이미지 벡터 vs. "a photo of [class]" 텍스트 벡터 비교 | 1000개 클래스 분류, Zero-shot ImageNet |
| 시맨틱 이미지 검색 | 텍스트 쿼리 벡터로 이미지 DB를 검색 | "아이와 함께 있는 개" 검색 |
| 이미지 캡셔닝 지원 | 여러 캡션 후보 중 이미지와 가장 유사한 것 선택 | BLIP 등 캡셔닝 모델과 결합 |
| 이미지 생성 가이드 | 생성된 이미지가 프롬프트와 얼마나 일치하는지 점수화 | Stable Diffusion의 CLIP Score 측정 |
| 멀티모달 검색엔진 | 텍스트와 이미지를 동시에 인덱싱 → 크로스모달 검색 | 사진 앱의 자연어 검색 기능 |

### (2) openai/clip-vit-base-patch32 모델 실습 환경 구성

CLIP 모델은 Hugging Face Hub에서 transformers 라이브러리를 통해 간편하게 로드할 수 있습니다. 가장 널리 사용되는 clip-vit-base-patch32는 ViT-B/32 이미지 인코더와 GPT-2 스타일의 텍스트 인코더를 사용하며 512차원 임베딩 벡터를 생성합니다.

| 모델 ID | 이미지 인코더 | 임베딩 차원 | VRAM | 특징 |
|---|---|---|---|---|
| openai/clip-vit-base-patch32 | ViT-B/32 | 512 | ~1.5GB | 가장 빠름, 경량. 실습 기본 모델 |
| openai/clip-vit-base-patch16 | ViT-B/16 | 512 | ~1.5GB | patch32보다 세밀한 이미지 인식 |
| openai/clip-vit-large-patch14 | ViT-L/14 | 768 | ~3.5GB | 고품질, Stable Diffusion에서 사용 |
| laion/CLIP-ViT-H-14-laion2B-s32B-b79K | ViT-H/14 | 1024 | ~8GB | 최고 성능, 대규모 학습 |

```python
# ── 환경 설치 ────────────────────────────────────────────
# pip install transformers torch torchvision Pillow open_clip_torch

# ── 라이브러리 및 모델 로드 ──────────────────────────────
import torch                                          # 딥러닝 연산(텐서)과 GPU 사용을 담당하는 핵심 라이브러리
from transformers import CLIPProcessor, CLIPModel     # 허깅페이스에서 CLIP 전처리기와 모델 클래스를 가져옴
from PIL import Image                                 # 이미지를 열고 변환하는 파이썬 표준 이미지 라이브러리
import requests                                       # 웹에서 이미지를 내려받을 때 사용(HTTP 요청)
from io import BytesIO                                # 바이트 데이터를 파일처럼 다루기 위한 메모리 버퍼

# GPU(cuda)가 사용 가능하면 GPU를, 아니면 CPU를 연산 장치로 선택 → 속도 차이가 큼
device = "cuda" if torch.cuda.is_available() else "cpu"
print(f"실행 장치: {device}")

# 사용할 CLIP 모델의 허깅페이스 Hub 식별자(ID). base-patch32는 가장 가볍고 빠른 기본 모델
model_id = "openai/clip-vit-base-patch32"

# CLIPProcessor: 이미지와 텍스트를 모델 입력 형식으로 변환
# (이미지 리사이즈·정규화 + 텍스트 토크나이징을 한 객체가 모두 담당)
processor = CLIPProcessor.from_pretrained(model_id)

# CLIPModel: 이미지 인코더 + 텍스트 인코더 통합 모델
# .to(device): 모델 가중치를 선택한 장치(GPU/CPU)로 이동시킴
model = CLIPModel.from_pretrained(model_id).to(device)
model.eval()                                          # 추론(평가) 모드로 전환 → 드롭아웃 등 학습 전용 동작 비활성화

print("CLIP 모델 로드 완료")
# projection_dim: 이미지/텍스트가 함께 놓이는 공동 임베딩 공간의 차원(여기선 512)
print(f"이미지 임베딩 차원: {model.config.projection_dim}")
# 출력: 이미지 임베딩 차원: 512

# ── 모델 파라미터 수 확인 ─────────────────────────────────
# 모든 가중치(parameter)의 원소 개수(numel)를 합산 → 모델 규모를 숫자로 확인
total_params = sum(p.numel() for p in model.parameters())
print(f"총 파라미터 수: {total_params:,} ({total_params/1e6:.1f}M)")   # 1e6으로 나눠 백만(M) 단위로 표기
# 출력: 총 파라미터 수: 151,277,313 (151.3M)
```

[코드 7-1] CLIP 모델 로드 및 기본 정보 확인

> CLIPProcessor는 이미지 전처리(리사이즈, 정규화)와 텍스트 토크나이징을 모두 담당합니다. 이미지와 텍스트를 별도의 전처리 과정 없이 `processor(images=[img], text=["..."], return_tensors="pt")` 한 줄로 처리할 수 있습니다.

### (3) 이미지 입력 → 벡터 생성 / 텍스트 입력 → 벡터 생성

CLIP의 이미지 인코더와 텍스트 인코더를 개별적으로 사용하여 각각의 임베딩 벡터를 생성하는 방법을 실습합니다. 이 임베딩 벡터는 코사인 유사도 계산, 검색 인덱스 구축, 분류 등 다양한 하위 태스크의 기반이 됩니다.

```python
import torch
import torch.nn.functional as F                       # F.normalize 같은 함수형 연산 모음(정규화 등에 사용)
from transformers import CLIPProcessor, CLIPModel
from PIL import Image
import numpy as np

# 전처리기와 모델을 다시 로드(이 코드 블록만 단독 실행해도 동작하도록)
processor = CLIPProcessor.from_pretrained("openai/clip-vit-base-patch32")
model = CLIPModel.from_pretrained("openai/clip-vit-base-patch32")
model.eval()                                          # 추론 모드


#
# 이미지 임베딩 생성
#
def encode_image(image_input, normalize: bool = True) -> torch.Tensor:
    """
    PIL Image 또는 파일 경로로부터 CLIP 이미지 임베딩 벡터를 생성합니다.
    Args:
        image_input: PIL.Image 객체 또는 이미지 파일 경로 문자열
        normalize: True이면 L2 정규화 적용 (코사인 유사도 계산 시 내적으로 대체 가능)
    Returns:
        shape [1, 512] 의 임베딩 텐서
    """
    # 입력이 문자열(파일 경로)이면 파일을 열어 RGB 이미지 객체로 변환
    if isinstance(image_input, str):
        image_input = Image.open(image_input).convert("RGB")

    # CLIPProcessor로 이미지 전처리
    # - 224×224 리사이즈
    # - RGB 정규화 (mean=[0.48145, 0.4578, 0.40821], std=[...])
    # return_tensors="pt": 결과를 PyTorch 텐서 형태로 반환
    inputs = processor(images=image_input, return_tensors="pt")

    with torch.no_grad():                             # 그래디언트 계산 비활성화 → 메모리 절약 + 추론 속도 향상
        # get_image_features(): 이미지 인코더 출력 → Projection Layer 통과
        # 결과로 [1, 512] 형태의 이미지 임베딩 벡터를 얻음
        image_features = model.get_image_features(**inputs)

    if normalize:
        # L2 정규화: 벡터 길이를 1로 만듦 → 코사인 유사도를 단순 내적으로 계산 가능
        image_features = F.normalize(image_features, p=2, dim=-1)

    return image_features


#
# 텍스트 임베딩 생성
#
def encode_text(texts, normalize: bool = True) -> torch.Tensor:
    """
    텍스트 문자열 또는 리스트로부터 CLIP 텍스트 임베딩 벡터를 생성합니다.
    Args:
        texts: 문자열 또는 문자열 리스트
        normalize: True이면 L2 정규화 적용
    Returns:
        shape [N, 512] 의 임베딩 텐서 (N = 텍스트 수)
    """
    # 문자열 하나가 들어와도 리스트로 감싸 일관되게 배치 처리
    if isinstance(texts, str):
        texts = [texts]

    # CLIPProcessor로 텍스트 토크나이징
    # max_length=77, padding=True, truncation=True 자동 적용
    # (padding: 짧은 문장을 맞춰 채움 / truncation: 77 토큰 초과 시 잘라냄)
    inputs = processor(text=texts, return_tensors="pt", padding=True, truncation=True)

    with torch.no_grad():                             # 추론 전용: 그래디언트 미계산
        # get_text_features(): 텍스트 인코더 출력 → Projection Layer 통과
        # 결과로 [N, 512] 형태(N=문장 수)의 텍스트 임베딩을 얻음
        text_features = model.get_text_features(**inputs)

    if normalize:
        # 이미지 임베딩과 같은 방식으로 L2 정규화 → 두 벡터를 직접 비교 가능
        text_features = F.normalize(text_features, p=2, dim=-1)

    return text_features
```

[코드 7-2] CLIP 이미지 및 텍스트 임베딩 생성 함수

```python
# ── 테스트 실행 ──────────────────────────────────────────
from PIL import Image
import numpy as np

# 테스트 이미지 생성 (실제 사용 시 실제 이미지 경로 사용)
# Image.open("your_image.jpg") 형태로 사용
test_image = Image.new("RGB", (224, 224), color=(100, 150, 200))  # 임시 파란색 이미지

# 이미지 임베딩 생성 → 앞서 정의한 encode_image 함수 호출
img_embedding = encode_image(test_image)
print(f"이미지 임베딩 shape: {img_embedding.shape}")                  # 벡터 형태 확인 ([1, 512])
# .norm(): 벡터 길이를 계산 → 정규화가 잘 됐으면 1.0이 나옴
print(f"이미지 임베딩 크기(L2 norm): {img_embedding.norm(dim=-1).item():.4f}")
# 출력:
# 이미지 임베딩 shape: torch.Size([1, 512])
# 이미지 임베딩 크기(L2 norm): 1.0000 (정규화 후)

# 텍스트 임베딩 생성: 여러 문장을 한 번에 처리(배치)
texts = [
    "a photo of a cat",
    "a blue sky with clouds",
    "a person riding a bicycle",
    "a bowl of ramen noodles",
]

text_embeddings = encode_text(texts)
print(f"텍스트 임베딩 shape: {text_embeddings.shape}")               # 문장 4개 → [4, 512]
# 출력: 텍스트 임베딩 shape: torch.Size([4, 512])

# 첫 번째 텍스트 임베딩 상위 5차원 값 확인 (벡터 내부 값이 실수임을 확인)
print("첫 텍스트 임베딩 (앞 5차원):", text_embeddings[0, :5].tolist())
# 출력: [0.0234, -0.0187, 0.0312, -0.0089, 0.0156] (예시)
```

### (4) 벡터 간 코사인 유사도 계산 → 가장 잘 맞는 이미지 찾기

L2 정규화된 벡터 간의 코사인 유사도는 단순한 내적(dot product) 연산으로 계산됩니다. 코사인 유사도는 -1에서 1 사이의 값으로, 1에 가까울수록 의미적으로 유사하다는 것을 의미합니다.

```python
import torch
import torch.nn.functional as F
from PIL import Image
import os


#
# 코사인 유사도 계산 함수
#
def cosine_similarity(vec_a: torch.Tensor, vec_b: torch.Tensor) -> torch.Tensor:
    """L2 정규화된 벡터 간 코사인 유사도 = 내적 (dot product)"""
    # vec_a: [1, 512], vec_b: [N, 512]
    # 결과: [1, N] - 각 vec_b에 대한 유사도 점수
    # torch.mm: 행렬 곱셈. vec_b.T(전치)와 곱해 한 번에 모든 쌍의 유사도를 계산
    return torch.mm(vec_a, vec_b.T)


#
# 이미지 검색 시스템 구현
#
class CLIPImageSearcher:
    """
    CLIP 기반 시맨틱 이미지 검색 클래스.
    이미지 컬렉션을 인덱싱하고 텍스트 쿼리로 검색합니다.
    """
    def __init__(self):
        # 인덱싱된 이미지들의 임베딩을 모아 둘 텐서(아직 비어 있음)
        self.image_embeddings = None  # shape: [N, 512]
        self.image_paths = []         # 이미지 경로 리스트 (임베딩 순서와 1:1 대응)
        self.image_metadata = []      # 추가 메타데이터 (선택)

    def index_images(self, image_paths: list, batch_size: int = 16):
        """이미지 목록을 배치 단위로 임베딩하여 인덱스 구축"""
        all_embeddings = []           # 배치별 임베딩을 모을 리스트
        valid_paths = []              # 정상 로드된 이미지 경로만 기록

        # batch_size 개씩 끊어서 반복 → 메모리 효율적으로 대량 이미지 처리
        for i in range(0, len(image_paths), batch_size):
            batch_paths = image_paths[i:i + batch_size]   # 이번 배치에 처리할 경로 묶음
            batch_images = []

            for path in batch_paths:
                try:
                    img = Image.open(path).convert("RGB") # 이미지 열기 + RGB 변환
                    batch_images.append(img)
                    valid_paths.append(path)              # 성공한 경로만 순서대로 저장
                except Exception as e:
                    # 깨진 파일 등 로드 실패 시 건너뛰고 계속 진행(전체 중단 방지)
                    print(f"이미지 로드 실패: {path} — {e}")
                    continue

            if not batch_images:                          # 배치가 통째로 비었으면 건너뜀
                continue

            # 배치 처리로 효율적 임베딩 (이미지 여러 장을 한 번에 인코더에 통과)
            inputs = processor(images=batch_images, return_tensors="pt", padding=True)
            with torch.no_grad():
                embeddings = model.get_image_features(**inputs)        # 배치 이미지 임베딩
                embeddings = F.normalize(embeddings, p=2, dim=-1)      # 유사도 비교용 L2 정규화

            all_embeddings.append(embeddings.cpu())       # GPU 메모리 절약 위해 CPU로 옮겨 누적
            print(f"인덱싱 진행: {min(i+batch_size, len(image_paths))}/{len(image_paths)}")

        if all_embeddings:
            # 배치별 임베딩들을 세로로 이어붙여 [N, 512] 하나의 인덱스 텐서로 만듦
            self.image_embeddings = torch.cat(all_embeddings, dim=0)
            self.image_paths = valid_paths
            print(f"인덱스 구축 완료: {len(self.image_paths)}장 이미지")
        else:
            raise ValueError("유효한 이미지가 없습니다.")

    def search(self, query_text: str, top_k: int = 5) -> list:
        """텍스트 쿼리로 가장 유사한 이미지를 검색"""
        if self.image_embeddings is None:               # 인덱스가 없으면 검색 불가
            raise RuntimeError("먼저 index_images()를 실행하세요.")

        # 텍스트 쿼리를 이미지와 같은 512차원 공간의 벡터로 변환
        text_embedding = encode_text(query_text)  # [1, 512]

        # 텍스트 벡터 1개 vs 전체 이미지 벡터 N개의 코사인 유사도 계산 → [1, N]
        similarities = cosine_similarity(text_embedding.cpu(), self.image_embeddings)
        similarities = similarities.squeeze(0)  # [N]   # 불필요한 차원 제거 → 1차원 점수 배열

        # 상위 top_k 결과 추출 (topk: 가장 큰 값과 그 위치(인덱스)를 함께 반환)
        top_scores, top_indices = similarities.topk(min(top_k, len(self.image_paths)))

        results = []
        for score, idx in zip(top_scores.tolist(), top_indices.tolist()):
            results.append({
                "rank": len(results) + 1,               # 1위부터 순위 부여
                "image_path": self.image_paths[idx],    # 인덱스로 원본 이미지 경로 역추적
                "similarity_score": round(score, 4),
                "similarity_pct": f"{score * 100:.1f}%" # 사람이 보기 쉽게 퍼센트로 변환
            })
        return results

    def search_by_image(self, query_image, top_k: int = 5) -> list:
        """이미지로 유사한 이미지를 검색 (이미지-to-이미지 검색)"""
        img_embedding = encode_image(query_image)  # [1, 512]   # 쿼리 이미지도 벡터화
        similarities = cosine_similarity(img_embedding.cpu(), self.image_embeddings)
        similarities = similarities.squeeze(0)
        # top_k+1: 쿼리 이미지 자신이 1위로 잡히므로 한 개 더 뽑아둠
        top_scores, top_indices = similarities.topk(min(top_k + 1, len(self.image_paths)))

        results = []
        for score, idx in zip(top_scores.tolist(), top_indices.tolist()):
            results.append({
                "rank": len(results) + 1,
                "image_path": self.image_paths[idx],
                "similarity_score": round(score, 4),
            })
        return results[1:]  # 자기 자신 제외     # 1위(자기 자신)를 빼고 반환
```

```python
# ── 검색 시스템 사용 예시 ────────────────────────────────
import glob

# 이미지 인덱스 구축
searcher = CLIPImageSearcher()

# 현재 디렉토리의 모든 JPG/PNG 파일 인덱싱
# glob: 패턴에 맞는 파일 목록 수집, recursive=True + ** 로 하위 폴더까지 탐색
image_files = glob.glob("./images/**/*.jpg", recursive=True)
image_files += glob.glob("./images/**/*.png", recursive=True)   # PNG도 추가

searcher.index_images(image_files, batch_size=32)   # 32장씩 배치로 임베딩 인덱스 생성

# 텍스트 쿼리로 검색 (자연어 문장으로 이미지 검색)
query = "a dog playing in the park"
results = searcher.search(query, top_k=5)           # 상위 5개 결과 반환

print(f"쿼리: \"{query}\"에 대한 검색 결과 TOP 5")
print("-" * 55)
for r in results:
    print(f" {r['rank']}위 {r['similarity_pct']:6s} {r['image_path']}")   # 순위·점수·경로 출력


# ── CLIP으로 Zero-shot 이미지 분류 ──────────────────────
def zero_shot_classify(image, candidate_labels: list) -> dict:
    """레이블 목록을 기반으로 Zero-shot 이미지 분류 수행"""
    # 학습 없이도 레이블 텍스트만 주면 분류 가능한 것이 CLIP의 핵심 강점
    img_emb = encode_image(image)             # [1, 512]  이미지 벡터
    txt_emb = encode_text(candidate_labels)   # [N, 512]  후보 레이블 벡터들

    # 유사도 계산 후 softmax로 확률 변환
    logit_scale = model.logit_scale.exp()     # CLIP 내부 스케일 팩터 (유사도를 확률로 더 또렷하게 만듦)
    logits = logit_scale * cosine_similarity(img_emb, txt_emb)  # [1, N]  스케일 적용 유사도
    probs = logits.softmax(dim=-1).squeeze(0) # softmax로 합이 1인 확률 분포로 변환

    # {레이블: 확률} 형태의 딕셔너리로 묶어 반환
    return {label: round(prob.item(), 4)
            for label, prob in zip(candidate_labels, probs)}


# 테스트
test_image = Image.open("./test_dog.jpg")
labels = ["a cat", "a dog", "a bird", "a fish", "a car"]   # 분류 후보 레이블
results = zero_shot_classify(test_image, labels)

print("Zero-shot 분류 결과:")
# 확률 내림차순 정렬(-x[1])하여 가장 확신하는 레이블부터 출력
for label, prob in sorted(results.items(), key=lambda x: -x[1]):
    bar = "█" * int(prob * 30)                 # 확률을 막대그래프 길이로 시각화
    print(f" {label:15s} {bar:30s} {prob:.2%}")
```

## 7.2 멀티모달 챗봇 만들기 (LLaVA & MiniGPT-4)

### (1) MiniGPT-4 / LLaVA 개요 및 모델 구조

MiniGPT-4와 LLaVA는 비전 인코더(Vision Encoder)를 대형 언어 모델(LLM)에 연결하는 멀티모달 대화 모델입니다. 이미지를 입력받아 자연어로 상세히 설명하거나, 이미지에 대한 질문에 답하거나, 이미지 기반 추론을 수행할 수 있습니다. 이는 GPT-4 Vision, Google Gemini Vision과 유사한 오픈소스 구현입니다.

| 모델 | 비전 인코더 | 언어 모델 | 연결 방식 | 특징 |
|---|---|---|---|---|
| LLaVA-1.5 | CLIP ViT-L/14 | Vicuna-7B/13B | MLP Projection | 가장 널리 사용, 우수한 VQA 능력 |
| LLaVA-1.6 (Next) | CLIP ViT-L/14 (고해상도) | Mistral-7B/Qwen | MLP (고해상도 지원) | 고해상도 이미지, 뛰어난 OCR |
| MiniGPT-4 | EVA-ViT-G | Vicuna-13B | Q-Former (BLIP-2) | 최초의 오픈소스 GPT-4V 유사 모델 |
| InternVL | InternViT-6B | InternLM-2 | 동적 고해상도 타일링 | 고해상도, 다국어(한국어 지원) |
| Qwen-VL | ViT-bigG | Qwen-7B | Position-aware VQA | 알리바바 개발, 중/영어 강세 |
| moondream2 | SigLIP | Phi-1.5 (1.8B) | MLP | 초경량(~2GB), 엣지 배포 가능 |

LLaVA-1.5의 전체 아키텍처 흐름을 살펴봅니다. 핵심은 비전 인코더의 출력(이미지 패치 임베딩)을 LLM이 이해할 수 있는 텍스트 토큰과 동일한 공간으로 투영하는 MLP Projection입니다.

#### LLaVA-1.5 처리 흐름 (이미지+텍스트 → 응답 생성)

① 이미지 입력: 사용자 이미지 (임의 크기) → 336×336 리사이즈

② 비전 인코딩: CLIP ViT-L/14 → 256개 이미지 패치 임베딩 생성

- 각 패치: 1024차원 벡터

③ 투영 변환: MLP Projection (2-layer) → 256개 패치 × 4096차원

- (LLM의 토큰 임베딩 차원과 동일하게 맞춤)

④ 토큰 시퀀스 구성: [시스템 프롬프트 토큰] + [이미지 토큰 256개] + [질문 토큰]

⑤ LLM 추론: Vicuna-7B가 이 전체 시퀀스를 처리하여 응답 텍스트 자동 생성

⑥ 출력: 자연어 응답 텍스트 (스트리밍 또는 완성 후 반환)

### (2) "이미지를 보고 설명해줘" → 비전+텍스트 통합 프롬프트 설계

LLaVA 계열 모델은 특정 프롬프트 형식(Prompt Template)을 따릅니다. 모델마다 최적 프롬프트 형식이 다르며, 이를 올바르게 사용하지 않으면 응답 품질이 크게 저하됩니다. Hugging Face transformers의 `apply_chat_template()`을 사용하면 모델별 최적 형식이 자동 적용됩니다.

```python
# LLaVA-Next 전용 전처리기와 모델 클래스 (이미지+텍스트를 함께 다룸)
from transformers import LlavaNextProcessor, LlavaNextForConditionalGeneration
import torch
from PIL import Image

# ── LLaVA-1.6 (LLaVA-Next) 로드 ─────────────────────────
# 참고: 첫 다운로드 시 ~16GB 소요 (7B 모델)
model_id = "llava-hf/llava-v1.6-mistral-7b-hf"

# 전처리기: 이미지 전처리 + 텍스트 토크나이징 + 채팅 템플릿 적용 담당
processor = LlavaNextProcessor.from_pretrained(model_id)
model = LlavaNextForConditionalGeneration.from_pretrained(
    model_id,
    torch_dtype=torch.float16,   # 16비트 부동소수점으로 로드 → VRAM 절반 절약
    low_cpu_mem_usage=True,      # 로딩 시 CPU 메모리 사용량 최소화
    device_map="auto"     # GPU/CPU 자동 분산
)
model.eval()                     # 추론 모드


# ── 프롬프트 구성 방법 (apply_chat_template 사용) ────────
def build_llava_prompt(question: str) -> str:
    """
    LLaVA-1.6 Mistral 전용 채팅 템플릿 구성.
    이미지 위치는 <image> 토큰으로 지정됩니다.
    """
    # 채팅 형식: user 역할 메시지 안에 이미지와 텍스트를 함께 담음
    conversation = [
        {
            "role": "user",
            "content": [
                {"type": "image"},                    # 이미지 위치 지정 (실제 이미지는 processor가 끼워 넣음)
                {"type": "text", "text": question},   # 텍스트 질문
            ],
        }
    ]
    # apply_chat_template: 모델별 올바른 특수 토큰/형식으로 프롬프트 문자열 생성
    prompt = processor.apply_chat_template(
        conversation,
        add_generation_prompt=True   # 모델이 답변을 이어 쓰도록 응답 시작 표시를 덧붙임
    )
    return prompt


# ── 다양한 프롬프트 예시 ─────────────────────────────────
prompt_examples = {
    "이미지 설명": "Please describe this image in detail.",
    "한국어 설명": "이 이미지에 대해 한국어로 자세히 설명해주세요.",
    "물체 나열": "List all the objects you can see in this image.",
    "장면 분석": "What is happening in this image? What is the mood?",
    "텍스트 추출": "Please transcribe all the text visible in this image.",
    "감성 분석": "What emotions or feelings does this image convey?",
    "비교 분석": "What are the main differences between the left and right parts of this image?",
    "스토리 생성": "Create a short story inspired by this image (3~4 sentences).",
    "제품 설명": "Describe this product as if you were writing an e-commerce listing.",
}

# 각 예시 질문으로 프롬프트를 만들어 앞 80자만 미리보기 출력
for task, question in prompt_examples.items():
    print(f"[{task}]: {build_llava_prompt(question)[:80]}...")
```

[코드 7-6] LLaVA-1.6 모델 로드 및 다양한 프롬프트 구성

```python
# ── 이미지 + 텍스트 → 응답 생성 핵심 함수 ──────────────
def generate_response(
    image: Image.Image,
    question: str,
    max_new_tokens: int = 512,
    temperature: float = 0.2,
    do_sample: bool = False
) -> str:
    """
    LLaVA 모델로 이미지에 대한 질문에 답변을 생성합니다.

    Args:
        image:          PIL Image 객체
        question:       이미지에 대한 질문 (자연어)
        max_new_tokens: 생성할 최대 토큰 수
        temperature:    생성 다양성 (낮을수록 결정론적)
        do_sample:      True이면 샘플링, False이면 greedy 디코딩
    Returns:
        모델의 응답 텍스트 문자열
    """
    prompt = build_llava_prompt(question)   # 질문을 LLaVA 채팅 형식 프롬프트로 변환

    # 이미지 + 텍스트 프롬프트 → 모델 입력 형식으로 변환
    # (이미지는 패치 임베딩으로, 텍스트는 토큰 ID로 변환되어 하나의 입력이 됨)
    inputs = processor(images=image, text=prompt, return_tensors="pt")
    inputs = {k: v.to(model.device) for k, v in inputs.items()}   # 모든 입력 텐서를 모델과 같은 장치로 이동

    with torch.inference_mode():            # no_grad보다 강한 추론 전용 모드 (속도·메모리 최적)
        output_ids = model.generate(        # 자동회귀 방식으로 응답 토큰을 한 개씩 생성
            **inputs,
            max_new_tokens=max_new_tokens,  # 새로 생성할 최대 토큰 수 제한
            temperature=temperature,        # 낮을수록 일관적, 높을수록 다양한 답변
            do_sample=do_sample,            # False면 항상 가장 확률 높은 토큰 선택(greedy)
            pad_token_id=processor.tokenizer.eos_token_id,   # 패딩 토큰을 문장 종료 토큰으로 지정(경고 방지)
        )

    # 입력 토큰 제외하고 생성된 부분만 디코딩
    input_len = inputs["input_ids"].shape[1]      # 입력 프롬프트의 토큰 길이
    generated_ids = output_ids[:, input_len:]     # 입력 이후 새로 생성된 토큰만 잘라냄
    response = processor.decode(generated_ids[0], skip_special_tokens=True)   # 토큰 ID → 사람이 읽는 문자열
    return response.strip()                        # 앞뒤 공백 제거 후 반환


# ── 실행 예시 ────────────────────────────────────────────
image = Image.open("./sample_image.jpg").convert("RGB")

questions = [
    "What is shown in this image?",
    "이 이미지에서 보이는 것을 한국어로 설명해주세요.",
    "How many people are in this image?",
]

# 같은 이미지에 대해 질문 3개를 차례로 보내 답변 생성
for q in questions:
    print(f"\nQ: {q}")
    answer = generate_response(image, q, max_new_tokens=200)
    print(f"A: {answer}")
```

### (4) FastAPI로 간단한 챗봇 API 구성: 이미지 + 질문 → 설명 응답

멀티모달 챗봇을 FastAPI로 감싸서 이미지 파일과 텍스트 질문을 받아 자연어 응답을 반환하는 REST API를 구성합니다.

```python
# multimodal_chat_api.py
from contextlib import asynccontextmanager                 # 앱 수명주기(lifespan) 관리용 데코레이터
from fastapi import FastAPI, UploadFile, File, Form, HTTPException   # FastAPI 핵심 + 파일/폼 입력 도구
from pydantic import BaseModel                             # 응답 데이터 구조(스키마) 정의용
from typing import Optional
import torch, io, time
from PIL import Image
from transformers import AutoModelForCausalLM, AutoTokenizer

# ── 전역 모델 컨텍스트 ───────────────────────────────────
# 무거운 모델을 매 요청마다 로드하지 않도록, 한 번 로드해 전역 딕셔너리에 보관
mm_model = {}


@asynccontextmanager
async def lifespan(app: FastAPI):
    # 앱 시작 시 moondream2 로드 (경량, CPU/GPU 모두 동작)
    # lifespan: yield 이전은 서버 시작 시, 이후는 서버 종료 시 실행됨
    print("멀티모달 모델 로딩 중...")
    model_id = "vikhyatk/moondream2"
    revision = "2024-08-26"
    device = "cuda" if torch.cuda.is_available() else "cpu"

    mm_model["tokenizer"] = AutoTokenizer.from_pretrained(
        model_id, revision=revision
    )
    mm_model["model"] = AutoModelForCausalLM.from_pretrained(
        model_id,
        trust_remote_code=True,
        revision=revision,
        torch_dtype=torch.float16 if device == "cuda" else torch.float32,   # GPU면 float16, CPU면 float32
    ).to(device)
    mm_model["model"].eval()      # 추론 모드 고정
    mm_model["device"] = device
    print(f"모델 로드 완료 ({device})")

    yield                          # 여기서 서버가 요청을 처리하며 실행 상태로 머무름
    # ── 서버 종료 시 정리 작업 ──
    mm_model.clear()               # 전역 모델 참조 해제
    if torch.cuda.is_available():
        torch.cuda.empty_cache()   # GPU 캐시 메모리 반환


app = FastAPI(
    title="멀티모달 챗봇 API",
    description="이미지+텍스트를 함께 처리하는 멀티모달 AI API",
    version="1.0.0",
    lifespan=lifespan          # 위에서 정의한 모델 로딩/정리 수명주기 연결
)


# 응답 형식 정의: FastAPI가 이 구조대로 JSON을 검증·직렬화
class ChatResponse(BaseModel):
    answer: str                # AI가 생성한 답변
    question: str              # 사용자가 보낸 질문(되돌려줌)
    model: str                 # 사용한 모델 이름
    processing_time_ms: float  # 처리 소요 시간(ms)


# POST 엔드포인트: multipart/form-data로 이미지 파일 + 폼 텍스트를 함께 수신
@app.post("/api/v1/chat", response_model=ChatResponse)
async def multimodal_chat(
    image: UploadFile = File(..., description="이미지 파일 (JPG/PNG/WebP)"),   # File(...): 필수 업로드 파일
    question: str = Form(..., description="이미지에 대한 질문 (자연어)"),       # Form(...): 필수 폼 텍스트 필드
    max_tokens: int = Form(default=300, ge=50, le=1000)   # 폼 정수, 50~1000 범위 자동 검증
):
    """이미지 파일과 텍스트 질문을 받아 AI 응답을 반환합니다."""
    # 이미지 유효성 검사: MIME 타입이 image/ 로 시작하는지 확인
    if not image.content_type.startswith("image/"):
        raise HTTPException(400, "이미지 파일만 업로드 가능합니다.")   # 400 Bad Request

    content = await image.read()             # 업로드 파일을 바이트로 모두 읽음(비동기)
    if len(content) > 20 * 1024 * 1024:  # 20MB 제한
        raise HTTPException(413, "파일 크기가 20MB를 초과합니다.")     # 413 Payload Too Large

    try:
        # 바이트 → 메모리 버퍼 → PIL 이미지(RGB)로 변환 (디스크 저장 없이 처리)
        pil_image = Image.open(io.BytesIO(content)).convert("RGB")
    except Exception:
        raise HTTPException(400, "이미지 파일을 파싱할 수 없습니다.")

    start = time.time()                      # 추론 시간 측정 시작
    model = mm_model["model"]                # 시작 시 로드해 둔 전역 모델 재사용
    tokenizer = mm_model["tokenizer"]

    enc_image = model.encode_image(pil_image)                    # 이미지 인코딩
    answer = model.answer_question(enc_image, question, tokenizer)   # 이미지+질문 → 답변

    elapsed_ms = (time.time() - start) * 1000   # 경과 시간을 밀리초로 환산

    # 정의한 스키마에 맞춰 응답 반환 → FastAPI가 JSON으로 변환해 클라이언트에 전달
    return ChatResponse(
        answer=answer,
        question=question,
        model="moondream2",
        processing_time_ms=round(elapsed_ms, 2)
    )
```

## 7.3 FastAPI 기반 멀티모달 API 구성하기

### (1) 이미지와 텍스트를 함께 POST로 보내는 방법

REST API에서 이미지 파일과 텍스트 데이터를 함께 전송하는 방법은 여러 가지가 있습니다. 멀티모달 API에서 가장 일반적으로 사용되는 방식은 multipart/form-data 방식이며, FastAPI는 이를 UploadFile과 Form의 조합으로 간편하게 처리합니다.

| 전송 방식 | 이미지 전달 | 텍스트 전달 | 특징 | 권장 상황 |
|---|---|---|---|---|
| multipart/form-data | UploadFile (바이너리) | Form 필드 | HTTP 표준, Swagger 지원 | 대부분의 멀티모달 API |
| JSON + base64 | JSON 필드 (base64 인코딩) | JSON 필드 | 순수 JSON, 33% 크기 증가 | 모든 입출력을 JSON으로 통일할 때 |
| JSON + URL | JSON에 이미지 URL 포함 | JSON 필드 | 서버가 URL에서 직접 다운로드 | 이미지가 이미 웹에 공개된 경우 |
| Streaming (WebSocket) | 스트림 전송 | 스트림 전송 | 실시간, 저지연 | 실시간 카메라 피드 |

```python
# ── 방법별 클라이언트 요청 예시 ─────────────────────────

# 방법 1: multipart/form-data (권장)
import requests

with open("./image.jpg", "rb") as f:      # 이미지를 바이너리("rb") 모드로 열기
    response = requests.post(
        "http://localhost:8000/api/v1/chat",
        files={"image": ("image.jpg", f, "image/jpeg")},   # files= 로 보내면 파일 파트로 전송됨
        data={"question": "이 이미지에 무엇이 보이나요?", "max_tokens": 300}   # data= 는 폼 텍스트 필드
    )
print(response.json())                    # 서버 응답(JSON)을 파이썬 딕셔너리로 출력


# 방법 2: base64 JSON
import base64, json

with open("./image.jpg", "rb") as f:
    # 이미지 바이트를 base64 문자열로 인코딩 → JSON에 텍스트로 담아 전송 가능(용량 33% 증가)
    b64_image = base64.b64encode(f.read()).decode("utf-8")

response = requests.post(
    "http://localhost:8000/api/v1/chat-json",
```

### (2) UploadFile + Form 조합 처리

FastAPI에서 파일과 폼 데이터를 함께 처리할 때는 UploadFile과 Form을 함수 파라미터로 동시에 선언합니다. 단, 이 경우 request body가 multipart/form-data가 되므로 JSON body 파라미터(Pydantic 모델)와 혼용할 수 없다는 점에 주의해야 합니다.

```python
# schemas.py — 멀티모달 API 요청/응답 스키마
from pydantic import BaseModel, Field
from typing import Optional, List


# JSON body 스키마 (base64 방식용)
# Field(...): 첫 인자가 ... 이면 필수값. ge/le/min_length 등으로 입력값 자동 검증
class MultimodalJSONRequest(BaseModel):
    image_base64: str = Field(..., description="base64 인코딩된 이미지 데이터")    # 필수
    image_type: str = Field(default="image/jpeg", description="이미지 MIME 타입") # 기본값 지정
    question: str = Field(..., min_length=1, max_length=1000)   # 1~1000자 길이 제한
    max_tokens: int = Field(default=300, ge=50, le=2000)        # 50 이상 2000 이하만 허용
    temperature: float = Field(default=0.2, ge=0.0, le=2.0)     # 0.0~2.0 범위
    stream: bool = Field(default=False, description="스트리밍 응답 여부")

    class Config:
        # Swagger 문서에 보여줄 예시 입력값 (사용자가 형식을 쉽게 이해하도록)
        json_schema_extra = {
            "example": {
                "image_base64": "base64_encoded_image_data_here...",
                "image_type": "image/jpeg",
                "question": "이 이미지에서 보이는 것을 설명해주세요.",
                "max_tokens": 300
            }
        }


# 응답 스키마
class MultimodalResponse(BaseModel):
    success: bool                         # 처리 성공 여부
    answer: str                           # AI 답변
    question: str
    model_id: str
    image_size: Optional[tuple] = None  # (width, height)   # 선택값(없을 수 있음)
    processing_time_ms: float
    tokens_generated: Optional[int] = None   # 생성된 토큰 수(선택)


# 여러 건을 한 번에 처리한 결과를 담는 배치 응답 스키마
class MultimodalBatchResponse(BaseModel):
    results: List[MultimodalResponse]     # 개별 결과들의 리스트
    total_time_ms: float                  # 전체 처리 시간
    success_count: int                    # 성공 건수
    fail_count: int                       # 실패 건수
```

```python
# multimodal_router.py — 멀티모달 엔드포인트 모음
from fastapi import APIRouter, UploadFile, File, Form, HTTPException, Depends
from fastapi.responses import StreamingResponse
from typing import Optional, List
import io, base64, time, json

from schemas import MultimodalResponse, MultimodalJSONRequest

# APIRouter: 엔드포인트를 모듈별로 나눠 관리. prefix로 공통 경로, tags로 문서 그룹 지정
router = APIRouter(prefix="/api/v1", tags=["Multimodal"])


# ── 엔드포인트 1: multipart/form-data (파일 업로드) ─────
@router.post("/chat", response_model=MultimodalResponse,
    summary="멀티모달 채팅 (파일 업로드)",
    description="이미지 파일과 텍스트 질문을 함께 전송하여 AI 응답을 받습니다."
)
async def chat_with_file(
    image: UploadFile = File(                 # 업로드 파일(필수)
        ...,
        description="JPG, PNG, WebP 형식의 이미지 파일"
    ),
    question: str = Form(                     # 폼 텍스트 질문(필수)
        ...,
        description="이미지에 대한 자연어 질문",
        example="이 이미지에서 무엇이 보이나요?"   # Swagger 예시값
    ),
    max_tokens: int = Form(default=300, ge=50, le=2000),
    temperature: float = Form(default=0.2, ge=0.0, le=2.0),
) -> MultimodalResponse:
    # 이미지 파일 검증: 허용된 MIME 타입 집합에 속하는지 확인
    allowed_types = {"image/jpeg", "image/png", "image/webp", "image/gif"}
    if image.content_type not in allowed_types:
        raise HTTPException(415, f"지원하지 않는 이미지 형식: {image.content_type}")   # 415 미지원 타입

    content = await image.read()              # 파일 바이트 읽기
    if len(content) > 20 * 1024 * 1024:
        raise HTTPException(413, "파일 크기가 20MB를 초과합니다.")

    try:
        pil_image = Image.open(io.BytesIO(content)).convert("RGB")   # 인메모리로 이미지 변환
    except Exception:
        raise HTTPException(400, "이미지 파일 파싱에 실패했습니다.")

    # 공통 추론 함수로 위임 → 세 엔드포인트가 같은 추론 로직 재사용
    return await _run_inference(pil_image, question, max_tokens, temperature)


# ── 엔드포인트 2: JSON + base64 ─────────────────────────
@router.post("/chat-json", response_model=MultimodalResponse,
    summary="멀티모달 채팅 (JSON/base64)"
)
async def chat_with_json(request: MultimodalJSONRequest) -> MultimodalResponse:
    # 본문 전체가 JSON → Pydantic 모델(request)로 자동 파싱·검증됨
    try:
        image_bytes = base64.b64decode(request.image_base64)   # base64 문자열 → 원본 바이트 복원
        pil_image = Image.open(io.BytesIO(image_bytes)).convert("RGB")
    except Exception as e:
        raise HTTPException(400, f"base64 이미지 디코딩 실패: {str(e)}")

    return await _run_inference(
        pil_image, request.question, request.max_tokens, request.temperature
    )


# ── 엔드포인트 3: URL에서 이미지 로드 ──────────────────
@router.post("/chat-url", response_model=MultimodalResponse,
    summary="멀티모달 채팅 (이미지 URL)"
)
async def chat_with_url(
    image_url: str = Form(..., description="공개 접근 가능한 이미지 URL"),
    question: str = Form(...)
) -> MultimodalResponse:
    import httpx                               # 비동기 HTTP 클라이언트

    # async with: 요청 후 클라이언트를 자동으로 닫음. timeout으로 무한 대기 방지
    async with httpx.AsyncClient(timeout=10.0) as client:
        try:
            resp = await client.get(image_url)  # 주어진 URL에서 이미지 다운로드
            resp.raise_for_status()             # 4xx/5xx 응답이면 예외 발생
        except Exception as e:
            raise HTTPException(400, f"이미지 URL 접근 실패: {str(e)}")

    pil_image = Image.open(io.BytesIO(resp.content)).convert("RGB")   # 받은 바이트 → 이미지
    return await _run_inference(pil_image, question, 300, 0.2)
```

### (3) FormData → 모델 입력 전처리 파이프라인

업로드된 파일과 폼 데이터를 모델이 요구하는 입력 형식으로 변환하는 전처리 파이프라인을 체계적으로 구성합니다. 입력 검증, 이미지 전처리, 텍스트 정규화, 배치 조립의 4단계로 구성됩니다.

```python
# preprocessing.py — 멀티모달 전처리 파이프라인
from PIL import Image
import io, re
from typing import Union
import torch


class MultimodalPreprocessor:
    """이미지+텍스트 입력의 전체 전처리 파이프라인을 담당합니다."""

    # 지원 이미지 형식 (클래스 전체에서 공유하는 상수)
    ALLOWED_FORMATS = {"JPEG", "PNG", "WEBP", "GIF", "BMP"}
    MAX_FILE_SIZE = 20 * 1024 * 1024  # 20 MB
    TARGET_SIZE = (336, 336)          # LLaVA 기본 입력 크기

    @staticmethod                     # 인스턴스 없이 클래스에서 바로 호출하는 유틸 함수
    def validate_image_bytes(content: bytes) -> None:
        """이미지 바이트 데이터의 유효성을 검사합니다."""
        if len(content) == 0:
            raise ValueError("빈 파일이 업로드되었습니다.")
        if len(content) > MultimodalPreprocessor.MAX_FILE_SIZE:
            raise ValueError(f"파일 크기가 20MB를 초과합니다: {len(content)/1024/1024:.1f}MB")

    @staticmethod
    def load_image(content: bytes) -> Image.Image:
        """바이트 데이터를 PIL Image로 변환하고 기본 전처리를 수행합니다."""
        MultimodalPreprocessor.validate_image_bytes(content)   # 먼저 크기/빈 파일 검증

        try:
            image = Image.open(io.BytesIO(content))            # 바이트 → 이미지 객체
        except Exception as e:
            raise ValueError(f"이미지 파싱 실패: {str(e)}")

        # 형식 검증: 허용 목록(JPEG/PNG 등)에 없는 포맷이면 거부
        if image.format not in MultimodalPreprocessor.ALLOWED_FORMATS:
            raise ValueError(f"지원하지 않는 이미지 형식: {image.format}")

        # RGBA → RGB 변환 (투명 채널 제거)
        # 모델은 보통 RGB 3채널을 기대 → 투명/팔레트 이미지를 흰 배경에 합성
        if image.mode in ("RGBA", "P"):
            background = Image.new("RGB", image.size, (255, 255, 255))   # 흰 배경 생성
            if image.mode == "P":
                image = image.convert("RGBA")          # 팔레트(P) → RGBA로 먼저 변환
            background.paste(image, mask=image.split()[3])   # 알파 채널을 마스크로 합성
            image = background
        else:
            image = image.convert("RGB")               # 그 외에는 단순 RGB 변환

        return image

    @staticmethod
    def resize_image(
        image: Image.Image,
        target_size: tuple = None,
        keep_aspect_ratio: bool = True,
        pad_to_square: bool = True
    ) -> Image.Image:
        """이미지를 모델 입력 크기로 리사이즈합니다."""
        if target_size is None:
            target_size = MultimodalPreprocessor.TARGET_SIZE   # 지정 없으면 기본 크기(336x336)

        if not keep_aspect_ratio:
            # 비율 무시하고 강제로 목표 크기로 늘림(이미지가 찌그러질 수 있음). LANCZOS=고품질 보간
            return image.resize(target_size, Image.LANCZOS)

        # 비율 유지 리사이즈: thumbnail은 가로세로 비율을 지키며 목표 크기 안에 맞춤
        image.thumbnail(target_size, Image.LANCZOS)

        if pad_to_square and image.size != target_size:
            # 흰색 배경으로 패딩하여 정사각형으로 만들기 (남는 여백을 흰색으로 채움)
            padded = Image.new("RGB", target_size, (255, 255, 255))
            offset = ((target_size[0] - image.width) // 2,    # 가운데 정렬 위한 좌상단 오프셋 계산
                      (target_size[1] - image.height) // 2)
            padded.paste(image, offset)                       # 배경 중앙에 이미지 붙이기
            return padded

        return image

    @staticmethod
    def normalize_question(question: str) -> str:
        """텍스트 질문을 정규화합니다."""
        # 앞뒤 공백 제거
        question = question.strip()
        # 연속 공백 단일화 (탭/줄바꿈 포함 모든 공백을 한 칸으로)
        question = re.sub(r"\s+", " ", question)
        # 최대 길이 초과 시 잘라내기 (너무 긴 입력으로 인한 오류 방지)
        if len(question) > 500:
            question = question[:500] + "..."
        return question

    def preprocess(
        self,
        image_bytes: bytes,
        question: str,
        resize: bool = True
    ) -> dict:
        """전체 전처리 파이프라인을 실행하고 정제된 입력을 반환합니다."""
        pil_image = self.load_image(image_bytes)   # 1) 검증 + 이미지 로드
        original_size = pil_image.size             # 원본 크기 기록(응답 메타데이터용)

        if resize:
            pil_image = self.resize_image(pil_image)   # 2) 모델 입력 크기로 리사이즈

        normalized_question = self.normalize_question(question)   # 3) 질문 텍스트 정규화

        # 모델 추론에 바로 쓸 수 있는 정제된 입력 묶음 반환
        return {
            "image": pil_image,
            "question": normalized_question,
            "original_size": original_size,
            "processed_size": pil_image.size,
        }


# 전처리기 싱글턴 (애플리케이션 전체가 하나의 인스턴스를 공유)
preprocessor = MultimodalPreprocessor()
```

### (4) 이미지 추론 결과 + 텍스트 응답 결합 구조

추론 결과를 구조화된 응답으로 결합하고, 다양한 형식(JSON, 스트리밍 등)으로 반환하는 방법을 구현합니다. 특히 스트리밍(Server-Sent Events) 방식은 응답이 길 때 사용자 경험을 크게 향상시킵니다.

```python
# inference_runner.py — 추론 실행 및 응답 결합
import asyncio, time
from concurrent.futures import ThreadPoolExecutor
from fastapi.responses import StreamingResponse
from PIL import Image
from schemas import MultimodalResponse

# 추론은 CPU/GPU 연산이므로 별도 스레드 풀에서 실행
# (무거운 동기 작업을 워커 스레드에 맡겨 비동기 이벤트 루프가 막히지 않게 함)
_executor = ThreadPoolExecutor(max_workers=2)


def _sync_inference(pil_image: Image.Image, question: str,
                    max_tokens: int, temperature: float) -> dict:
    """동기식 모델 추론 (스레드에서 실행)"""
    from main import mm_model  # 앱 전역 모델 참조 (순환 import 방지 위해 함수 내부에서 import)

    start = time.time()
    enc_image = mm_model["model"].encode_image(pil_image)          # 이미지 인코딩
    answer = mm_model["model"].answer_question(                    # 이미지+질문 → 답변 생성
        enc_image, question, mm_model["tokenizer"]
    )
    elapsed_ms = (time.time() - start) * 1000                      # 추론 소요 시간(ms)

    return {
        "answer": answer,
        "processing_time_ms": round(elapsed_ms, 2),
        "image_size": pil_image.size,
    }


async def _run_inference(
    pil_image: Image.Image,
    question: str,
    max_tokens: int = 300,
    temperature: float = 0.2
) -> MultimodalResponse:
    """비동기 래퍼: 스레드 풀에서 동기 추론을 실행합니다."""
    loop = asyncio.get_event_loop()
    # run_in_executor: 동기 함수(_sync_inference)를 스레드 풀에서 비동기로 await
    result = await loop.run_in_executor(
        _executor,
        _sync_inference,
        pil_image, question, max_tokens, temperature
    )
    # 추론 결과를 응답 스키마로 포장해 반환
    return MultimodalResponse(
        success=True,
        answer=result["answer"],
        question=question,
        model_id="moondream2",
        image_size=result["image_size"],
        processing_time_ms=result["processing_time_ms"]
    )


# ── 스트리밍 응답 엔드포인트 (Server-Sent Events) ────────
from fastapi import APIRouter
router = APIRouter()


@router.post("/api/v1/chat/stream")
async def chat_stream(
    image: UploadFile = File(...),
    question: str = Form(...)
):
    content = await image.read()
    pil_image = Image.open(io.BytesIO(content)).convert("RGB")

    # 제너레이터: yield 할 때마다 한 조각씩 클라이언트로 흘려보냄 (SSE 형식)
    async def event_generator():
        # 시작 이벤트: "data: ...\n\n" 가 SSE 한 개 이벤트 단위
        yield f"data: {json.dumps({'status': 'processing', 'question': question})}\n\n"

        # 추론 실행 (스트리밍은 별도 생성 구현 필요)
        loop = asyncio.get_event_loop()
        result = await loop.run_in_executor(   # 무거운 추론은 스레드 풀에서 실행
            _executor, _sync_inference, pil_image, question, 300, 0.2
        )

        # 응답 청크 전송 (단어 단위 시뮬레이션) — 완성된 답변을 단어로 쪼개 순차 전송
        words = result["answer"].split()
        for i, word in enumerate(words):
            chunk = word + (" " if i < len(words) - 1 else "")   # 마지막 단어 외엔 뒤에 공백 추가
            yield f"data: {json.dumps({'chunk': chunk, 'done': False})}\n\n"
            await asyncio.sleep(0.03)  # 자연스러운 타이핑 효과 (단어 사이 약간의 지연)

        # 완료 이벤트: done=True 로 스트림 끝과 총 처리 시간을 알림
        yield f"data: {json.dumps({'done': True, 'time_ms': result['processing_time_ms']})}\n\n"

    # StreamingResponse: 위 제너레이터가 만드는 데이터를 실시간 스트림으로 응답
    return StreamingResponse(
        event_generator(),
        media_type="text/event-stream",   # SSE 전용 MIME 타입
        headers={"Cache-Control": "no-cache", "X-Accel-Buffering": "no"}   # 프록시 버퍼링 비활성화로 즉시 전달
    )
```

## 7.4 업로드 + 프롬프트 통합 처리

### (1) 사용자 업로드 이미지 파일 → 임시 저장 or 메모리 처리

업로드된 이미지를 처리하는 방법은 크게 두 가지입니다. 파일로 저장하지 않고 메모리에서 직접 처리하는 In-Memory 방식과, 임시 파일로 저장한 후 처리하는 방식입니다. 각 방식은 용도와 환경에 따라 장단점이 뚜렷합니다.

| 처리 방식 | 구현 방법 | 장점 | 단점 | 권장 상황 |
|---|---|---|---|---|
| 인메모리 처리 | io.BytesIO → PIL Image 직접 변환 | 빠름, 디스크 I/O 없음, 보안(파일 없음) | 메모리 사용량 증가, 대용량 불리 | 소~중규모 API, 일반적인 경우 |
| tempfile 임시 저장 | tempfile.NamedTemporaryFile | 라이브러리 호환성 우수, 경로 필요 시 | 디스크 I/O, 정리 필요 | 경로 필요 도구 (OpenCV 등) |
| 영구 저장 후 처리 | os.makedirs + file.write | 캐싱, 재사용, 로그 추적 가능 | 스토리지 관리 필요 | 감사 로그, 이미지 캐시 필요 시 |
| 스트리밍 처리 | SpooledTemporaryFile | 초대용량 이미지, 청크 처리 | 복잡한 구현 | 매우 큰 파일 (50MB+) |

```python
# file_handler.py — 이미지 파일 처리 전략 구현
import io, os, uuid, tempfile, shutil
from contextlib import asynccontextmanager
from PIL import Image
from fastapi import UploadFile


#
# 방법 1: 인메모리 처리 (권장 — 대부분의 경우)
#
async def load_image_to_memory(upload: UploadFile) -> Image.Image:
    """업로드 파일을 메모리에서 직접 PIL Image로 변환합니다."""
    content = await upload.read()                     # 파일 내용을 통째로 바이트로 읽음
    try:
        image = Image.open(io.BytesIO(content)).convert("RGB")   # 디스크 저장 없이 메모리에서 변환
    except Exception as e:
        raise ValueError(f"이미지 파싱 실패: {e}")
    finally:
        await upload.seek(0)  # 다시 읽을 수 있도록 포인터 리셋   # 읽기 위치를 처음으로 되돌림
    return image


#
# 방법 2: tempfile 임시 저장 (경로가 필요한 도구 사용 시)
#
@asynccontextmanager
async def save_as_tempfile(upload: UploadFile):
    """
    업로드 파일을 임시 파일로 저장하는 컨텍스트 매니저.
    with 블록 종료 시 임시 파일이 자동 삭제됩니다.

    Usage:
        async with save_as_tempfile(upload) as tmp_path:
            result = cv2.imread(tmp_path)  # 경로가 필요한 경우
    """
    # 확장자 추출 (원본 파일명에서 .jpg 등을 떼어냄, 없으면 .tmp)
    original_name = upload.filename or "upload"
    ext = os.path.splitext(original_name)[1] or ".tmp"

    # 임시 파일 생성 (delete=False: 컨텍스트 안에서 우리가 직접 삭제 제어)
    tmp_file = tempfile.NamedTemporaryFile(
        delete=False,
        suffix=ext,           # 확장자 유지 → 일부 도구가 형식 판별에 사용
        prefix="upload_"
    )
    try:
        content = await upload.read()
        tmp_file.write(content)   # 업로드 내용을 임시 파일에 기록
        tmp_file.flush()          # 버퍼를 디스크에 즉시 반영
        tmp_file.close()
        yield tmp_file.name  # 임시 파일 경로 반환   # with 블록에 경로를 넘겨줌
    finally:
        # 컨텍스트 종료 시 임시 파일 삭제 (성공/실패와 무관하게 항상 정리)
        if os.path.exists(tmp_file.name):
            os.unlink(tmp_file.name)


#
# 방법 3: 영구 저장 (캐싱/감사 로그 필요 시)
#
UPLOAD_DIR = "./uploads"
os.makedirs(UPLOAD_DIR, exist_ok=True)   # 업로드 폴더 생성(이미 있으면 통과)


async def save_upload_permanently(
    upload: UploadFile,
    subdirectory: str = ""
) -> str:
    """
    업로드 파일을 영구적으로 저장하고 저장 경로를 반환합니다.
    고유 UUID 파일명으로 충돌을 방지합니다.
    """
    ext = os.path.splitext(upload.filename or "img")[1] or ".bin"   # 확장자 추출
    unique_name = f"{uuid.uuid4().hex}{ext}"   # UUID 기반 고유 파일명 → 덮어쓰기 충돌 방지

    save_dir = os.path.join(UPLOAD_DIR, subdirectory)   # 하위 폴더 경로 조합(예: 연/월)
    os.makedirs(save_dir, exist_ok=True)
    save_path = os.path.join(save_dir, unique_name)

    content = await upload.read()
    with open(save_path, "wb") as f:   # 바이너리 쓰기 모드로 디스크에 저장
        f.write(content)
```

### (2) 프롬프트는 자연어 그대로 처리 (예: "이 그림은 무슨 내용이야?")

사용자가 입력하는 자연어 프롬프트는 한국어든 영어든 그대로 처리할 수 있도록 설계해야 합니다. 단, 언어 모델마다 최적의 언어가 다를 수 있으므로 필요에 따라 언어 감지 및 번역 옵션을 제공하면 사용자 경험을 높일 수 있습니다.

```python
# prompt_handler.py — 자연어 프롬프트 처리
from typing import Optional
import re


# ── 프롬프트 전처리 ───────────────────────────────────────
def clean_prompt(text: str) -> str:
    """사용자 입력 프롬프트를 정제합니다."""
    # 앞뒤 공백 제거
    text = text.strip()
    # 연속 공백 정리 (모든 공백 문자를 한 칸으로)
    text = re.sub(r"\s+", " ", text)
    # 최대 길이 제한 (CLIP 등 max 77 tokens, LLaVA max ~500 chars)
    if len(text) > 800:
        text = text[:800].rsplit(" ", 1)[0]  # 단어 중간에서 자르지 않도록   # 마지막 공백 기준으로 잘라 온전한 단어 유지
    return text


# ── 간단한 언어 감지 (langdetect 패키지 불필요) ──────────
def detect_language_simple(text: str) -> str:
    """간단한 규칙 기반 언어 감지 (한국어 vs 영어)"""
    # 한글 유니코드 범위: AC00-D7A3 (가-힣), 1100-11FF, 3130-318F
    # 한글 문자 개수를 세어 비율로 판단(외부 라이브러리 없이 가볍게)
    korean_chars = sum(1 for c in text if "가" <= c <= "힣"
                       or "ᄀ" <= c <= "ᇿ")
    korean_ratio = korean_chars / max(len(text), 1)   # 0으로 나누기 방지(max로 최소 1 보장)
    return "ko" if korean_ratio > 0.15 else "en"      # 한글 비율 15% 초과면 한국어로 판정


# ── 프롬프트 의도 분류 ────────────────────────────────────
INTENT_PATTERNS = {
    "describe": ["설명", "describe", "tell me about", "what is", "무엇", "어떤", "what do you see"],
    "count":     ["몇", "how many", "count", "number of"],
    "identify": ["누구", "who", "identify", "which"],
    "ocr":      ["텍스트", "글자", "읽어", "text", "read", "transcribe", "written"],
    "compare": ["비교", "compare", "difference", "차이", "같은", "다른"],
    "sentiment": ["감정", "느낌", "mood", "emotion", "feel"],
}


def classify_intent(question: str) -> str:
    """질문의 의도를 분류합니다."""
    question_lower = question.lower()   # 대소문자 구분 없이 매칭하기 위해 소문자화
    # 의도별 키워드 목록을 돌며, 질문에 하나라도 포함되면 해당 의도로 분류
    for intent, keywords in INTENT_PATTERNS.items():
        if any(kw in question_lower for kw in keywords):
            return intent
    return "general"   # 어떤 키워드에도 안 걸리면 일반 질문으로 처리


# ── 언어별 시스템 프롬프트 선택 ──────────────────────────
SYSTEM_PROMPTS = {
    "en": "You are a helpful visual AI assistant. Analyze the image carefully and provide accurate, detailed answers.",
    "ko": "당신은 도움이 되는 시각적 AI 어시스턴트입니다. 이미지를 주의 깊게 분석하고 정확하고 상세한 답변을 제공하세요.",
}


def build_full_prompt(question: str, include_system: bool = True) -> dict:
    """전처리된 프롬프트 정보와 감지된 언어를 반환합니다."""
    cleaned = clean_prompt(question)                # 1) 공백/길이 정제
    lang = detect_language_simple(cleaned)          # 2) 언어 감지(ko/en)
    intent = classify_intent(cleaned)               # 3) 의도 분류
    system = SYSTEM_PROMPTS.get(lang, SYSTEM_PROMPTS["en"])   # 4) 언어에 맞는 시스템 프롬프트 선택(없으면 영어)

    # 정제 결과와 메타정보를 한 딕셔너리로 묶어 반환
    return {
        "original": question,
        "cleaned": cleaned,
        "language": lang,
        "intent": intent,
        "system_prompt": system if include_system else None,
    }


# ── 테스트 ────────────────────────────────────────────────
test_questions = [
    "이 그림은 무슨 내용이야?",
    "How many people are in this photo?",
    "사진 속 텍스트를 읽어줘",
    "Describe the mood of this image",
]

# 각 질문을 처리해 감지된 언어와 의도를 한 줄로 출력
for q in test_questions:
    info = build_full_prompt(q)
    print(f"[{info['language']:2s}|{info['intent']:10s}] {q}")
```

### (3) 통합 처리 흐름: 업로드 → 전처리 → 모델 추론 → 응답

지금까지 다룬 모든 구성 요소를 통합하여 완전한 멀티모달 API 서버를 구성합니다. 이미지 업로드 처리, 텍스트 전처리, 모델 추론, 응답 구조화까지의 전체 흐름을 하나의 완성된 코드로 구현합니다.

#### 완전한 멀티모달 API 처리 흐름 (5단계)

① HTTP 요청 수신: POST /api/v1/chat

- multipart/form-data: image 파일 + question 텍스트

② 입력 검증 (Validation):

- 파일 크기 검사 (≤ 20MB)
- MIME 타입 검증 (image/jpeg, image/png, image/webp)
- 텍스트 길이 검사 (1 ~ 800자)

③ 전처리 (Preprocessing):

- 이미지: BytesIO → PIL Image → RGB 변환 → 336×336 리사이즈
- 텍스트: 공백 정리 → 언어 감지 → 의도 분류 → 시스템 프롬프트 결합

④ 모델 추론 (Inference):

- 비동기 스레드 풀에서 동기 추론 실행 (이벤트 루프 블로킹 방지)
- 이미지 인코딩 → 텍스트 임베딩 결합 → LLM 생성

⑤ 응답 구조화 및 반환:

- MultimodalResponse Pydantic 모델로 직렬화
- 처리 시간, 이미지 메타데이터, 언어 정보 포함

```python
# complete_api.py — 모든 구성 요소 통합 완전한 멀티모달 API 서버

from contextlib import asynccontextmanager
from fastapi import FastAPI, UploadFile, File, Form, HTTPException
from fastapi.middleware.cors import CORSMiddleware
from pydantic import BaseModel
from typing import Optional
import torch, io, time, asyncio
from concurrent.futures import ThreadPoolExecutor
from PIL import Image

# 자체 모듈 임포트 (앞 절들에서 만든 헬퍼/전처리기 재사용)
from file_handler import load_image_to_memory
from prompt_handler import build_full_prompt, clean_prompt
from preprocessing import MultimodalPreprocessor

# ── 전역 상태 ────────────────────────────────────────────
app_state = {}                                # 모델/토크나이저 등 전역 보관
_executor = ThreadPoolExecutor(max_workers=2) # 추론용 워커 스레드 풀
preprocessor = MultimodalPreprocessor()       # 전처리기 공용 인스턴스


@asynccontextmanager
async def lifespan(app: FastAPI):
    """앱 시작/종료 시 모델 초기화 및 정리"""
    print("=" * 50)
    print("멀티모달 API 서버 시작 중...")

    # 장치 선택 (GPU 우선)
    device = "cuda" if torch.cuda.is_available() else "cpu"
    dtype = torch.float16 if device == "cuda" else torch.float32   # GPU=16비트, CPU=32비트
    print(f"실행 장치: {device}")

    from transformers import AutoModelForCausalLM, AutoTokenizer
    model_id = "vikhyatk/moondream2"
    revision = "2024-08-26"

    app_state["tokenizer"] = AutoTokenizer.from_pretrained(
        model_id, revision=revision)
    app_state["model"] = AutoModelForCausalLM.from_pretrained(
        model_id, trust_remote_code=True, revision=revision,
        torch_dtype=dtype).to(device)        # 모델을 선택 장치로 이동
    app_state["model"].eval()                # 추론 모드
    app_state["device"] = device
    print("모델 로드 완료")
    print("=" * 50)

    yield                                    # 서버 실행(요청 처리) 구간

    # ── 종료 시 정리 ──
    app_state.clear()
    if torch.cuda.is_available():
        torch.cuda.empty_cache()             # GPU 메모리 반환
    print("서버 종료 완료")


app = FastAPI(
    title="멀티모달 AI API v1.0",
    description="이미지+텍스트 통합 멀티모달 AI 서비스",
    version="1.0.0",
    lifespan=lifespan
)

# CORS 설정 (프론트엔드 연동 시 필요)
# 다른 도메인의 웹 페이지에서 이 API를 호출할 수 있게 허용("*"=모든 출처 허용, 운영에선 제한 권장)
app.add_middleware(
    CORSMiddleware,
    allow_origins=["*"],     # 허용 출처
    allow_methods=["*"],     # 허용 HTTP 메서드
    allow_headers=["*"],     # 허용 요청 헤더
)


# ── 응답 스키마 ──────────────────────────────────────────
# 답변뿐 아니라 언어/의도/이미지 크기 등 처리 메타데이터까지 함께 반환
class FullMultimodalResponse(BaseModel):
    success: bool
    answer: str
    question_original: str        # 원본 질문
    question_cleaned: str         # 정제된 질문
    detected_language: str        # 감지 언어
    detected_intent: str          # 분류된 의도
    image_original_size: list     # 원본 이미지 크기 [W, H]
    image_processed_size: list    # 리사이즈 후 크기
    model_id: str
    processing_time_ms: float


# ── 핵심 추론 함수 (동기, 스레드에서 실행) ───────────────
def _run_model(image: Image.Image, question: str) -> str:
    """모델 추론 실행 (동기 함수 - 스레드에서 호출됨)"""
    model = app_state["model"]
    tokenizer = app_state["tokenizer"]
    enc = model.encode_image(image)                       # 이미지 인코딩
    return model.answer_question(enc, question, tokenizer)  # 이미지+질문 → 답변


# ── 메인 멀티모달 엔드포인트 ─────────────────────────────
@app.post("/api/v1/chat",
    response_model=FullMultimodalResponse,
    summary="멀티모달 채팅 API (통합)",
)
async def full_multimodal_chat(
    image: UploadFile = File(..., description="분석할 이미지 (JPG/PNG/WebP)"),
    question: str = Form(..., description="이미지에 대한 자연어 질문",
                         example="이 이미지에서 무엇이 보이나요?"),
    max_tokens: int = Form(default=300, ge=50, le=2000)
):
    # ① 모델 준비 확인 (시작 시 로딩이 안 끝났으면 503으로 재시도 유도)
    if "model" not in app_state:
        raise HTTPException(503, "모델이 아직 로드되지 않았습니다. 잠시 후 다시 시도해주세요.")

    # ② 이미지 로드 (인메모리)
    content = await image.read()
    if len(content) > 20 * 1024 * 1024:
        raise HTTPException(413, "파일 크기가 20MB를 초과합니다.")

    try:
        pil_image = Image.open(io.BytesIO(content)).convert("RGB")
        original_size = list(pil_image.size)   # 원본 크기 기록
    except Exception:
        raise HTTPException(400, "이미지 파일을 열 수 없습니다.")

    # ③ 이미지 전처리 (모델 입력 크기로 리사이즈)
    processed = preprocessor.resize_image(pil_image)
    processed_size = list(processed.size)

    # ④ 텍스트 전처리 (정제 + 언어/의도 감지)
    prompt_info = build_full_prompt(question)

    # ⑤ 비동기 추론 (스레드 풀에서 실행해 이벤트 루프 블로킹 방지)
    start = time.time()
    loop = asyncio.get_event_loop()
    try:
        answer = await loop.run_in_executor(
            _executor,
            _run_model,
            processed,
            prompt_info["cleaned"]   # 정제된 질문을 모델에 전달
        )
    except Exception as e:
        raise HTTPException(500, f"모델 추론 오류: {str(e)}")   # 추론 실패 시 500

    elapsed_ms = (time.time() - start) * 1000

    # ⑥ 응답 구조화 (답변 + 모든 메타데이터를 스키마에 담아 반환)
    return FullMultimodalResponse(
        success=True,
        answer=answer,
        question_original=question,
        question_cleaned=prompt_info["cleaned"],
        detected_language=prompt_info["language"],
        detected_intent=prompt_info["intent"],
        image_original_size=original_size,
        image_processed_size=processed_size,
        model_id="moondream2",
        processing_time_ms=round(elapsed_ms, 2)
```
