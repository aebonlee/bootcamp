# 3장. 자연어 처리 웹 서비스 만들기

이 장에서는 Hugging Face의 다양한 자연어처리(NLP) 모델을 FastAPI와 연동하여 실제 웹 API 서비스 형태로 구현합니다. 단순한 챗봇부터 시작해서 요약기, 번역기, 문서 유사도 검색기까지 실습하며, 사용자 요청을 받아 모델이 예측을 수행하고 결과를 응답하는 AI 웹 서비스의 전체 흐름을 이해하게 됩니다. 또한 Swagger UI 기반으로 프론트엔드 없이 API를 체험하고 REST API 설계와 데이터 흐름에 집중합니다.

## 이 장의 학습 목표

- FastAPI와 Hugging Face NLP 모델을 연동하여 실제 동작하는 웹 API 서비스를 구현합니다.
- 챗봇, 요약기, 번역기, 문서 유사도 검색기 등 핵심 NLP 서비스를 직접 만들어봅니다.
- Swagger UI를 통해 프론트엔드 없이 AI API를 체험하고 검증하는 방법을 익힙니다.

### 이 장의 개요

자연어 처리(NLP, Natural Language Processing)는 인공지능 분야에서 가장 활발히 응용되고 있는 영역 중 하나입니다. 텍스트를 이해하고, 변환하고, 생성하는 NLP 기술은 챗봇, 번역, 검색, 요약 등 수많은 실용 서비스의 핵심 엔진으로 자리 잡았습니다.

Hugging Face의 Transformers 라이브러리는 BERT, GPT-2, T5, MarianMT, sentence-transformers 등 수백 가지 NLP 모델을 단일 API로 제공하며, FastAPI와 결합하면 이러한 모델들을 빠르게 웹 서비스로 배포할 수 있습니다.

이 장에서 구현할 서비스 목록은 다음과 같습니다.

| 구현 서비스 | 사용 모델 / 기술 |
| --- | --- |
| 3.1 챗봇 API | KoGPT / GPT-2, DialoGPT (텍스트 생성) |
| 3.2 문서 요약 API | T5, KETI-AIR/ke-t5-base (Seq2Seq) |
| 3.3 번역 API | Helsinki-NLP/opus-mt-en-ko (MarianMT) |
| 3.4 유사도 검색 API | sentence-transformers + FAISS |
| 3.5 Swagger UI 체험 | FastAPI 내장 /docs, /redoc |

## 3.1 FastAPI 챗봇 만들기

챗봇(Chatbot)은 사용자의 텍스트 입력을 받아 자연스러운 응답을 생성하는 가장 직관적인 NLP 서비스입니다. 이 절에서는 Hugging Face의 텍스트 생성 모델을 FastAPI로 래핑하여 REST API 형태의 챗봇을 만듭니다.

### 3.1.1 챗봇의 기본 원리: 입력 텍스트 → 모델 추론 → 응답 텍스트

챗봇의 핵심 동작 원리는 단순합니다. 사용자가 입력한 텍스트(prompt)를 언어 모델에게 전달하면, 모델은 사전 학습된 지식을 기반으로 다음에 이어질 확률이 높은 토큰(token)들을 순서대로 생성하여 응답 텍스트를 만들어냅니다.

| 단계 | 설명 | 코드 역할 |
| --- | --- | --- |
| ① 사용자 입력 | HTTP POST 요청으로 텍스트 전달 | FastAPI 엔드포인트가 수신 |
| ② 토크나이징 | 텍스트 → 토큰 ID 시퀀스 변환 | `AutoTokenizer.encode()` |
| ③ 모델 추론 | 토큰 시퀀스 → 다음 토큰 예측 | `model.generate()` |
| ④ 디코딩 | 토큰 ID 시퀀스 → 텍스트 변환 | `tokenizer.decode()` |
| ⑤ 응답 반환 | JSON 형태로 클라이언트에 전달 | FastAPI Response |

텍스트 생성 모델의 동작 방식을 더 깊이 이해하려면 '생성 전략(Decoding Strategy)'에 대해 알아야 합니다. 어떤 전략을 선택하느냐에 따라 생성되는 텍스트의 품질과 다양성이 크게 달라집니다.

| 생성 전략 | 특징 및 사용 상황 |
| --- | --- |
| Greedy Search | 매 스텝마다 확률이 가장 높은 토큰 선택. 빠르지만 반복적이고 단조로운 텍스트 생성. |
| Beam Search | 여러 후보 시퀀스를 동시에 탐색. 품질 높은 번역, 요약에 적합. `num_beams` 파라미터로 조절. |
| Top-k Sampling | 확률 상위 k개 토큰 중 무작위 선택. 다양하고 창의적인 텍스트. k 값이 클수록 다양성 증가. |
| Top-p (Nucleus) Sampling | 누적 확률 p% 이내 토큰 중 선택. Top-k보다 유연하고 자연스러운 텍스트 생성. |
| Temperature | 로짓 분포를 스케일링. 낮을수록(0~1) 집중, 높을수록(>1) 다양하고 창의적. |

### 3.1.2 KoGPT 또는 GPT-2 기반 챗봇 구현

실습에서는 두 가지 모델을 사용합니다. 영어 챗봇에는 Microsoft의 DialoGPT를, 한국어 챗봇에는 SKT-AI의 KoGPT2를 활용합니다. 모델에 따라 입력 포맷과 생성 파라미터를 조금씩 다르게 설정해야 합니다.

**사용 모델 정보**

`microsoft/DialoGPT-medium`

- 영어 대화에 특화된 GPT-2 기반 모델
- 아키텍처: GPT-2 (124M ~ 345M 파라미터)
- 학습 데이터: Reddit 대화 데이터 147M 쌍
- 라이선스: MIT

`skt/kogpt2-base-v2`

- 한국어 GPT-2 기반 생성 모델
- 아키텍처: GPT-2 (124M 파라미터)
- 학습 데이터: 모두의말뭉치, CC-100-Kor, 나무위키 등 40GB
- 라이선스: Apache 2.0

먼저 챗봇 서비스 로직을 작성합니다. 서비스 레이어에서 모델을 로드하고 텍스트를 생성하는 핵심 로직을 구현합니다.

**services/chatbot_service.py**

```python
# services/chatbot_service.py
import torch  # PyTorch: 모델 연산과 GPU/CPU 텐서 처리를 담당
from transformers import (
    AutoTokenizer,          # 모델 ID만 주면 알맞은 토크나이저를 자동 선택해 로드
    AutoModelForCausalLM,   # GPT 계열(다음 단어 예측) 생성 모델용 클래스
    AutoModelForSeq2SeqLM,  # 인코더-디코더(번역/요약) 모델용 클래스 (여기선 import만)
)
from typing import Optional


class ChatbotService:
    """텍스트 생성 기반 챗봇 서비스"""

    def __init__(self):
        # 모델/토크나이저는 처음엔 비워두고(None) 서버 시작 시 load_* 로 채운다
        self.en_tokenizer = None
        self.en_model = None
        self.ko_tokenizer = None
        self.ko_model = None
        # GPU(cuda)가 있으면 GPU를, 없으면 CPU를 연산 장치로 선택
        self.device = 'cuda' if torch.cuda.is_available() else 'cpu'

    def load_english_model(self):
        """DialoGPT-medium 로드 (영어 챗봇)"""
        model_id = 'microsoft/DialoGPT-medium'  # Hugging Face 모델 저장소 ID
        print(f'[INFO] Loading English chatbot: {model_id}')
        # 텍스트 <-> 토큰 ID 변환을 담당하는 토크나이저 다운로드/로드
        self.en_tokenizer = AutoTokenizer.from_pretrained(model_id)
        self.en_model = AutoModelForCausalLM.from_pretrained(
            model_id,
            # GPU면 float16(메모리 절반)로, CPU면 float32로 가중치를 적재
            torch_dtype=torch.float16 if self.device == 'cuda' else torch.float32,
        ).to(self.device)  # 모델을 선택한 장치(GPU/CPU)로 이동
        self.en_model.eval()  # 추론 전용 모드 (드롭아웃 등 비활성화, 결과 안정화)
        print('[INFO] English chatbot loaded.')

    def load_korean_model(self):
        """KoGPT2 로드 (한국어 챗봇)"""
        model_id = 'skt/kogpt2-base-v2'  # SKT가 공개한 한국어 GPT-2 모델
        print(f'[INFO] Loading Korean chatbot: {model_id}')
        self.ko_tokenizer = AutoTokenizer.from_pretrained(
            model_id,
            # KoGPT2는 특수 토큰을 직접 지정해야 생성/패딩이 올바르게 동작한다
            bos_token='</s>', eos_token='</s>', unk_token='<unk>',
            pad_token='<pad>', mask_token='<mask>',
        )
        self.ko_model = AutoModelForCausalLM.from_pretrained(
            model_id
        ).to(self.device)  # 모델을 GPU/CPU로 이동
        self.ko_model.eval()  # 추론 모드로 전환
        print('[INFO] Korean chatbot loaded.')

    def generate_english_response(
        self,
        user_input: str,
        max_length: int = 200,    # 생성될 전체 시퀀스의 최대 토큰 길이
        temperature: float = 0.7, # 낮을수록 안정적, 높을수록 다양한 응답
        top_p: float = 0.9,       # 누적 확률 90% 안의 토큰만 후보로 사용(Nucleus)
    ) -> str:
        """DialoGPT를 이용한 영어 응답 생성"""
        if self.en_model is None:
            # 모델이 아직 로드되지 않았으면 명확한 오류로 알린다
            raise RuntimeError('English model not loaded.')

        # DialoGPT 형식: 입력 + EOS 토큰 추가
        # 사용자 발화 끝에 EOS를 붙여 "여기까지가 입력"임을 모델에 알려준다
        input_ids = self.en_tokenizer.encode(
            user_input + self.en_tokenizer.eos_token,
            return_tensors='pt',  # PyTorch 텐서로 반환
        ).to(self.device)

        with torch.no_grad():  # 추론이므로 기울기 계산 비활성화 (속도/메모리 절약)
            output_ids = self.en_model.generate(
                input_ids,
                max_length=max_length,
                # 패딩 토큰이 없는 모델이라 EOS ID를 패딩 ID로 대체 지정
                pad_token_id=self.en_tokenizer.eos_token_id,
                do_sample=True,            # 확률 샘플링 사용(다양성 확보)
                temperature=temperature,
                top_p=top_p,
                repetition_penalty=1.3,   # 반복 억제 (같은 말 반복 방지)
            )

        # 입력 부분을 제외한 응답만 디코딩
        # generate 결과에는 입력 토큰이 그대로 포함되므로, 입력 길이 이후만 잘라낸다
        response_ids = output_ids[:, input_ids.shape[-1]:]
        response = self.en_tokenizer.decode(
            response_ids[0],
            skip_special_tokens=True,  # EOS 등 특수 토큰은 출력에서 제거
        )
        return response.strip()  # 앞뒤 공백 제거 후 반환

    def generate_korean_response(
        self,
        user_input: str,
        max_length: int = 128,
        temperature: float = 0.8,
        top_k: int = 50,          # 확률 상위 50개 토큰 중에서만 샘플링
    ) -> str:
        """KoGPT2를 이용한 한국어 응답 생성"""
        if self.ko_model is None:
            raise RuntimeError('Korean model not loaded.')

        # KoGPT2 형식: <usr> 입력 <sys> 패턴
        # <usr>/<sys> 마커로 "사용자 발화 -> 시스템 답변" 구조를 모델에 알려준다
        prompt = f'<usr> {user_input} <sys> '
        input_ids = self.ko_tokenizer.encode(
            prompt, return_tensors='pt'
        ).to(self.device)

        with torch.no_grad():
            output_ids = self.ko_model.generate(
                input_ids,
                max_length=max_length,
                do_sample=True,
                temperature=temperature,
                top_k=top_k,
                repetition_penalty=2.0,   # 한국어는 반복이 잦아 더 강하게 억제
                pad_token_id=self.ko_tokenizer.pad_token_id,
                eos_token_id=self.ko_tokenizer.eos_token_id,  # EOS 만나면 생성 종료
            )

        generated = self.ko_tokenizer.decode(
            output_ids[0],
            skip_special_tokens=True,
        )
        # <sys> 이후 텍스트만 추출
        # 모델 출력에는 프롬프트가 섞여 있으므로 <sys> 뒤(=실제 답변)만 골라낸다
        if '<sys>' in generated:
            response = generated.split('<sys>')[-1].strip()
        else:
            # 특수 토큰이 디코딩에서 사라졌을 때를 대비한 폴백 처리
            response = generated[len(prompt):].strip()
        return response


# 싱글턴 인스턴스
# 앱 전체에서 모델을 한 번만 로드해 공유하기 위해 모듈 수준 단일 객체로 생성
chatbot_service = ChatbotService()
```

### 3.1.3 대화 이력을 저장하지 않는 단순 챗팅 구조

이 절에서 구현하는 챗봇은 '무상태(Stateless)' 구조입니다. 즉, 각 API 요청은 이전 대화 이력을 기억하지 않고 독립적으로 처리됩니다. 이는 서버 구현을 단순하게 유지하고, 수평 확장(Horizontal Scaling)이 용이하다는 장점이 있습니다.

| 구조 유형 | 특징 |
| --- | --- |
| 무상태 챗봇 (이 절 구현) | 각 요청이 독립적. 이전 대화 기억 없음. 서버 상태 관리 불필요. 확장 용이. |
| 상태 유지 챗봇 (심화) | 대화 히스토리를 서버(Redis 등) 또는 클라이언트에 저장. 문맥 연속성 지원. |

**다음 챕터에서 다룰 내용**

대화 이력을 유지하는 멀티턴(Multi-turn) 챗봇은 이 교재의 후속 챕터에서 다룹니다. `session_id` 기반의 대화 상태 관리와 Redis를 이용한 히스토리 저장 방법을 배울 예정입니다.

### 3.1.4 POST /chat API로 사용자 입력 처리

이제 FastAPI 라우터를 작성합니다. 클라이언트는 POST /chat 엔드포인트로 사용자 메시지와 언어 설정을 전송하고, 서버는 해당 언어의 생성 모델을 통해 응답을 반환합니다.

**models/schemas.py – Pydantic 공통 스키마**

```python
# models/schemas.py
from pydantic import BaseModel, Field  # 요청/응답 데이터 검증용 베이스 클래스와 필드 제약
from typing import Optional, List
from enum import Enum


# 허용되는 언어 값을 문자열 Enum으로 고정 -> 잘못된 값 입력을 자동 차단
class Language(str, Enum):
    korean = 'ko'
    english = 'en'


# ─── 챗봇 스키마 ───────────────────────────────────
# BaseModel 상속: 들어온 JSON을 자동으로 검증/파싱하는 요청 모델
class ChatRequest(BaseModel):
    message: str = Field(
        ...,              # ... 은 필수 입력값(생략 불가)을 의미
        min_length=1,     # 빈 문자열 거부
        max_length=500,   # 너무 긴 입력 차단
        description='사용자 입력 메시지 (최대 500자)',  # Swagger 문서에 표시
    )
    language: Language = Field(
        Language.english,  # 기본값: 영어
        description='응답 언어: ko(한국어) / en(영어)',
    )
    max_length: int = Field(200, ge=50, le=500,   # ge/le: 50 이상 500 이하로 범위 제한
                            description='생성 최대 토큰 길이')
    temperature: float = Field(0.7, ge=0.1, le=1.5,
                               description='생성 다양성 조절 (낮을수록 결정론적)')

    class Config:
        # Swagger UI의 입력창에 미리 채워질 예시 값
        json_schema_extra = {
            'example': {
                'message': 'What is artificial intelligence?',
                'language': 'en',
                'max_length': 200,
                'temperature': 0.7
            }
        }


# 챗봇 응답 형태를 정의 -> FastAPI가 이 구조로 직렬화/검증
class ChatResponse(BaseModel):
    user_message: str       # 사용자가 보낸 원본 메시지(에코)
    bot_response: str       # 모델이 생성한 답변
    language: str           # 사용된 언어 코드
```

## 3.2 요약기 API 만들기 (T5 기반)

문서 요약(Text Summarization)은 긴 텍스트에서 핵심 내용을 추출하거나 재구성하여 짧은 형태로 만드는 NLP 태스크입니다. 이 절에서는 T5(Text-to-Text Transfer Transformer) 아키텍처를 기반으로 한 요약 모델을 FastAPI로 구현합니다.

### 3.2.1 T5 모델 개념 소개 ('summarize: ' prefix 사용)

T5는 2019년 Google이 발표한 모델로, '모든 NLP 태스크를 텍스트→텍스트 변환 문제로 통일'한다는 혁신적인 아이디어를 제시했습니다. 번역, 요약, 분류, 질의응답 등 다양한 태스크를 별도의 출력 헤드 없이 단일 Seq2Seq 구조로 처리합니다.

T5의 핵심은 태스크 prefix입니다. 입력 텍스트 앞에 어떤 태스크를 수행할지를 나타내는 텍스트 접두사를 붙여 모델에게 의도를 전달합니다.

| T5 태스크 Prefix | 예시 입력 형식 |
| --- | --- |
| 요약 (Summarize) | `summarize: The stock market fell sharply today...` |
| 번역 (Translate) | `translate English to German: Hello, how are you?` |
| 문법 교정 (Grammar) | `grammar: She go to school yesterday.` |
| 질의응답 (Q&A) | `question: What is the capital? context: Paris is...` |

**T5 주요 변형 모델**

- `t5-small` : 60M 파라미터. 가볍고 빠름. 학습/테스트 환경에 적합.
- `t5-base` : 220M 파라미터. 성능과 속도의 균형.
- `t5-large` : 770M 파라미터. 높은 품질. GPU 권장.
- `google/flan-t5-base` : instruction-tuned 버전. 더 다양한 태스크 지원.
- `KETI-AIR/ke-t5-base` : 한국어 특화 T5. 한글 텍스트 요약에 최적화.

### 3.2.2 한글 뉴스 요약 모델 활용 실습 (KETI-AIR/ke-t5-base)

한국어 요약을 위해 KETI-AIR(한국전자기술연구원)에서 공개한 ke-t5-base 모델을 활용합니다. 이 모델은 T5 아키텍처에 한국어 데이터로 사전 학습되어, 한글 뉴스, 보고서 등의 요약에 효과적입니다.

**services/summarizer_service.py**

```python
# services/summarizer_service.py
import torch
from transformers import AutoTokenizer, AutoModelForSeq2SeqLM  # 요약은 인코더-디코더(Seq2Seq) 구조
from typing import Optional


class SummarizerService:
    """T5 기반 문서 요약 서비스"""

    # 영어 요약: facebook/bart-large-cnn 또는 t5-base
    EN_MODEL_ID = 'facebook/bart-large-cnn'  # CNN 뉴스로 학습된 영어 요약 특화 모델
    # 한국어 요약: KETI-AIR/ke-t5-base
    KO_MODEL_ID = 'KETI-AIR/ke-t5-base'      # 한국어로 사전학습된 T5 모델

    def __init__(self):
        # 영어/한국어 각각의 토크나이저·모델 슬롯 초기화
        self.en_tokenizer = None
        self.en_model = None
        self.ko_tokenizer = None
        self.ko_model = None
        self.device = 'cuda' if torch.cuda.is_available() else 'cpu'

    def load_model(self):
        """영어/한국어 요약 모델 로드"""
        print(f'[INFO] Loading English summarizer: {self.EN_MODEL_ID}')
        self.en_tokenizer = AutoTokenizer.from_pretrained(self.EN_MODEL_ID)
        self.en_model = AutoModelForSeq2SeqLM.from_pretrained(
            self.EN_MODEL_ID
        ).to(self.device)
        self.en_model.eval()  # 추론 모드

        print(f'[INFO] Loading Korean summarizer: {self.KO_MODEL_ID}')
        self.ko_tokenizer = AutoTokenizer.from_pretrained(self.KO_MODEL_ID)
        self.ko_model = AutoModelForSeq2SeqLM.from_pretrained(
            self.KO_MODEL_ID
        ).to(self.device)
        self.ko_model.eval()
        print('[INFO] Both summarizers loaded.')

    def summarize_english(
        self,
        text: str,
        max_length: int = 150,  # 요약문 최대 토큰 수
        min_length: int = 40,   # 요약문 최소 토큰 수(너무 짧아지지 않게)
        num_beams: int = 4,     # Beam Search 후보 수(품질↑, 속도↓)
    ) -> dict:
        """BART를 이용한 영어 문서 요약"""
        # 원문을 토큰화. max_length 초과분은 잘라냄(truncation)
        inputs = self.en_tokenizer(
            text,
            return_tensors='pt',
            max_length=1024,   # BART 입력 한계
            truncation=True,
        ).to(self.device)

        with torch.no_grad():  # 추론이므로 기울기 추적 끔
            summary_ids = self.en_model.generate(
                inputs['input_ids'],                  # 토큰 ID 시퀀스
                attention_mask=inputs['attention_mask'],  # 패딩 위치 무시용 마스크
                max_length=max_length,
                min_length=min_length,
                num_beams=num_beams,
                length_penalty=2.0,    # 긴 요약 선호 (1보다 크면 길이에 후한 점수)
                early_stopping=True,   # 모든 빔이 EOS에 도달하면 조기 종료
                no_repeat_ngram_size=3,  # 같은 3-gram 반복 금지(중복 문구 방지)
            )

        # 생성된 토큰 ID를 사람이 읽는 텍스트로 복원
        summary = self.en_tokenizer.decode(
            summary_ids[0],
            skip_special_tokens=True,
        )
        input_tokens = inputs['input_ids'].shape[1]   # 입력 토큰 개수
        output_tokens = summary_ids.shape[1]          # 출력 토큰 개수
        return {
            'summary': summary,
            'input_tokens': input_tokens,
            'output_tokens': output_tokens,
            # 압축 비율 = 입력/출력 토큰 수 (0으로 나눔 방지 위해 max(...,1))
            'compression_ratio': round(input_tokens / max(output_tokens, 1), 2),
        }

    def summarize_korean(
        self,
        text: str,
        max_length: int = 128,
        min_length: int = 20,
        num_beams: int = 4,
    ) -> dict:
        """ke-t5를 이용한 한국어 문서 요약"""
        # ke-t5는 'summarize: ' prefix 사용
        # T5 계열은 태스크를 prefix로 지정 -> "요약하라"는 지시를 텍스트로 전달
        input_text = f'summarize: {text}'
        inputs = self.ko_tokenizer(
            input_text,
            return_tensors='pt',
            max_length=512,    # T5-base 입력 한계
            truncation=True,
        ).to(self.device)

        with torch.no_grad():
            summary_ids = self.ko_model.generate(
                inputs['input_ids'],
                max_length=max_length,
                min_length=min_length,
                num_beams=num_beams,
                no_repeat_ngram_size=3,
                early_stopping=True,
            )

        summary = self.ko_tokenizer.decode(
            summary_ids[0],
            skip_special_tokens=True,
        )
        input_tokens = inputs['input_ids'].shape[1]
        output_tokens = summary_ids.shape[1]
        return {
            'summary': summary,
            'input_tokens': input_tokens,
            'output_tokens': output_tokens,
            'compression_ratio': round(input_tokens / max(output_tokens, 1), 2),
        }


# 앱 전역에서 공유할 요약 서비스 단일 인스턴스
summarizer_service = SummarizerService()
```

```python
                min_length=request.min_length,
```

### 3.2.4 길이 제한, max_length 조정, 미리보기 등 기능 포함

요약 품질을 제어하는 가장 중요한 파라미터인 max_length, min_length, num_beams에 대해 더 자세히 살펴봅니다.

| 파라미터 | 역할 및 효과 | 권장 설정값 |
| --- | --- | --- |
| max_length | 생성되는 요약 토큰의 최대 개수. 너무 작으면 내용이 잘림. | 원문 길이의 20~30% |
| min_length | 최소 생성 토큰 수. 너무 짧은 요약 방지. | max_length의 30% |
| num_beams | Beam Search 후보 수. 높을수록 품질↑, 속도↓ | 4~6 (균형점) |
| length_penalty | > 1.0이면 긴 출력 선호. < 1.0이면 짧은 출력 선호. | 요약: 1.0~2.0 |
| no_repeat_ngram_size | n-gram 반복 억제. 동일 구절 반복 방지. | 2~3 |
| early_stopping | 빔 탐색 조기 종료로 속도 향상. | True 권장 |

다음은 미리보기(Preview) 기능을 추가한 확장 엔드포인트입니다. 원문의 앞부분만 미리보기로 반환하여 사용자가 요약 전 내용을 확인할 수 있게 합니다.

## 3.3 번역 API 만들기 (MarianMT 기반)

번역(Machine Translation)은 NLP에서 가장 실용적인 태스크 중 하나입니다. 이 절에서는 Helsinki-NLP 그룹이 공개한 MarianMT 모델을 활용하여 다국어 번역 API를 구현합니다.

### 3.3.1 MarianMT로 영어 ↔ 한국어, 일본어 등 다국어 번역 실습

MarianMT는 Edinburgh 대학교와 Microsoft가 협력하여 개발한 고속 신경망 기계 번역(NMT, Neural Machine Translation) 프레임워크입니다. Helsinki-NLP 그룹은 이를 기반으로 1,000개 이상의 언어 쌍(language pair) 번역 모델을 Hugging Face Hub에 공개했습니다.

| 언어 쌍 | 모델 ID | 특징 |
| --- | --- | --- |
| 영어 → 한국어 | Helsinki-NLP/opus-mt-en-ko | 영문 → 한글 번역. 뉴스, 기술 문서에 강점. |
| 한국어 → 영어 | Helsinki-NLP/opus-mt-ko-en | 한글 → 영문 번역. |
| 영어 → 일본어 | Helsinki-NLP/opus-mt-en-jap | 영문 → 일본어 번역. |
| 영어 → 중국어 | Helsinki-NLP/opus-mt-en-zh | 영문 → 중국어 간체 번역. |
| 영어 → 프랑스어 | Helsinki-NLP/opus-mt-en-fr | 영문 → 프랑스어 번역. |
| 다국어 → 영어 | Helsinki-NLP/opus-mt-mul-en | 100개 이상 언어 → 영문 번역. |

**MarianMT 아키텍처 특징**

- Marian C++ 번역 엔진을 Python/PyTorch로 포팅한 Transformer 기반 Seq2Seq 모델
- 각 언어 쌍마다 별도 모델 파일 (약 300MB ~ 1GB)
- 모델 ID 패턴: `Helsinki-NLP/opus-mt-{소스언어}-{목적언어}`
- OPUS 병렬 코퍼스로 학습 (위키피디아 번역, OpenSubtitles, CCAligned 등)
- MarianTokenizer 사용 (SentencePiece 기반)
- `from_pretrained()`으로 tokenizer와 model 모두 로드 가능

### 3.3.2 모델 예시: Helsinki-NLP/opus-mt-en-ko, opus-mt-ko-en

**services/translator_service.py**

```python
# services/translator_service.py
import torch
from transformers import MarianMTModel, MarianTokenizer  # MarianMT 전용 모델/토크나이저
from typing import List, Dict


class TranslatorService:
    """MarianMT 기반 다국어 번역 서비스"""

    # 지원 번역 방향 정의
    # 키는 'source-target' 형식, 값은 해당 방향의 Hugging Face 모델 ID
    SUPPORTED_PAIRS = {
        'en-ko': 'Helsinki-NLP/opus-mt-en-ko',
        'ko-en': 'Helsinki-NLP/opus-mt-ko-en',
        'en-ja': 'Helsinki-NLP/opus-mt-en-jap',
        'en-zh': 'Helsinki-NLP/opus-mt-en-zh',
        'en-fr': 'Helsinki-NLP/opus-mt-en-fr',
        'en-de': 'Helsinki-NLP/opus-mt-en-de',
    }

    def __init__(self):
        # 언어 쌍별로 모델/토크나이저를 따로 보관 (방향마다 별도 모델이 필요)
        self.models: Dict[str, MarianMTModel] = {}
        self.tokenizers: Dict[str, MarianTokenizer] = {}
        self.device = 'cuda' if torch.cuda.is_available() else 'cpu'

    def load_models(self, pairs: List[str] = None):
        """지정 언어 쌍 모델 로드. pairs=None이면 en-ko, ko-en 로드."""
        if pairs is None:
            pairs = ['en-ko', 'ko-en']  # 기본: 한국어-영어 쌍만 (메모리 절약)

        for pair in pairs:
            # 지원 목록에 없는 방향이면 건너뜀
            if pair not in self.SUPPORTED_PAIRS:
                print(f'[WARN] Unsupported pair: {pair}')
                continue
            model_id = self.SUPPORTED_PAIRS[pair]
            print(f'[INFO] Loading translation model: {model_id}')
            # 방향별 토크나이저/모델을 각각의 딕셔너리에 저장
            self.tokenizers[pair] = MarianTokenizer.from_pretrained(model_id)
            self.models[pair] = MarianMTModel.from_pretrained(
                model_id
            ).to(self.device)
            self.models[pair].eval()  # 추론 모드
            print(f'[INFO] Loaded: {pair}')

    def translate(
        self,
        text: str,
        source_lang: str,
        target_lang: str,
        num_beams: int = 4,      # Beam Search 후보 수
        max_length: int = 512,   # 번역문 최대 길이
    ) -> dict:
        pair = f'{source_lang}-{target_lang}'  # 'en-ko' 같은 방향 키 구성
        # 요청한 방향 모델이 로드돼 있지 않으면 명확한 오류
        if pair not in self.models:
            raise ValueError(
                f'번역 방향 {pair}가 로드되지 않았습니다. '
                f'지원 방향: {list(self.models.keys())}'
            )

        tokenizer = self.tokenizers[pair]  # 방향에 맞는 토크나이저 선택
        model = self.models[pair]          # 방향에 맞는 모델 선택

        # 입력 토크나이징
        inputs = tokenizer(
            [text],            # 리스트로 감싸 배치 형태로 처리
            return_tensors='pt',
            padding=True,
            truncation=True,
            max_length=512,
        ).to(self.device)

        with torch.no_grad():
            # **inputs: input_ids, attention_mask 등을 한 번에 전달
            translated_ids = model.generate(
                **inputs,
                num_beams=num_beams,
                max_length=max_length,
                early_stopping=True,
            )

        # 번역된 토큰 ID들을 텍스트로 복원 (배치 디코딩)
        translated = tokenizer.batch_decode(
            translated_ids,
            skip_special_tokens=True,
        )
        return {
            'translated': translated[0],  # 입력이 1개였으므로 첫 결과 사용
            'source_lang': source_lang,
            'target_lang': target_lang,
            'model_id': self.SUPPORTED_PAIRS[pair],  # 사용된 모델 ID
        }

    def translate_batch(
        self,
        texts: List[str],
        source_lang: str,
        target_lang: str,
        batch_size: int = 8,   # 한 번에 처리할 문장 묶음 크기
    ) -> List[str]:
        """다수 문장 배치 번역"""
        pair = f'{source_lang}-{target_lang}'
        tokenizer = self.tokenizers[pair]
        model = self.models[pair]
        results = []

        # texts를 batch_size 단위로 잘라 순차 처리 (메모리 폭증 방지)
        for i in range(0, len(texts), batch_size):
            batch = texts[i:i + batch_size]  # 현재 묶음
            inputs = tokenizer(
                batch, return_tensors='pt',
                padding=True, truncation=True  # 길이 다른 문장들을 패딩으로 정렬
            ).to(self.device)
            with torch.no_grad():
                ids = model.generate(**inputs, num_beams=4)
            decoded = tokenizer.batch_decode(ids, skip_special_tokens=True)
            results.extend(decoded)  # 묶음 결과를 누적

        return results


# 번역 서비스 전역 단일 인스턴스
translator_service = TranslatorService()
```

### 3.3.3 입력 언어 자동 감지 기능 없이 모델별 route 구성

각 번역 방향별로 독립적인 엔드포인트를 제공하거나, 단일 엔드포인트에 source_lang, target_lang 파라미터를 받는 방식으로 구성합니다. 언어 자동 감지(auto-detect)는 외부 라이브러리(langdetect 등)와 추가 모델이 필요하므로 이 절에서는 명시적 언어 지정 방식을 사용합니다.

**models/schemas.py – 번역 스키마**

```python
    en = 'en'
    ko = 'ko'
    ja = 'ja'
    zh = 'zh'
    fr = 'fr'
    de = 'de'


# 단일 문장 번역 요청 모델
class TranslateRequest(BaseModel):

        ...,
        min_length=1,

        description='번역할 원문 텍스트 (최대 2000자)',
    )
    source_lang: SupportedLang = Field(
        SupportedLang.en,  # 기본 원문 언어: 영어
        description='원문 언어 코드 (en/ko/ja/zh/fr/de)',
    )
    target_lang: SupportedLang = Field(
        SupportedLang.ko,  # 기본 목적 언어: 한국어
        description='번역 목적 언어 코드',
    )
    num_beams: int = Field(4, ge=1, le=8,
                           description='Beam Search 빔 수')

    class Config:
        # Swagger 입력 예시
        json_schema_extra = {
            'example': {
                'text': 'Artificial intelligence is transforming every industry.',
                'source_lang': 'en',
                'target_lang': 'ko',
                'num_beams': 4
            }
        }


# 단일 번역 응답 모델
class TranslateResponse(BaseModel):

# 배치 번역 응답 모델

    translated_texts: List[str]  # 번역문 목록 (입력과 같은 순서)

    target_lang: str
    count: int                   # 번역된 문장 수
```

**routers/translator.py**

```python
# routers/translator.py
from fastapi import APIRouter, HTTPException, status
from models.schemas import (TranslateRequest, TranslateResponse,
                            BatchTranslateRequest, BatchTranslateResponse)
from services.translator_service import translator_service

# /translate 접두사를 가진 번역 라우터
router = APIRouter(prefix='/translate', tags=['Translator'])


@router.post(
    '/',                            # 최종 경로: POST /translate/
    response_model=TranslateResponse,
    summary='텍스트 번역',
    description='지정 언어로 텍스트를 번역합니다. 지원 방향: en↔ko, en→ja/zh/fr/de',
)
async def translate_text(request: TranslateRequest):
    # 원문 언어와 목적 언어가 같으면 번역 의미가 없으므로 차단
    if request.source_lang == request.target_lang:
        raise HTTPException(
            status_code=status.HTTP_422_UNPROCESSABLE_ENTITY,
            detail='원문과 목적 언어가 동일합니다.',
        )

    try:
        # 서비스에 실제 번역 위임 (.value 로 Enum -> 문자열 코드 전달)
        result = translator_service.translate(
            text=request.text,
            source_lang=request.source_lang.value,
            target_lang=request.target_lang.value,
            num_beams=request.num_beams,
        )
        return TranslateResponse(
            original_text=request.text,
            translated_text=result['translated'],
            source_lang=request.source_lang.value,
            target_lang=request.target_lang.value,
            model_used=result['model_id'],
            char_count_original=len(request.text),
            char_count_translated=len(result['translated']),
        )
    except ValueError as e:
        # 미지원/미로드 방향 등은 400(잘못된 요청)으로 반환
        raise HTTPException(status_code=status.HTTP_400_BAD_REQUEST,
```

## 3.4 문서 유사도 검색 API (sentence-transformers + FAISS)

문서 유사도 검색(Semantic Search)은 단순한 키워드 매칭을 넘어, 텍스트의 '의미(Semantics)'를 벡터로 표현하여 의미적으로 가장 유사한 문서를 찾는 기술입니다. 이 절에서는 sentence-transformers와 Facebook의 FAISS 라이브러리를 조합하여 FAQ 검색, 뉴스 제목 검색 등을 구현합니다.

### 3.4.1 문장 임베딩 → 벡터 유사도 기반 검색 시스템 구조 설명

의미 검색 시스템의 전체 구조는 크게 '인덱싱 단계'와 '검색 단계'로 나뉩니다.

| 단계 | 수행 작업 |
| --- | --- |
| ① 문서 준비 | 검색 대상 문서(FAQ, 뉴스 제목 등) 목록 준비 |
| ② 임베딩 생성 | sentence-transformers로 각 문서를 고차원 벡터로 변환 |
| ③ FAISS 인덱스 구축 | 생성된 벡터들을 FAISS에 추가하여 검색 인덱스 구성 |
| ④ 쿼리 임베딩 | 사용자 검색어를 동일 모델로 벡터로 변환 |
| ⑤ 유사도 검색 | FAISS에서 쿼리 벡터와 가장 유사한 K개 문서 벡터 검색 |
| ⑥ 결과 반환 | 유사도 점수와 함께 가장 관련성 높은 문서 목록 반환 |

벡터 유사도 측정 방식으로는 코사인 유사도(Cosine Similarity)가 가장 널리 사용됩니다. 두 벡터가 이루는 각도의 코사인 값으로 유사도를 측정하며, 1.0에 가까울수록 의미가 비슷한 문서입니다.

**코사인 유사도 vs 유클리드 거리**

코사인 유사도 (Cosine Similarity):

- 벡터 방향의 유사성 측정 (크기 무관)
- NLP 임베딩에 가장 적합
- FAISS의 IndexFlatIP (내적) + 정규화로 구현
- 값 범위: -1.0 ~ 1.0 (높을수록 유사)

유클리드 거리 (L2 Distance):

- 두 벡터 간 직선 거리 측정
- FAISS의 IndexFlatL2로 구현
- 값 범위: 0 ~ ∞ (낮을수록 유사)

### 3.4.2 sentence-transformers 모델로 임베딩 생성

sentence-transformers 라이브러리는 문장 단위의 의미론적 임베딩을 생성하는 데 특화된 모델들을 제공합니다. 기존 BERT 계열 모델을 'Siamese Network' 방식으로 파인튜닝하여 문장 간 유사도 계산에 최적화되었습니다.

| 모델 ID | 특징 | 임베딩 차원 |
| --- | --- | --- |
| all-MiniLM-L6-v2 | 경량(22M). 빠른 속도. 영어 특화. 가장 많이 사용됨. | 384 |
| all-mpnet-base-v2 | 고품질(109M). 영어 최고 성능. GPU 권장. | 768 |
| paraphrase-multilingual-MiniLM-L12-v2 | 50개 언어 지원. 한국어 포함. 범용 다국어. | 384 |
| jhgan/ko-sroberta-multitask | 한국어 특화. STS, NLI 학습. 한국어 문장 유사도 최고. | 768 |

**services/similarity_service.py**

```python
# services/similarity_service.py
import json
import numpy as np  # 벡터 연산 및 폴백 유사도 계산용
from typing import List, Dict, Optional
from pathlib import Path

try:
    import faiss          # 고속 벡터 검색 라이브러리 (설치돼 있으면 사용)
    FAISS_AVAILABLE = True
except ImportError:
    # FAISS 미설치 환경에서도 동작하도록 numpy 폴백으로 전환
    FAISS_AVAILABLE = False
    print('[WARN] FAISS not installed. Using numpy fallback.')

from sentence_transformers import SentenceTransformer  # 문장 -> 의미 벡터 변환 모델


class SimilarityService:
    """sentence-transformers + FAISS 기반 문서 유사도 검색"""

    # 50여 개 언어를 지원하는 다국어 경량 임베딩 모델(한국어 포함)
    MODEL_ID = 'paraphrase-multilingual-MiniLM-L12-v2'

    def __init__(self):
        self.model: Optional[SentenceTransformer] = None
        self.documents: List[Dict] = []   # {'id', 'text', 'metadata'}  검색 대상 원본 문서
        self.embeddings: Optional[np.ndarray] = None  # 문서들의 임베딩 벡터 행렬
        self.index = None                 # FAISS 인덱스 (검색 자료구조)
        self.embedding_dim: int = 384     # 임베딩 차원(모델 로드 후 실제 값으로 갱신)

    def load_model(self):
        print(f'[INFO] Loading sentence-transformer: {self.MODEL_ID}')
        self.model = SentenceTransformer(self.MODEL_ID)
        # 모델이 출력하는 벡터 차원 수를 가져와 인덱스 차원과 맞춘다
        self.embedding_dim = self.model.get_sentence_embedding_dimension()
        print(f'[INFO] Embedding dim: {self.embedding_dim}')

    def add_documents(self, docs: List[Dict]):
        """문서 추가 (id, text, metadata 포함)"""
        self.documents.extend(docs)  # 기존 목록 뒤에 새 문서들을 이어붙임

    def build_index(self, data_path: str = 'data/sample_docs.json'):
        """FAISS 인덱스 구축"""
        # 샘플 데이터 로드
        # 문서가 비어 있으면 파일에서 읽고, 파일도 없으면 기본 샘플을 사용
        if not self.documents:
            path = Path(data_path)
            if path.exists():
                with open(path, 'r', encoding='utf-8') as f:
                    self.documents = json.load(f)
            else:
                self.documents = self._get_default_documents()

        texts = [doc['text'] for doc in self.documents]  # 임베딩할 문장만 추출
        print(f'[INFO] Building index for {len(texts)} documents...')

        # 임베딩 생성
        # 모든 문서를 한 번에 벡터로 변환
        self.embeddings = self.model.encode(
            texts,
            convert_to_numpy=True,
            normalize_embeddings=True,  # 코사인 유사도를 위해 정규화 (내적=코사인 성립)
            show_progress_bar=True,
        )

        if FAISS_AVAILABLE:
            # FAISS IndexFlatIP (내적 = 정규화 후 코사인 유사도)
            self.index = faiss.IndexFlatIP(self.embedding_dim)  # 내적 기반 인덱스 생성
            self.index.add(self.embeddings.astype(np.float32))  # 문서 벡터들을 인덱스에 등록
            print(f'[INFO] FAISS index built. Total: {self.index.ntotal} vectors')
        else:
            # FAISS가 없으면 검색 시 numpy로 직접 계산
            print('[INFO] Using numpy for similarity (no FAISS)')

    def search(
        self,
        query: str,
        top_k: int = 5,   # 반환할 상위 유사 문서 개수
    ) -> List[Dict]:
        """쿼리와 가장 유사한 문서 검색"""
        if self.model is None:
            raise RuntimeError('모델이 로드되지 않았습니다.')

        # 쿼리 임베딩
        # 검색어도 문서와 동일한 모델/정규화로 벡터화해야 비교가 정확
        query_embedding = self.model.encode(
            [query],
            convert_to_numpy=True,
            normalize_embeddings=True,
        ).astype(np.float32)

        if FAISS_AVAILABLE and self.index is not None:
            # FAISS로 가장 유사한 top_k개 검색 (점수와 문서 인덱스 반환)
            scores, indices = self.index.search(query_embedding, top_k)
            results = []
            for score, idx in zip(scores[0], indices[0]):
                if idx < 0:   # 결과가 부족하면 -1이 채워지므로 건너뜀
                    continue
                doc = self.documents[idx].copy()       # 원본 훼손 방지 위해 복사
                doc['similarity_score'] = float(score) # 유사도 점수 부착
                results.append(doc)
        else:
            # numpy fallback (코사인 유사도 직접 계산)
            # 정규화된 벡터끼리의 내적 = 코사인 유사도
            scores = np.dot(self.embeddings, query_embedding.T).flatten()
            top_indices = np.argsort(scores)[::-1][:top_k]  # 점수 내림차순 상위 k개
            results = []
            for idx in top_indices:
                doc = self.documents[idx].copy()
                doc['similarity_score'] = float(scores[idx])
                results.append(doc)

        return results

    def _get_default_documents(self) -> List[Dict]:
        """기본 FAQ 샘플 문서"""
        # 외부 데이터 파일이 없을 때 사용할 검색 대상 예시 문서 목록
        return [
            {'id': 1, 'text': 'FastAPI는 Python 기반의 고성능 웹 프레임워크입니다.',
             'category': 'FastAPI'},
            {'id': 2, 'text': 'Hugging Face는 AI 모델을 공유하는 플랫폼입니다.',
             'category': 'HuggingFace'},
            {'id': 3, 'text': 'Transformer는 어텐션 메커니즘 기반의 딥러닝 모델입니다.',
             'category': 'AI'},
            {'id': 4, 'text': 'BERT는 양방향 트랜스포머 기반의 언어 모델입니다.',
             'category': 'AI'},
            {'id': 5, 'text': 'GPT는 자동회귀 방식의 텍스트 생성 모델입니다.',
             'category': 'AI'},
            {'id': 6, 'text': 'Docker는 컨테이너 기반의 가상화 플랫폼입니다.',
             'category': 'DevOps'},
            {'id': 7, 'text': 'REST API는 HTTP 기반의 웹 서비스 설계 방식입니다.',
             'category': 'Web'},
            {'id': 8, 'text': 'PyTorch는 딥러닝 모델 학습에 사용되는 프레임워크입니다.',
             'category': 'AI'},
            {'id': 9, 'text': 'uvicorn은 ASGI 기반의 Python 웹 서버입니다.',
             'category': 'FastAPI'},
            {'id': 10, 'text': 'sentence-transformers는 문장 임베딩 생성 라이브러리입니다.',
             'category': 'AI'},
        ]


# 유사도 검색 서비스 전역 단일 인스턴스
similarity_service = SimilarityService()
```

### 3.4.3 FAISS를 이용한 유사도 검색 흐름 (간단 인덱스 구조)

FAISS(Facebook AI Similarity Search)는 Facebook AI Research가 개발한 고성능 벡터 유사도 검색 라이브러리입니다. 수백만 개의 벡터에서도 밀리초 단위로 최근접 이웃(Nearest Neighbor)을 찾을 수 있습니다.

| FAISS 인덱스 유형 | 특징 및 사용 적합 상황 |
| --- | --- |
| IndexFlatL2 | L2 거리 기반 정확한 검색. 소규모(~100K) 데이터에 적합. 메모리 큼. |
| IndexFlatIP | 내적(코사인 유사도) 기반. 정규화된 벡터에서 코사인 유사도와 동일. |
| IndexIVFFlat | Inverted File 방식. 대규모(~1M) 데이터. 학습 단계 필요. 속도↑, 정확도 약간↓ |
| IndexHNSW | 계층적 그래프 기반. 매우 빠른 검색. 메모리 사용 큼. 프로덕션 권장. |
| IndexPQ | Product Quantization으로 벡터 압축. 매우 대규모 데이터(~1B). 메모리↓ |

```python
                text=r['text'],
                category=r.get('category'),  # category 없으면 None
                similarity_score=round(r['similarity_score'], 4),
            )
            for r in raw_results
            if r['similarity_score'] >= request.threshold
        ]

        # 인덱스 크기: FAISS가 있으면 등록 벡터 수, 없으면 문서 개수
        index_size = (similarity_service.index.ntotal
                      if similarity_service.index else
                      len(similarity_service.documents))

        return SearchResponse(
            query=request.query,
            results=filtered,
            total_found=len(filtered),
            model_used=similarity_service.MODEL_ID,
            index_size=index_size,
        )

    except Exception as e:
        raise HTTPException(status_code=500,
                            detail=f'검색 오류: {str(e)}')


# 새 문서를 추가하고 인덱스를 다시 만드는 엔드포인트
@router.post('/add-documents', summary='문서 추가 및 인덱스 갱신')
async def add_documents(documents: list):
    """동적으로 문서를 추가하고 인덱스를 재구축합니다."""
    try:
        similarity_service.add_documents(documents)  # 문서 목록에 추가
        similarity_service.build_index()             # 임베딩/인덱스 재구축
        return {
            'message': f'{len(documents)}개 문서 추가 완료',
            'total_documents': len(similarity_service.documents),
        }
    except Exception as e:
        raise HTTPException(status_code=500, detail=str(e))


# 인덱스/모델의 현재 상태를 조회하는 통계 엔드포인트
@router.get('/stats', summary='인덱스 통계 정보')
async def get_index_stats():
    return {
        'total_documents': len(similarity_service.documents),  # 등록 문서 수
        'model_loaded': similarity_service.model is not None,  # 모델 로드 여부
        'index_built': similarity_service.index is not None,   # 인덱스 구축 여부
        'embedding_dim': similarity_service.embedding_dim,     # 임베딩 차원
        'faiss_available': FAISS_AVAILABLE,                    # FAISS 사용 가능 여부
    }
```

## 3.5 Swagger UI로 NLP API 체험하기

프론트엔드 개발 경험이 없더라도, FastAPI가 자동으로 생성해주는 Swagger UI를 통해 모든 API를 브라우저에서 직접 체험할 수 있습니다. 이 절에서는 Swagger UI를 최대한 활용하는 방법과 API 사용자 경험을 향상시키는 다양한 설정 방법을 다룹니다.

### 3.5.1 Swagger UI 기본 사용법 복습

FastAPI 서버를 실행한 후 브라우저에서 `http://localhost:8000/docs` 로 접속하면 Swagger UI가 열립니다. Swagger UI의 주요 구성 요소와 사용 방법을 정리합니다.

| Swagger UI 구성 요소 | 설명 및 활용 방법 |
| --- | --- |
| API 제목 및 버전 | `FastAPI()` 생성 시 title, version, description 파라미터로 설정 |
| 태그(Tags) 그룹 | 라우터의 tags 파라미터로 API를 카테고리별 그룹화 |
| 엔드포인트 목록 | HTTP 메서드(POST/GET 등)와 URL 경로 표시. 클릭하면 확장됨. |
| Try it out 버튼 | 클릭하면 입력 필드가 활성화되어 API 직접 실행 가능 |
| Request Body | JSON 형식 입력. example 값이 미리 채워져 있으면 편리함. |
| Parameters | 경로 매개변수, 쿼리 매개변수를 개별 입력란으로 표시 |
| Execute 버튼 | API 요청 전송. curl 명령어도 함께 표시해줌. |
| Response 영역 | HTTP 상태 코드, 응답 헤더, 응답 Body를 보기 좋게 출력 |
| Schemas 섹션 | Pydantic 모델의 JSON Schema를 하단에 자동 표시 |

### 3.5.2 POST, GET 요청 실습 방법

Swagger UI에서 각 HTTP 메서드별 요청을 실행하는 방법을 구체적으로 설명합니다.

**GET 요청 실습 방법**

1. Swagger UI에서 GET /chat/health 엔드포인트를 클릭합니다.
2. 'Try it out' 버튼을 클릭하여 활성화합니다.
3. 'Execute' 버튼을 클릭합니다.
4. 'Responses' 섹션에서 HTTP 상태 코드와 JSON 응답을 확인합니다.
5. 'Curl' 섹션에 자동 생성된 curl 명령어를 복사해 터미널에서도 테스트해보세요.

**POST 요청 실습 방법**

6. POST /chat/ 엔드포인트를 클릭합니다.
7. 'Try it out' 버튼을 클릭합니다.
8. 'Request body' 텍스트 영역에 이미 채워진 example 값을 확인합니다.
9. 필요한 경우 값을 수정합니다 (예: message 변경, language 변경 등).
10. 'Execute'를 클릭하고 응답 결과를 확인합니다.

**Swagger UI 활용 팁**

1. /docs 우측 상단 'Authorize' 버튼: JWT 토큰 등 인증이 필요한 API 테스트 시 사용
2. Schemas 섹션: 하단에 모든 Pydantic 모델의 JSON Schema 자동 문서화
3. 응답 예시 'curl' 복사: 자동 생성된 curl을 복사해 CI/CD 테스트 스크립트로 활용
4. Response 200: 성공 응답 스키마 미리 확인 가능 (response_model 지정 시)
5. Expand All/Collapse All: 스크롤 줄이기 위한 전체 접기/펼치기

### 3.5.4 각 API마다 response 모델 지정해서 명확한 출력 제공

response_model을 명시적으로 지정하면 Swagger UI에서 예상 응답 스키마를 미리 확인할 수 있고, FastAPI가 응답 데이터를 자동으로 검증·직렬화합니다. 또한 민감한 필드를 응답에서 자동으로 제외하는 용도로도 활용됩니다.

```python
from fastapi import APIRouter
from pydantic import BaseModel
from typing import List, Optional


# ─── 응답 모델 계층 구조 ───────────────────────────
# 공통 필드를 가진 베이스를 두고 상속으로 중복을 줄인다
class BaseResponse(BaseModel):
    """모든 API 응답의 기본 클래스"""
    success: bool = True   # 성공 여부 기본값
    message: str = 'OK'    # 상태 메시지 기본값


# BaseResponse를 상속해 success/message를 그대로 물려받음
class ChatResponse(BaseResponse):
    user_message: str
    bot_response: str
    language: str
    model_used: str


# 오류 응답 전용 스키마
class ErrorResponse(BaseModel):
    success: bool = False
    error_code: str
    error_message: str
    details: Optional[dict] = None  # 추가 상세 정보(선택)


# ─── response_model과 responses 조합 ───────────────
# responses: 상태 코드별 응답 스키마/예시를 문서에 명시 -> Swagger가 모두 보여줌
@router.post(
    '/',
    response_model=ChatResponse,  # 기본(200) 성공 응답 스키마
    responses={
        200: {
            'description': '챗봇 응답 성공',
            'model': ChatResponse,
        },
        422: {  # 입력 검증 실패 시 반환 형태 예시
            'description': '입력 데이터 유효성 오류',
            'content': {
                'application/json': {
                    'example': {
                        'detail': [{'loc': ['body', 'message'], 'msg': 'value too short'}]
                    }
                }
            }
        },
        503: {  # 모델 미로드 등 서비스 불가
            'description': '모델 로드 실패',
            'model': ErrorResponse,
        },
    },
    summary='챗봇 응답 생성',
    description='**사용법**: message에 텍스트 입력, language로 언어 선택(ko/en).',
)
async def chat(request: ChatRequest) -> ChatResponse:
    ...  # 본문 로직은 생략(문서화 설정 예시)
```

### 3.5.5 브라우저에서 쉽게 체험하며 API 개념 숙지

| 개발 시 사용 | API 테스트 | 레퍼런스 문서 확인 |
| 팀 공유 | 프로토타입 시연 | API 명세 공유 |

- T5/BART 기반 요약 모델로 문서 요약 API를 만들고 파라미터 조정법을 배웠습니다.
- MarianMT로 6가지 언어 쌍 번역 API를 구현하고 배치 번역 기능을 추가했습니다.
- sentence-transformers + FAISS로 의미 기반 문서 유사도 검색 시스템을 구축했습니다.
- Swagger UI를 통해 프론트엔드 없이 모든 API를 체험하고 문서화하는 방법을 익혔습니다.
- Pydantic BaseModel로 요청/응답 스키마를 정의하고 자동 유효성 검사를 적용했습니다.
- 서비스 레이어 분리 구조로 유지보수 가능한 FastAPI 프로젝트를 완성했습니다.

6. [코딩 실습] 이 장의 번역 API를 확장하여, 입력 텍스트와 함께 glossary(전문용어 사전) 딕셔너리를 받아서 번역 후 특정 단어를 치환해주는 POST /translate/with-glossary 엔드포인트를 구현하세요.

### 3장 요약

이 장에서는 Hugging Face의 다양한 NLP 모델을 FastAPI와 결합하여 4가지 핵심 웹 서비스를 구현했습니다.

- **챗봇 API**: DialoGPT(영어)와 KoGPT2(한국어)를 활용한 무상태 텍스트 생성 챗봇. 생성 전략(Greedy/Beam/Sampling)과 temperature, top-p 파라미터의 역할을 이해했습니다.
- **요약기 API**: BART(영어)와 ke-t5(한국어)를 활용한 Seq2Seq 요약 서비스. T5의 prefix 기반 태스크 통합 철학과 max_length, num_beams 파라미터 조정 방법을 학습했습니다.
- **번역 API**: MarianMT 기반 다국어 번역 서비스. 단일/배치 번역 엔드포인트 구성과 언어 쌍별 모델 관리 방법을 익혔습니다.
- **유사도 검색 API**: sentence-transformers로 문장 임베딩을 생성하고 FAISS로 고속 유사도 검색을 구현. 코사인 유사도 기반 의미 검색의 원리를 이해했습니다.
- **Swagger UI 활용**: response_model, example, tags, description을 활용하여 사용하기 쉬운 API 문서를 자동 생성하는 방법을 학습했습니다.
